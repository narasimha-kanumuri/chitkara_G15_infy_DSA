# Quick Solution — Pebble Pulse Trail

## Idea (Simple)

The frog must reach the last stone with minimum energy.
From each stone, it can jump `1` step or `2` steps.
A jump costs the height difference between the current and next stone.
If a `2-step` jump comes immediately after another `2-step` jump, we add the fatigue penalty.

## Pattern Recognition Clues

- The problem asks for minimum energy, so this is optimization recursion.
- Each stone creates choices: jump `1` or jump `2`.
- The cost of a `2-step` jump may depend on the previous jump.
- This means the recursion needs extra state for the previous jump type.

## State Discovery

The simple state is `solve(i)`: minimum energy from stone `i` to the last stone.
But `solve(i)` is not enough.
It does not remember whether the previous jump was a `2-step` jump.
That information is needed to decide whether the next `2-step` jump gets a penalty.
So the final state is:

- `solve(i, lastWasTwo)`

Here, `i` is the current stone.
`lastWasTwo` tells whether the jump used to reach stone `i` was a `2-step` jump.

## Pattern

Optimization Recursion with Previous Move State

## Core Insight

- A `1-step` jump never creates fatigue penalty.
- A `2-step` jump creates penalty only if `lastWasTwo` is already true.
- For each allowed jump, compute jump cost + best future cost, then take the minimum.

## Invariant

For every call `solve(i, lastWasTwo)`, the function returns the minimum extra energy needed to reach the last stone from stone `i`, assuming `lastWasTwo` correctly describes the previous jump.

## Algorithm (with one-line why at important steps)

1. Start recursion from stone `0` with `lastWasTwo = false`.
   - Why: no jump was taken before the first stone.
2. If `i == n - 1`, return `0`.
   - Why: the frog already reached the destination.
3. Try a `1-step` jump if `i + 1` is inside the trail.
   - Why: this is one valid movement choice.
4. Add `abs(height[i] - height[i + 1])` for the `1-step` jump.
   - Why: energy depends on height difference.
5. Try a `2-step` jump if `i + 2` is inside the trail.
   - Why: this is the second valid movement choice.
6. Add the fatigue penalty only when `lastWasTwo` is true.
   - Why: penalty applies only to consecutive `2-step` jumps.
7. Return the smaller value among valid choices.
   - Why: the task asks for minimum energy.

## Mental Cheat Sheet

| Item | Meaning |
|---|---|
| Pattern | Optimization recursion |
| State | Current stone + whether previous jump was `2` |
| Choice | Jump `1` or jump `2` |
| Penalty rule | Add `p` only for consecutive `2-step` jumps |
| Answer | `solve(0, false)` |

## Complexity

- Time Complexity: `O(2^n)` in the worst case without memoization
- Space Complexity: `O(n)` because of recursion depth
- Auxiliary Space: `O(n)` for the recursion call stack

## Tiny Dry Run

For `heights = [10, 20, 30, 10]` and `p = 10`:

- Path `0 -> 1 -> 3` has cost `10 + 10 = 20`.
- Path `0 -> 2 -> 3` has cost `20 + 20 = 40`.
- Path `0 -> 1 -> 2 -> 3` has cost `10 + 10 + 20 = 40`.

Answer = `20`.

## Notes / Pitfalls

- Do not add fatigue penalty after every `2-step` jump.
- Add fatigue only when the previous jump was also `2-step`.
- Do not jump beyond the last stone.
- Use `min`, not addition across alternatives, because this is an optimization problem.
