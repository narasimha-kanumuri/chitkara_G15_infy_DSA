Java

### Complete Solution Code

```java

import java.io.BufferedReader;

import java.io.InputStreamReader;

import java.math.BigInteger;

public class Main {

private static BigInteger countWays(int n) {

BigInteger[] endWithOne = new BigInteger[n + 1];

BigInteger[] endWithTwo = new BigInteger[n + 1];

for (int i = 0; i <= n; i++) {

endWithOne[i] = BigInteger.ZERO;

endWithTwo[i] = BigInteger.ZERO;

}

endWithOne[1] = BigInteger.ONE;

if (n >= 2) {

endWithTwo[2] = BigInteger.ONE;

}

for (int i = 2; i <= n; i++) {

endWithOne[i] = endWithOne[i - 1].add(endWithTwo[i - 1]);

if (i >= 3) {

endWithTwo[i] = endWithOne[i - 2];

}

}

return endWithOne[n].add(endWithTwo[n]);

}

public static void main(String[] args) throws Exception {

BufferedReader br = new BufferedReader(new InputStreamReader(System.in));

int n = Integer.parseInt(br.readLine().trim());

System.out.println(countWays(n));

}

}
```


### Code Deconstruction

- `countWays(int n)` is the main DP function.
  - It returns the exact number of valid jump sequences.
- `endWithOne[i]`
  - Stores ways to reach step `i` where the last jump is `1`.
- `endWithTwo[i]`
  - Stores ways to reach step `i` where the last jump is `2`.
- Base setup:
  - `endWithOne[1] = 1` handles the only way to reach step `1`.
  - `endWithTwo[2] = 1` handles the direct jump from ground to step `2`.
- Transition for `endWithOne[i]`
  - `endWithOne[i - 1] + endWithTwo[i - 1]`
  - A final `1` can follow any valid path to the previous step.
- Transition for `endWithTwo[i]`
  - `endWithOne[i - 2]`
  - A final `2` cannot follow another `2`, so only paths ending with `1` are allowed.
- Final return:
  - `endWithOne[n] + endWithTwo[n]`
  - All valid paths must end with one of these two last-move states.
- `BigInteger`
  - Used because the count can become too large for `long`.

### Language-Specific Notes

- `BigInteger` is necessary when exact counts may exceed primitive numeric limits.
- Use `.add(...)` instead of `+` for `BigInteger` values.
- Array slots must be filled with `BigInteger.ZERO` before use.
- The code is intentionally kept block-based and student-readable.
