# Coupon Memo Minimizer — Quick Solution

## Idea (Simple)

We need the minimum toll to finish the checkpoint journey.
At each landed checkpoint, we can either pay its toll or use the one coupon if it is still unused.
Then we may move `1` or `2` checkpoints forward.

Because the same checkpoint can be reached with the coupon used or unused, we use memoization with coupon state.

## Pattern Recognition Clues

- The problem asks for a minimum cost.
- The same future checkpoint can be reached from different previous routes.
- A one-time coupon changes future choices.
- Repeated recursive paths suggest top-down DP with memoization.

## State Discovery

The first simple state is `solve(i)`.

But `solve(i)` is not enough.
It only tells the current checkpoint.
It does not tell whether the coupon is still available.

That missing information changes the best future cost.

So we use:

```text
solve(i, used)
```

Meaning:

```text
minimum cost to finish from checkpoint i,
where used tells whether the coupon has already been used
```

## Pattern

Top-Down DP with Coupon State

## Core Insight

- Paying the current toll is always allowed.
- Using the coupon is allowed only when it has not been used before.
- After handling the current checkpoint, choose the cheaper of moving to `i + 1` or `i + 2`.
- Cache every `(i, used)` state.

## Invariant

For every state `solve(i, used)`:

- The agent is currently at checkpoint `i`.
- `used = 0` means the coupon is still available.
- `used = 1` means the coupon cannot be used again.
- The function returns the minimum remaining toll from that state.

## Algorithm (with one-line why at important steps)

1. Start with `solve(0, 0)`.
   - Why: the first move lands on checkpoint `0`, and the coupon is unused.
2. If `i >= n`, return `0`.
   - Why: moving beyond the last checkpoint completes the journey.
3. If the state is already cached, return it.
   - Why: the same `(i, used)` can appear many times.
4. Compute the cost if we pay `cost[i]`.
   - Why: paying is always a valid choice.
5. If `used == 0`, compute the cost if we use the coupon here.
   - Why: this cancels the current toll and changes coupon state.
6. Store and return the smaller value.
   - Why: the goal is minimum total toll.

## Mental Cheat Sheet

| Item | Meaning |
|---|---|
| Pattern | Memoized min-cost DP |
| State | `(i, used)` |
| `i` | current checkpoint |
| `used` | whether coupon is already used |
| Choices | pay current toll or use coupon if available |
| Answer | `solve(0, 0)` |

## Complexity

- Time Complexity: `O(n)`
- Space Complexity: `O(n)`
- Auxiliary Space: `O(n)` for memoization and recursion stack

There are only `2 * n` useful states.
Each state performs constant work.

## Tiny Dry Run

For:

```text
cost = [5, 100, 5]
```

Start:

```text
solve(0, 0)
```

Main choices:

```text
Pay checkpoint 0: 5 + min(solve(1,0), solve(2,0))
Use coupon at checkpoint 0: 0 + min(solve(1,1), solve(2,1))
```

The best final answer is `5`.

## Notes / Pitfalls

- Do not cache only by checkpoint index.
- The coupon-used status must be part of the state.
- Moving beyond the last checkpoint adds `0` cost.
- Use `long` because toll values can be large.
- A valid minimum can be `0`, so use a separate visited table.
