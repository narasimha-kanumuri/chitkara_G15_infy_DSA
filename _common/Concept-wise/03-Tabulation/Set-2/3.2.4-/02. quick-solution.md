## Idea (Simple)
We count ways to reach each step from the ground.
Locked steps cannot receive any ways.
From a normal step, we can move `1` or `2` steps.
From a prime-numbered step, the next move must be only `1` step.

## Pattern Recognition Clues
- We are counting valid ways.
- Each step receives ways from earlier steps.
- Some steps are blocked, so transitions must skip them.
- A prime-numbered current step changes which moves are allowed.

## State Discovery
The first idea is `dp[i]`: number of ways to reach step `i`.
This is enough here.
The prime rule depends only on the current step number.
The locked rule depends only on whether the target step is locked.
No hidden past choice is needed.
So the final state is:
- `dp[i]` = number of valid ways to stand on step `i`.

## Pattern
Bottom-Up Counting DP with Constraint-Based Transitions

## Core Insight
- Locked steps should never receive ways.
- A normal current step can send ways to `i + 1` and `i + 2`.
- A prime current step can send ways only to `i + 1`.

## Invariant
After processing step `i`, all valid ways that can move out of steps `0` through `i` have been added to reachable unlocked future steps.
Every `dp[x]` contains only paths that respect locked-step and prime-step rules.

## Algorithm (with one-line why at important steps)
1. Mark all locked steps in a boolean array.
   - Why: we need fast checking before landing on a step.
2. Precompute prime numbers up to `n`.
   - Why: the allowed next move depends on whether the current step is prime.
3. Set `dp[0] = 1`.
   - Why: there is one way to start on the ground.
4. Iterate from step `0` to step `n`.
   - Why: smaller positions build larger positions in bottom-up order.
5. If the current step is locked, skip it.
   - Why: no valid path may stand on a locked step.
6. If the current step is prime, try only a `1`-step move.
   - Why: prime steps force the next move to be exactly `1`.
7. Otherwise, try both `1`-step and `2`-step moves.
   - Why: normal steps allow both moves.
8. Add ways only to unlocked target steps within range.
   - Why: landing outside the staircase or on locked steps is invalid.
9. Print `dp[n]`.
   - Why: it stores all valid ways to reach the target step.

## Mental Cheat Sheet
| Item | Meaning |
|---|---|
| Pattern | Counting DP with blocked and forced transitions |
| State | `dp[i] = ways to reach step i` |
| Locked step | Cannot receive or send valid paths |
| Prime step | Allows only `+1` move |
| Normal step | Allows `+1` and `+2` moves |
| Answer | `dp[n]` |

## Complexity
- Time Complexity: `O(n log log n)` because of prime precomputation
- Space Complexity: `O(n)`
- Auxiliary Space: `O(n)`

## Tiny Dry Run
For `n = 4`, no locked steps:
- Start: `dp[0] = 1`
- From `0`: add to `1` and `2`
- From `1`: add to `2` and `3`
- From `2`: prime, add only to `3`
- From `3`: prime, add only to `4`
- Answer = `dp[4] = 3`

## Notes / Pitfalls
- Step `1` is not prime.
- Step `0` is not prime.
- Do not add ways to locked steps.
- From a prime step, do not allow a `2`-step move.
- Use `BigInteger` for exact large counts.
