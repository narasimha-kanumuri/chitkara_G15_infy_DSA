# Silent Sequence Cache — Java Solution

## Language

Java

## Plain-English Implementation Walkthrough

1. We use `BigInteger` because the number of valid sequences can become very large.

2. We store the restriction pattern in a `String` named `blocked`.
   This lets us check each position using `charAt(pos)`.

3. We create a recursive method `solve(pos, lastBit, n, blocked, memo)`.
   It returns the number of valid ways to fill the sequence from position `pos` onward.

4. The parameter `pos` tells which position we are filling now.
   This changes after every placed bit.

5. The parameter `lastBit` tells what bit was placed before this position.
   This is needed because `1` cannot follow `1`.

6. If `pos == n`, the whole sequence is filled.
   We return `1` only if the last placed bit is `0`.

7. We check the memo table before doing the recursive work.
   If the answer already exists, we return it directly.

8. We always try placing `0`.
   This is valid for every position, including restricted positions.

9. We try placing `1` only when two conditions are true.
   The previous bit must not be `1`, and the current position must not be restricted.

10. We add the counts from all valid choices.
    This works because the task asks for the total number of valid sequences.

11. We save the total in `memo[pos][lastBit]`.
    This makes repeated states fast.

12. The final answer is `solve(0, 0, n, blocked, memo)`.
    At the start, there is no previous `1`, so `lastBit` is set to `0`.

## Complete Solution Code

```java
import java.io.BufferedReader;
import java.io.InputStreamReader;
import java.math.BigInteger;

public class Main {

    private static BigInteger solve(int pos, int lastBit, int n,
                                    String blocked, BigInteger[][] memo) {
        if (pos == n) {
            return lastBit == 0 ? BigInteger.ONE : BigInteger.ZERO;
        }

        if (memo[pos][lastBit] != null) {
            return memo[pos][lastBit];
        }

        BigInteger total = BigInteger.ZERO;

        // Place 0. This is always allowed.
        total = total.add(solve(pos + 1, 0, n, blocked, memo));

        // Place 1 only if previous bit is not 1 and this position is not restricted.
        if (lastBit == 0 && blocked.charAt(pos) == '0') {
            total = total.add(solve(pos + 1, 1, n, blocked, memo));
        }

        memo[pos][lastBit] = total;
        return total;
    }

    private static BigInteger countValidSequences(int n, String blocked) {
        BigInteger[][] memo = new BigInteger[n + 1][2];
        return solve(0, 0, n, blocked, memo);
    }

    public static void main(String[] args) throws Exception {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));

        int n = Integer.parseInt(br.readLine().trim());
        String blocked = br.readLine().trim();

        System.out.println(countValidSequences(n, blocked));
    }
}
```

## Code Deconstruction

### `solve` method

```java
private static BigInteger solve(int pos, int lastBit, int n,
                                String blocked, BigInteger[][] memo)
```

This method represents one memoized DP state.

- `pos` is the current position.
- `lastBit` is the previous placed bit.
- `blocked` stores which positions cannot contain `1`.
- `memo` stores solved states.

### Completed sequence case

```java
if (pos == n) {
    return lastBit == 0 ? BigInteger.ONE : BigInteger.ZERO;
}
```

This checks the ending rule.
A completed sequence is counted only if it ends with `0`.

### Memoization check

```java
if (memo[pos][lastBit] != null) {
    return memo[pos][lastBit];
}
```

This avoids solving the same `(pos, lastBit)` state again.

### Placing `0`

```java
total = total.add(solve(pos + 1, 0, n, blocked, memo));
```

A `0` can be placed anywhere.
After placing it, the new last bit becomes `0`.

### Placing `1`

```java
if (lastBit == 0 && blocked.charAt(pos) == '0') {
```

This condition enforces both rules for placing `1`.
It prevents consecutive `1`s and prevents `1` on restricted positions.

### Saving result

```java
memo[pos][lastBit] = total;
return total;
```

The computed answer is saved before returning.

## Language-Specific Notes

- `BigInteger` is immutable. Use `.add()` to combine counts.
- Do not use `+` for `BigInteger` addition.
- `memo[pos][lastBit] == null` means the state has not been solved yet.
- `charAt(pos)` reads the restriction character at the current position.
- The recursion depth can reach `n`; this is acceptable for memoization practice, but tabulation can avoid stack depth.

## Complexity Summary

- Time Complexity: `O(n)`
- Space Complexity: `O(n)`
- Auxiliary Space: `O(n)`

There are at most `2 * (n + 1)` states.
Each state performs constant work, except for `BigInteger` digit cost.
