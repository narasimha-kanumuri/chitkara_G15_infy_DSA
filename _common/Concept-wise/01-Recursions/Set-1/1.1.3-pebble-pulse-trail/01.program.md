# Pebble Pulse Trail

## Concepts Involved

- Base Case Design
- Return Meaning
- Choice Expansion
- Recursive State Definition
- Recursion Tree Analysis
- Constraint Propagation
- Optimization Recursion
- Cost-Based Decision Making

## Problem Statement

A frog starts on the first stone of a pebble trail.
There are `n` stones, numbered from `0` to `n - 1`.
Each stone has a height value.

From stone `i`, the frog can jump to:

- stone `i + 1`, or
- stone `i + 2`

The energy cost of a jump is the absolute difference between the heights of the starting stone and the landing stone.

There is one extra trail rule:

If the frog takes two `2-step` jumps consecutively, the second consecutive `2-step` jump adds an extra fatigue penalty `p`.

Your task is to return the minimum total energy required to reach the last stone.

A path is valid only if:

- The frog starts at stone `0`.
- The frog must end exactly at stone `n - 1`.
- Every jump is either `1` step or `2` steps.
- A fatigue penalty is added only when a `2-step` jump immediately follows another `2-step` jump.

## Input Format

```text
n p
h0 h1 h2 ... h(n-1)
```

Where:

- `n` is the number of stones.
- `p` is the fatigue penalty.
- `h[i]` is the height of stone `i`.

## Output Format

Print one integer: the minimum total energy required to reach the last stone.

## Constraints

```text
1 <= n <= 30
0 <= p <= 10000
0 <= h[i] <= 10000
```

## Examples

### Example 1

Input:

```text
4 10

```

Output:

```text
20
```

### Example 2

```text

```

## Edge-Case Thinking Prompts

- What should the answer be when there is only one stone?
- When exactly should the fatigue penalty be added?
- Why is jumping outside the trail invalid?
- Why is it not enough to know only the current stone?
- Is this a counting problem or an optimization problem?
