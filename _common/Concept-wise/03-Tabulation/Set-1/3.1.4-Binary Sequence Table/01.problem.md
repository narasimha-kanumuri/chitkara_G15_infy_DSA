## Day 03 — Tabulation (Bottom-Up DP) — Problem 4

## problem.md

### Title
Binary Sequence Table

### Concepts Involved
Tabulation, State Tracking, Counting DP

### Problem Statement
A security system generates a binary access pattern of length `n`.

Each position may contain either `0` or `1`.
However, the generated pattern must follow two rules:

- It must not contain two consecutive `1`s.
- It must end with `0`.

Find the total number of valid binary patterns of length `n`.

A pattern is considered different if at least one position contains a different bit.
The answer must be printed exactly, not modulo any number.

### Input Format
- A single integer `n`

### Output Format
- Print the total number of valid binary patterns of length `n`

### Constraints
- `1 <= n <= 5000`

### Examples
#### Example 1
Input:
`1`

Output:
`1`

#### Example 2
Input:
`3`

Output:
`3`

#### Example 3
Input:
`4`

Output:
`5`

### Edge-Case Thinking Prompts
- What happens when `n = 1`?
- Why is `11` invalid?
- Why must the final state specifically end with `0`?
- Can a sequence ending in `1` still help build future answers?

### Thinking Like a Programmer
- What information must be remembered at every position?
- Is it enough to store only the total number of valid sequences?
- Why do we split sequences based on the last bit?
- How does bottom-up DP replace recursive construction?
