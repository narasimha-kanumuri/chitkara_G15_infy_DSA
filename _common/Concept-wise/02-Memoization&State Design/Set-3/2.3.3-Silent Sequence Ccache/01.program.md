# Silent Sequence Cache

## Concepts Involved

Memoization, State Design, Counting DP, Binary Sequence Rules, Recursion to DP

## Problem Statement

A guard is building a silent binary sequence of length `n`.

Each position may contain either `0` or `1`.

The sequence must follow these rules:

1. It must not contain two consecutive `1`s.
2. It must end with `0`.
3. Some positions are restricted and do not allow `1`.

The positions are numbered from `0` to `n - 1`.
You are given a binary restriction string `blocked` of length `n`:

- `0` means this position may contain `0` or `1`.
- `1` means this position cannot contain `1`; it must contain `0`.

A plain recursive solution will repeatedly solve the same suffix-building states.
Use memoization to avoid recomputation.

Return the total number of valid sequences.

## Input Format

- The first line contains an integer `n`.
- The second line contains a binary string `blocked` of length `n`.

## Output Format

- Print the total number of valid silent binary sequences.

## Constraints

- `1 <= n <= 2000`
- `blocked.length == n`
- `blocked[i]` is either `0` or `1`
- The answer may be very large.
- Use exact large number arithmetic.

## Examples

### Example 1

Input:

```text
3
000
```

Output:

```text
3
```

Explanation:

The valid sequences are:

```text
000
010
100
```

The sequence `101` does not end with `0`, so it is not counted.
The sequence `110` has consecutive `1`s, so it is invalid.

### Example 2

Input:

```text
4
0100
```

Output:

```text
3
```

Explanation:

Position `1` cannot contain `1`.
The valid sequences are:

```text
0000
1000
0010
```

The sequence `0100` is not allowed because position `1` is restricted.

### Example 3

Input:

```text
1
0
```

Output:

```text
1
```

Explanation:

The only valid sequence is `0` because the sequence must end with `0`.

### Example 4

Input:

```text
5
00101
```

Output:

```text
6
```

Explanation:

Positions `2` and `4` cannot contain `1`.
The final position is also restricted, so it must be `0`.
All valid sequences that respect the restrictions and avoid consecutive `1`s are counted.

## Edge-Case Thinking Prompts

- What should the answer be when `n = 1`?
- Why is a sequence ending in `1` not counted?
- Why does the previous bit affect the next choice?
- What changes when a position is restricted?
- Is the current position alone enough to decide valid completions?

The current position alone is not enough.
We must know the previous bit because a `1` cannot be placed after another `1`.
So the memoized state must include both the current position and the last placed bit.
