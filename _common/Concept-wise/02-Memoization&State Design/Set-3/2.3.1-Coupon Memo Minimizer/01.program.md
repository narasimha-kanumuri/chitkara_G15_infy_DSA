# Coupon Memo Minimizer

## Concepts Involved

Memoization, State Design, Optimization DP, Coupon State, Recursion to DP

## Problem Statement

A delivery agent has to cross a row of `n` checkpoints.

Each checkpoint `i` has a toll value `cost[i]`.
The agent starts before checkpoint `0`, but the first move must land on checkpoint `0`.

After landing on a checkpoint, the agent may move either:

- `1` checkpoint forward
- `2` checkpoints forward

If the agent moves beyond checkpoint `n - 1`, the journey is complete.

Every landed checkpoint normally adds its toll to the total cost.
The agent has one coupon that can cancel the toll of exactly one landed checkpoint.
The coupon may be used at most once.

A plain recursive solution recomputes many repeated states.
Use memoization with coupon usage as part of the state.

Return the minimum total toll needed to complete the journey.

## Input Format

- The first line contains an integer `n`.
- The second line contains `n` integers, where `cost[i]` is the toll value of checkpoint `i`.

## Output Format

- Print the minimum total toll needed to move beyond the last checkpoint.

## Constraints

- `1 <= n <= 2000`
- `1 <= cost[i] <= 10^9`
- The answer fits in a signed 64-bit integer.

## Examples

### Example 1

Input:

```text
5
10 15 20 5 10
```

Output:

```text
15
```

Explanation:

One optimal route is:

```text
0 -> 1 -> 3 -> finish
```

The normal toll is:

```text
10 + 15 + 5 = 30
```

Use the coupon on checkpoint `1`.
The paid toll becomes:

```text
10 + 0 + 5 = 15
```

### Example 2

Input:

```text
3
5 100 5
```

Output:

```text
5
```

Explanation:

The agent must first land on checkpoint `0`.
A valid optimal route is:

```text
0 -> 2 -> finish
```

Use the coupon on checkpoint `2`.
The paid toll becomes:

```text
5 + 0 = 5
```

### Example 3

Input:

```text
1
8
```

Output:

```text
0
```

Explanation:

The agent lands on the only checkpoint and uses the coupon there.
So the total paid toll is `0`.

## Edge-Case Thinking Prompts

- What should happen when `n = 1`?
- Why is the first checkpoint always considered landed?
- Why is the checkpoint index alone not enough as a memoization state?
- What changes after the coupon is used?
- Why does moving beyond the last checkpoint add no cost?

The important state is not only the current checkpoint.

So the cache key must include both the checkpoint index and coupon-used status.
