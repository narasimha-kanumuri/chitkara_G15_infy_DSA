# Frog Memo Jump

## Concepts Involved

Memoization, State Design, Optimization DP, Recursion to DP, Minimum Energy Path

## Problem Statement

A frog is standing on stone `0` and wants to reach stone `n - 1`.

There are `n` stones in a straight line.
The height of stone `i` is given as `height[i]`.

From stone `i`, the frog may jump to:

- stone `i + 1`
- stone `i + 2`

The energy cost of a jump from stone `i` to stone `j` is:

```text
abs(height[i] - height[j])
```

There is one extra fatigue rule.
If the frog takes two `2-step` jumps consecutively, an extra fatigue penalty `p` is added on the second consecutive `2-step` jump.

Use memoization to avoid recomputing repeated subproblems.
Return the minimum total energy required to reach the last stone.

## Input Format

- The first line contains two integers `n` and `p`.
- The second line contains `n` integers, where `height[i]` is the height of stone `i`.

## Output Format

- Print the minimum total energy required to reach stone `n - 1`.

## Constraints

- `1 <= n <= 2000`
- `0 <= p <= 10^9`
- `0 <= height[i] <= 10^9`
- The answer fits in a signed 64-bit integer.

## Examples

### Example 1

Input:

```text
4 10
10 30 40 20
```

Output:

```text
30
```

Explanation:

One optimal path is:

```text
0 -> 1 -> 3
```

Energy:

```text
abs(10 - 30) + abs(30 - 20) = 20 + 10 = 30
```

### Example 2

Input:

```text
5 100
10 20 10 20 10
```

Output:

```text
0
```

Explanation:

The frog can jump:

```text
0 -> 2 -> 4
```

Both jumps have height difference `0`.
The second jump is also a consecutive `2-step` jump, so penalty `100` would apply.
But another route can avoid that penalty:

```text
0 -> 2 -> 3 -> 4
```

Energy:

```text
0 + 10 + 10 = 20
```

For this input, there is also a better route:

```text
0 -> 1 -> 3 -> 4
```

Energy:

```text
10 + 0 + 10 = 20
```

So the minimum cannot be obtained by simply choosing local height differences.

### Example 3

Input:

```text
1 50
7
```

Output:

```text
0
```

Explanation:

The frog is already on the last stone.
No jump is needed.

## Edge-Case Thinking Prompts

- What should the answer be when `n = 1`?
- Why is the current stone index not always enough as a state?
- What changes when the previous jump was a `2-step` jump?
- When should the fatigue penalty be added?
- Can a locally cheapest jump lead to a globally expensive path?
