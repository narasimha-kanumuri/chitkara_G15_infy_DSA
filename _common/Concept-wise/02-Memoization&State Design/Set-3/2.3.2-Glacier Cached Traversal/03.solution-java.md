## Plain-English Implementation Walkthrough

1. We store all platform heights in a `long[]` array.
   This helps compute height-difference costs quickly.

2. We store blue-platform information in a `boolean[] blue` array.
   This makes checking landing cold cost simple.

3. We use the method `solve(i, prevTwo, height, blue, coldCost, memo, seen)`.
   It returns the minimum energy needed from platform `i` to the final platform.

4. The parameter `i` stores the current platform.
   Every future jump starts from this platform.

5. The parameter `prevTwo` stores whether the previous jump was length `2`.
   This is required because another `2-step` jump may be blocked.

6. If `i == height.length - 1`, we return `0`.
   The climber has already reached the final platform.

7. We check `seen[i][prevTwo]` before using `memo[i][prevTwo]`.
   This is needed because a valid minimum answer can be `0`.

8. We first try a `1-step` jump if the destination exists.
   This jump is always allowed.

9. When computing jump cost, we add the height difference.
   If the destination platform is blue, we add `coldCost` too.

10. After a `1-step` jump, the next state gets `prevTwo = 0`.
    This is because the latest jump is not length `2`.

11. We try a `2-step` jump only when `prevTwo == 0`.
    This enforces the rule that two `2-step` jumps cannot be consecutive.

12. After a `2-step` jump, the next state gets `prevTwo = 1`.
    This tells the next state that the latest jump was length `2`.

13. We save the smaller valid cost in the memo table.
    If the same state appears again, the saved value is returned.

14. The final answer comes from `solve(0, 0, ...)`.
    There is no previous jump at the starting platform.

## Complete Solution Code

```java
import java.io.BufferedReader;
import java.io.InputStreamReader;
import java.util.StringTokenizer;

public class Main {

    private static final long INF = Long.MAX_VALUE / 4;

    private static long landingCost(int from, int to, long[] height, boolean[] blue, long coldCost) {
        long cost = Math.abs(height[from] - height[to]);
        if (blue[to]) {
            cost += coldCost;
        }
        return cost;
    }

    private static long solve(int i, int prevTwo, long[] height, boolean[] blue,
                              long coldCost, long[][] memo, boolean[][] seen) {
        int n = height.length;

        if (i == n - 1) {
            return 0L;
        }

        if (seen[i][prevTwo]) {
            return memo[i][prevTwo];
        }

        long best = INF;

        if (i + 1 < n) {
            long cost = landingCost(i, i + 1, height, blue, coldCost);
            long total = cost + solve(i + 1, 0, height, blue, coldCost, memo, seen);
            best = Math.min(best, total);
        }

        if (prevTwo == 0 && i + 2 < n) {
            long cost = landingCost(i, i + 2, height, blue, coldCost);
            long total = cost + solve(i + 2, 1, height, blue, coldCost, memo, seen);
            best = Math.min(best, total);
        }

        seen[i][prevTwo] = true;
        memo[i][prevTwo] = best;
        return best;
    }

    private static long minimumEnergy(long[] height, boolean[] blue, long coldCost) {
        int n = height.length;
        long[][] memo = new long[n][2];
        boolean[][] seen = new boolean[n][2];
        long answer = solve(0, 0, height, blue, coldCost, memo, seen);
        return answer >= INF ? -1L : answer;
    }

    public static void main(String[] args) throws Exception {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));

        StringTokenizer first = new StringTokenizer(br.readLine());
        int n = Integer.parseInt(first.nextToken());
        long coldCost = Long.parseLong(first.nextToken());

        long[] height = new long[n];
        StringTokenizer second = new StringTokenizer(br.readLine());
        for (int i = 0; i < n; i++) {
            height[i] = Long.parseLong(second.nextToken());
        }

        String types = br.readLine().trim();
        boolean[] blue = new boolean[n];
        for (int i = 0; i < n; i++) {
            blue[i] = types.charAt(i) == 'B';
        }

        System.out.println(minimumEnergy(height, blue, coldCost));
    }
}
```

## Code Deconstruction

### `INF`

```java
private static final long INF = Long.MAX_VALUE / 4;
```

This represents an impossible or very expensive path.
Dividing by `4` reduces overflow risk when adding costs.

### `landingCost` method

```java
private static long landingCost(int from, int to, long[] height, boolean[] blue, long coldCost)
```

This method computes the cost of one jump.
It adds height difference and blue-platform cold cost for the destination.

### `solve` method

```java
private static long solve(int i, int prevTwo, long[] height, boolean[] blue,
                          long coldCost, long[][] memo, boolean[][] seen)
```

This method represents one DP state.

- `i` is the current platform.
- `prevTwo` tells whether the previous jump was length `2`.
- `memo` stores computed minimum costs.
- `seen` marks solved states.

### Final platform base case

```java
if (i == n - 1) {
    return 0L;
}
```

No more energy is needed after reaching the final platform.

### One-step transition

```java
long total = cost + solve(i + 1, 0, height, blue, coldCost, memo, seen);
```

A `1-step` jump is always allowed.
The next state uses `prevTwo = 0`.

### Two-step transition

```java
if (prevTwo == 0 && i + 2 < n) {
```

This condition blocks consecutive `2-step` jumps.
If the previous jump was not `2`, the current `2-step` jump is allowed.

### Saving result

```java
seen[i][prevTwo] = true;
memo[i][prevTwo] = best;
return best;
```

The best answer for the state is saved before returning.

## Language-Specific Notes

- Use `long` for height values, cold cost, and total energy.
- Use `seen` because `0` can be a valid answer.
- `charAt(i)` is used to convert the platform-type string into a boolean array.
- `Math.abs(height[from] - height[to])` works safely here because heights are stored as `long`.
- The recursion depth can reach `n`; this is acceptable for memoization practice, but iterative DP can avoid stack depth.

## Complexity Summary

- Time Complexity: `O(n)`
- Space Complexity: `O(n)`
- Auxiliary Space: `O(n)`

There are at most `2 * n` states.
Each state checks at most two transitions.
