# Toll Staircase

## Concepts Involved

- Base Case Design
- Return Meaning

- Constraint Propagation
- Counting vs Optimization Difference
- Optimization Recursion

## Problem Statement

A courier is climbing a toll staircase with `n` steps, numbered from `0` to `n - 1`.
Each step has a toll cost.

The courier starts before step `0` and wants to move beyond the last step.
From any position, the courier may move either `1` step or `2` steps forward.
Whenever the courier lands on a step, the toll cost of that step is added to the total cost.

The courier has exactly one optional free pass.
The free pass may be used on one landed step to cancel that step's toll.
The pass can also be unused if using it does not help.

Your task is to find the minimum total toll needed to move beyond the last step.

A path is valid only if:

- Every move is either `1` or `2` steps.
- The courier may start by landing on step `0` or step `1`.
- The journey ends when the courier reaches index `n` or greater.
- The free pass can be used at most once.
- The free pass cancels only the toll of a step that is actually landed on.

## Input Format

```text
n
cost0 cost1 cost2 ... cost(n-1)
```

Where:

- `n` is the number of steps.
- `cost[i]` is the toll cost of step `i`.

## Output Format

Print one integer: the minimum possible toll cost.

## Constraints

```text
1 <= n <= 30
0 <= cost[i] <= 10000
```

## Examples

## Edge-Case Thinking Prompts

- What happens if there is only one step?
- Can the courier use the free pass on the first landed step?
- Can the courier finish without landing on the last step?
- Why is this not a counting problem?
- Why must the recursion remember whether the free pass was already used?
