# Mirror Beacon — Quick Solution

## Idea (Simple)
We compute the beacon brightness using recursion.

Each day depends on the previous day and the day before that.

After combining those two smaller answers, we subtract `1` if the current day is a multiple of `4`.

So the recursive function must clearly define what it returns for one day.

## Pattern Recognition Clues
- The current answer is defined using smaller day numbers.
- The formula calls the same logic for `day - 1` and `day - 2`.
- Day `0` and day `1` are fixed starting points.
- A periodic rule applies only after the current day value is computed.

## State Discovery
The first idea is to directly write the formula for day `n`.

But to code it recursively, we need a clear state.

The only changing value is the day number.

So the state is:

- `day`: the day whose brightness we want

No extra previous-choice state is needed because the penalty depends only on the current day number.

## Pattern
Recursive Recurrence Evaluation

## Core Insight
- Base days stop the recursion.
- Every non-base day expands into two smaller recursive calls.
- The multiple-of-4 penalty belongs to the current day, not to the previous days.

## Invariant
For every call `brightness(day)`, the function returns the correct brightness value for that exact day after applying the day’s own penalty if needed.

## Algorithm (with one-line why at important steps)
1. If `day == 0`, return `1`.
   - Why: day `0` is a fixed base case.
2. If `day == 1`, return `2`.
   - Why: day `1` is also directly given.
3. Recursively compute `brightness(day - 1)`.
   - Why: the formula needs the previous day.
4. Recursively compute `brightness(day - 2)`.
   - Why: the formula also needs the day before the previous day.
5. Combine them as `previous + 2 * secondPrevious`.
   - Why: this is the recurrence rule.
6. If `day % 4 == 0`, subtract `1`.
   - Why: mirror energy drops on multiples of `4`.
7. Return the final value.
   - Why: it represents brightness for this day.

## Mental Cheat Sheet
| Item | Meaning |
|---|---|
| Pattern | Recursive recurrence evaluation |
| State | `day` |
| Base cases | `brightness(0) = 1`, `brightness(1) = 2` |
| Transition | `brightness(day - 1) + 2 * brightness(day - 2)` |
| Extra rule | subtract `1` when `day % 4 == 0` |
| Answer | `brightness(n)` |

## Complexity
- Time Complexity: `O(2^n)` because the same smaller days are recomputed.
- Space Complexity: `O(n)` because the recursion stack can go as deep as `n`.
- Auxiliary Space: `O(n)` recursion stack.

## Tiny Dry Run
For `n = 4`:

```text
brightness(2) = 2 + 2 * 1 = 4
brightness(3) = 4 + 2 * 2 = 8
brightness(4) = 8 + 2 * 4 = 16
```

Since `4` is a multiple of `4`, subtract `1`.

Answer = `15`.

## Notes / Pitfalls
- Do not forget the base case for `day == 1`.
- Apply the penalty after computing the current day value.
- Do not apply the penalty to day `0` unless the problem explicitly says so.
- This recursive version repeats work; memoization comes later.
