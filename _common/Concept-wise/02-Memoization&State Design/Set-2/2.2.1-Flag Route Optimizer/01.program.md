# Flag Route Optimizer

## Concepts Involved

Memoization, State Design, Counting DP, Constraint DP, Recursion to DP

## Problem Statement

A volunteer is climbing a staircase of `n` steps to place flags.

From the current step, the volunteer may jump exactly:

- `1` step
- `2` steps
- `3` steps

However, the volunteer gets tired if the same jump size is used twice in a row.
So, two consecutive jumps cannot have the same length.

Many recursive paths reach the same step with the same previous jump size.
Use memoization to avoid recomputing those repeated states.

Return the total number of valid routes to reach exactly step `n`.

A route is defined by the sequence of jumps used.
Two routes are different if at least one jump length differs.

## Input Format

- A single integer `n`

## Output Format

- Print the total number of valid routes to reach exactly step `n`.

## Constraints

- `1 <= n <= 2000`
- The answer may be very large.
- Use exact large number arithmetic.

## Examples

### Example 1

Input:

```text
1
```

Output:

```text
1
```

Explanation:

Only one route is possible:

```text
1
```

### Example 2

Input:

```text
2
```

Output:

```text
1
```

Explanation:

The route `2` is valid.
The route `1 + 1` is invalid because the same jump size is repeated consecutively.

### Example 3

Input:

```text
3
```

Output:

```text
3
```

Explanation:

The valid routes are:

```text
3
1 + 2
2 + 1
```

The route `1 + 1 + 1` is invalid because it repeats the jump size `1` consecutively.

### Example 4

Input:

```text
4
```

Output:

```text
3
```

Explanation:

The valid routes are:

```text
1 + 3
3 + 1
1 + 2 + 1
```

The route `2 + 2` is invalid because the same jump size is repeated consecutively.

## Edge-Case Thinking Prompts

- What should the answer be when `n = 1`?
- Why is `1 + 1` invalid for `n = 2`?
- Why is remembering only the current step not enough?
- What information decides whether the next jump is allowed?
- What should happen if a jump crosses step `n`?
