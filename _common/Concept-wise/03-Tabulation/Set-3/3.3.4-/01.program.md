## Title
Tile Tabulation Layout

## Concepts Involved
Tabulation, Combinatorial DP, Space Optimization

## Problem Statement
A corridor has length `n` units.

A builder wants to fill the corridor completely using two types of tiles:
- small tile of length `1`
- long tile of length `2`

The builder must follow two layout rules:

1. Two long tiles of length `2` cannot be placed next to each other.
2. The final tile placed at the end of the corridor must be a small tile of length `1`.

A layout is valid if:
- the total covered length is exactly `n`,
- no two length-`2` tiles are adjacent,
- the last tile has length `1`.

Count the total number of valid layouts.
Print the exact count.

## Input Format
- A single integer `n`

## Output Format
- Print the total number of valid tile layouts

## Constraints
- `1 <= n <= 5000`

## Examples
### Example 1
Input:
`1`

Output:
`1`

### Example 2
Input:
`4`

Output:
`3`

### Example 3
Input:
`6`

Output:
`6`

## Edge-Case Thinking Prompts
- What happens when `n = 1`?
- Why is a single long tile invalid when `n = 2`?
- Why are adjacent long tiles not allowed?
- Why does the last tile condition affect the final answer?
- Can we count layouts by remembering only the current length?

## Thinking Like a Programmer
- What is the last tile in a valid partial layout?
- Why should we separate layouts ending with `1` and ending with `2`?
- Which previous layouts can accept a new long tile?
- Which state should be returned as the answer?
- How can we reduce the table to a few variables?
