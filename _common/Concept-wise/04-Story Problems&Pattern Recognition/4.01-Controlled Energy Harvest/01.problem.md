# Controlled Energy Harvest

### Concepts Involved

- Dynamic Programming, Take/Skip Pattern, Non-Adjacent Optimization

### Problem Statement

- A researcher is collecting energy from a line of power nodes arranged in sequence. Each node provides a certain amount of energy, but activating two adjacent nodes causes interference and invalidates the collection.
- However, the system introduces a twist:
- The researcher is allowed to override the restriction exactly once, enabling the activation of two adjacent nodes consecutively.
- Find the maximum total energy that can be collected while following the rules.

### Input Format

- Integer n — number of nodes
- Array energy[] of size n — energy value of each node

### Output Format

- Return an integer representing the maximum total energy that can be collected

### Constraints

- $1 \le n \le 10^5$
- $0 \le \text{energy}[i] \le 10^9$

---

## Examples

### Example 1

- **Input:**
>
> ```text
> energy = [2, 7, 9, 3, 1]
> ```
>
- **Output:**
>
> ```text
> 14
> ```
>
- **Explanation:**
- Select nodes: 2 + 9 + 3 = 14 (using override between 9 and 3)

### Example 2

- **Input:**
>
> ```text
> energy = [5, 5, 10, 100, 10, 5]
> ```
>
- **Output:**
>
> ```text
> 120
> ```

## Edge-Case Thinking Prompts

- What if n = 1?
- What if all values are zero?
- What happens if override is never used?
- What if using override gives worse result?

---

## Thinking Like a Programmer

- At each node, do we take or skip?
- What changes if override is already used?
- What state must we remember to make correct decisions?
