## Plain-English Implementation Walkthrough
1. We use `long` because the sum of many rewards may be larger than `int`.

2. We read `n` first because it tells us the array size.

3. We read all reward values into a `long[]`.
   This lets the DP method access any plot by index.

4. The method `maxReward` handles the DP logic.
   This makes the code easier to read and test.

5. If the array is empty, we return `0`.
   There are no plots to choose.

6. For the first plot, we store the better of taking it or taking nothing.
   This is why `dp[0] = Math.max(0L, reward[0])` exists.

7. For two plots, we cannot take both.
   So `dp[1]` becomes the best among first plot, second plot, and empty selection.

8. For every later plot, the code computes two choices.
   `skip` keeps the previous best.
   `take` adds the current reward to the best answer before the previous plot.

9. The update also compares with `0`.
   This keeps the empty selection valid when rewards are negative.

10. We return `dp[n - 1]` because it stores the best valid reward for the whole row.

## Complete Solution Code
```java
import java.io.BufferedReader;
import java.io.InputStreamReader;
import java.util.StringTokenizer;

public class Main {
    private static long maxReward(long[] reward) {
        int n = reward.length;

        if (n == 0) {
            return 0L;
        }

        long[] dp = new long[n];
        dp[0] = Math.max(0L, reward[0]);

        if (n == 1) {
            return dp[0];
        }

        dp[1] = Math.max(dp[0], Math.max(0L, reward[1]));

        for (int i = 2; i < n; i++) {
            long skip = dp[i - 1];
            long take = dp[i - 2] + reward[i];
            dp[i] = Math.max(skip, Math.max(take, 0L));
        }

        return dp[n - 1];
    }

    public static void main(String[] args) throws Exception {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));

        int n = Integer.parseInt(br.readLine().trim());
        long[] reward = new long[n];

        StringTokenizer st = new StringTokenizer(br.readLine());
        for (int i = 0; i < n; i++) {
            reward[i] = Long.parseLong(st.nextToken());
        }

        System.out.println(maxReward(reward));
    }
}
```

## Code Deconstruction
- `maxReward(long[] reward)`
  - Computes the maximum valid non-adjacent reward.

- `dp[i]`
  - Stores the best valid answer from plot `0` to plot `i`.

- `dp[0] = Math.max(0L, reward[0])`
  - Handles the first plot and keeps the option of selecting nothing.

- `dp[1] = Math.max(dp[0], Math.max(0L, reward[1]))`
  - Handles the first two adjacent plots safely.

- `skip = dp[i - 1]`
  - Means the current plot is ignored.

- `take = dp[i - 2] + reward[i]`
  - Means the current plot is selected, so the previous plot cannot be selected.

- `dp[i] = Math.max(skip, Math.max(take, 0L))`
  - Chooses the best valid action and protects all-negative input.

- `return dp[n - 1]`
  - Returns the answer for all plots.

## Language-Specific Notes
- `long` is safer than `int` for large reward sums.
- `StringTokenizer` expects the rewards to be space-separated.
- This version uses a DP array for teaching clarity.
- The space can be optimized to `O(1)` using two rolling variables.

## Complexity Summary
- Time Complexity: `O(n)`
- Space Complexity: `O(n)`
- Auxiliary Space: `O(n)`
