## Plain-English Implementation Walkthrough
1. We store the heights in an array because each jump needs the current and next stone height.
   Related code: `heights[i]`

2. We create a recursive method that knows the current stone, previous jump, and 3-step usage.
   Related code: `minEnergy(index, lastJump, usedThree, heights, penalty)`

3. If the current index is already the last stone, we return `0`.
   No more movement is needed from that state.

4. We keep `best` as a very large value.
   This helps us safely minimize over all possible jumps.

5. We try jump sizes from `1` to `3`.
   This keeps the choice expansion clean and compact.

6. If the next index crosses the array, we skip that jump.
   The robot cannot land beyond the final stone.

7. If the jump is `3` and `usedThree` is already true, we skip it.
   The special jump is allowed only once.

8. We compute the jump cost using height difference.
   If the jump is `3`, we double that cost.

9. If the current jump and previous jump are both `2`, we add the fatigue penalty.
   This correctly propagates the consecutive jump constraint.

10. We recursively solve the remaining route and keep the minimum total energy.
   We return `best` because it stores the cheapest valid route from this state.

## Complete Solution Code
```java
import java.util.Scanner;

public class Main {
    static final long INF = Long.MAX_VALUE / 4;

    static long minEnergy(int index, int lastJump, boolean usedThree, int[] heights, int penalty) {
        int n = heights.length;

        if (index == n - 1) {
            return 0;
        }

        long best = INF;

        for (int jump = 1; jump <= 3; jump++) {
            int next = index + jump;

            if (next >= n) {
                continue;
            }

            if (jump == 3 && usedThree) {
                continue;
            }

            long cost = Math.abs(heights[next] - heights[index]);

            if (jump == 3) {
                cost *= 2;
            }

            if (jump == 2 && lastJump == 2) {
                cost += penalty;
            }

            boolean nextUsedThree = usedThree || jump == 3;
            long remaining = minEnergy(next, jump, nextUsedThree, heights, penalty);

            if (remaining != INF) {
                best = Math.min(best, cost + remaining);
            }
        }

        return best;
    }

    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);

        int n = sc.nextInt();
        int[] heights = new int[n];

        for (int i = 0; i < n; i++) {
            heights[i] = sc.nextInt();
        }

        int penalty = sc.nextInt();

        System.out.println(minEnergy(0, 0, false, heights, penalty));
        sc.close();
    }
}
```

## Code Deconstruction
- `INF` is a safe large value.
  - It represents an answer that has not been found yet.

- `index` tells where the robot currently stands.
  - This is the main shrinking state.

- `lastJump` tells whether the previous jump was `2`.
  - This is needed for the fatigue rule.

- `usedThree` tells whether the special `3-step` jump has already been consumed.
  - This blocks using it more than once.

- The `for` loop expands all jump choices.
  - It tries `1`, `2`, and `3` in a single structure.

- `next >= n` skips invalid jumps.
  - The robot must land on a real stone.

- `jump == 3 && usedThree` enforces the one-time special jump rule.

- `cost += penalty` is applied only for consecutive `2-step` jumps.

- `best = Math.min(best, cost + remaining)` keeps the cheapest valid route.

## Language-Specific Notes
- `Math.abs(int)` returns an `int`, but it is stored in a `long` for safety.
- `Long.MAX_VALUE / 4` is used to avoid overflow when adding costs.
- This is a plain recursive solution, so it repeats work.
- A memoized version can store answers for `(index, lastJump, usedThree)`.

## Complexity Summary
- Time Complexity: `O(3^n)` worst case.
- Space Complexity: `O(n)` recursion stack.
- Auxiliary Space: `O(n)` recursion stack.
