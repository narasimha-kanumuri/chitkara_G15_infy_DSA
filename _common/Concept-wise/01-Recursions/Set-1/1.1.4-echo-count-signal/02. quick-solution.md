# Quick Solution — Echo Count Signal

## Idea (Simple)

We count all ways to grow from level `0` to level `n` using `+1` and `+2`.
When we reach exactly `n`, we check the ending of the sequence.
If the last two increases are both `+1`, that sequence is invalid.
So recursion must remember how many `+1` moves are currently at the end of the path.

## Pattern Recognition Clues

- The problem asks for the number of valid constructions.
- Each level gives repeated choices: add `1` or add `2`.
- Validity depends on the ending of the path.
- This suggests counting recursion with an ending-state variable.

## State Discovery

The simple state is `solve(level)`: number of ways to reach `n` from `level`.
But `solve(level)` is not enough.
It only knows the current level.
It does not know whether the path currently ends with one or two `+1` moves.
That ending matters when `level == n`.
So the final state is:

- `solve(level, endingOnes)`

Here, `endingOnes` stores how many consecutive `+1` moves are at the end of the current path.
We only need values `0`, `1`, or `2`.

## Pattern

Counting Recursion with End-State Validation

## Core Insight

- A `+1` move increases the ending `+1` count.
- A `+2` move resets the ending `+1` count to `0`.
- At the target level, return `0` only if the sequence ends with two `+1` moves.

## Invariant

For every call `solve(level, endingOnes)`, the function returns the number of valid ways to complete the signal from `level` to `n`, assuming `endingOnes` correctly describes the current ending of the path.

## Algorithm (with one-line why at important steps)

1. Start with `solve(0, 0)`.
   - Why: the signal begins at level `0` with no moves taken.
2. If `level > n`, return `0`.
   - Why: crossing the target is invalid.
3. If `level == n`, check `endingOnes`.
   - Why: the final rule depends on the last two moves.
4. If `endingOnes >= 2`, return `0`; otherwise return `1`.
   - Why: this decides whether one complete sequence is valid.
5. Try a `+1` move using `solve(level + 1, min(2, endingOnes + 1))`.
   - Why: the path now ends with one more `+1`.
6. Try a `+2` move using `solve(level + 2, 0)`.
   - Why: a `+2` breaks the ending chain of `+1` moves.
7. Return the sum of both choices.
   - Why: each valid choice creates separate construction sequences.

## Mental Cheat Sheet

| Item | Meaning |
|---|---|
| Pattern | Counting recursion with final-state check |
| State | Current level + trailing `+1` count |
| `+1` transition | Increase trailing `+1` count |
| `+2` transition | Reset trailing `+1` count to `0` |
| Answer | `solve(0, 0)` |

## Complexity

- Time Complexity: `O(2^n)` in the worst case without memoization
- Space Complexity: `O(n)` because of recursion depth
- Auxiliary Space: `O(n)` for the recursion call stack

## Tiny Dry Run

For `n = 3`:

- `1 + 1 + 1` ends with two `+1` moves, so it is invalid.
- `1 + 2` is valid.
- `2 + 1` is valid.

Answer = `2`.

## Notes / Pitfalls

- Do not reject every path that contains two `+1` moves earlier.
- Only the final two moves matter.
- Do not count paths that cross beyond `n`.
- Use addition between choices because this is counting recursion.
