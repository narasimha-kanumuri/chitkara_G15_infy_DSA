# Silent Binary March

## Concepts Involved

- Base Case Design
- Return Meaning
- Choice Expansion
- Recursive State Definition
- Recursion Tree Analysis
- Constraint Propagation
- Counting Recursion
- Optimization vs Counting Distinction

## Problem Statement

A guard must create a silent binary march code of length `n`.  
Each position in the code must contain either `0` or `1`.

The code must follow these rules:

1. The code must not contain two consecutive `1`s.
2. The final position must be `0`.
3. Some positions are restricted. A restricted position cannot contain `1`.

Return the total number of valid binary codes that can be formed.

The code is built from left to right using recursion.  
At each position, the guard chooses whether to place `0` or `1`, if the choice is allowed.

## Input Format

- The first line contains two integers `n` and `k`.
  - `n` is the length of the binary code.
  - `k` is the number of restricted positions.
- The second line contains `k` distinct integers: the restricted positions.
  - If `k = 0`, this line may be empty or absent.

Positions are numbered from `1` to `n`.

## Output Format

Print one integer: the number of valid binary codes.

## Constraints

- `1 <= n <= 40`
- `0 <= k <= n`
- `1 <= restrictedPosition <= n`
- The intended solution for this file is pure recursion, so the input size is small enough for recursive teaching.

## Examples

### Example 1

**Input**

```text
4 1
2
```

**Output**

```text
4
```

**Explanation**

Position `2` cannot contain `1`, and the code must end with `0`.

Valid codes are:

```text
0000
0010
1000
1010
```

So the answer is `4`.

### Example 2

**Input**

```text
5 2
3 4
```

**Output**

```text
3
```

**Explanation**

Positions `3` and `4` cannot contain `1`, and the last position must be `0`.

The first two positions can form only these valid prefixes:

```text
00
01
10
```

So the total number of valid codes is `3`.

### Example 3

**Input**

```text
1 0
```

**Output**

```text
1
```

**Explanation**

The only valid code is:

```text
0
```

The code `1` is not valid because the final position must be `0`.

## Edge-Case Thinking Prompts

- What should the recursion return after all positions are filled?
- Why is the final position different from other positions?
- Why does the function need to remember the previous bit?
- Can a restricted position still contain `0`?
- Is this problem asking for minimum or maximum value, or is it asking for a count?

## Thinking Like a Programmer

Before writing code, define:

```text
State:
Choice:
Transition:
Base case:
Order:
Answer:
```

For this problem:

```text
State: current position and previous bit
Choice: place 0, or place 1 if allowed
Transition: move to the next position with the chosen bit as previous bit
Base case: after position n, count 1 valid code only if the last bit is 0
Order: build the code from left to right
Answer: solve(1, 0)
```
