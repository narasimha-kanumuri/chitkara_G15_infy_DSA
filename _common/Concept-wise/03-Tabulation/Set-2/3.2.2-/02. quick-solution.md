## Idea (Simple)
We need the beacon brightness on day `n`.
Each day depends on the previous one or two days.
So we can build the answer from day `1` up to day `n`.
If the day number is a multiple of `4`, subtract `1` after computing that day.

## Pattern Recognition Clues
- The current value depends on smaller day numbers.
- Recursive computation would repeat the same previous days many times.
- The recurrence moves forward from small days to large days.
- This suggests bottom-up DP with one value per day.

## State Discovery
The first useful state is direct:
`dp[i]` = brightness value on day `i`.
No extra state is needed.
The only changing information is the day number.
The multiple-of-4 penalty depends only on `i`, not on any hidden past choice.
So `dp[i]` keeps all required information.

## Pattern
Bottom-Up 1D Recurrence DP

## Core Insight
- Store brightness for each day in a table.
- Use the previous two days to compute the current day.
- Apply the maintenance penalty only when `i % 4 == 0`.

## Invariant
After computing day `i`, `dp[i]` stores the correct brightness value for that day after applying the multiple-of-4 penalty if needed.

## Algorithm (with one-line why at important steps)
1. Create a DP array `dp` of size `n + 1`.
   - Why: we need to store brightness for each day.
2. Set `dp[1] = 1`.
   - Why: day `1` is a fixed base case.
3. If `n >= 2`, set `dp[2] = 2`.
   - Why: day `2` is also fixed.
4. For each day `i` from `3` to `n`, compute `dp[i] = dp[i - 1] + dp[i - 2]`.
   - Why: the current day depends on the previous two days.
5. If `i` is divisible by `4`, subtract `1` from `dp[i]`.
   - Why: maintenance drain applies on multiples of `4`.
6. Print `dp[n]`.
   - Why: it stores the final brightness value.

## Mental Cheat Sheet
| Item | Meaning |
|---|---|
| Pattern | Bottom-up recurrence DP |
| State | `dp[i] = brightness on day i` |
| Base cases | `dp[1] = 1`, `dp[2] = 2` |
| Transition | `dp[i] = dp[i - 1] + dp[i - 2]` |
| Penalty | subtract `1` when `i % 4 == 0` |
| Answer | `dp[n]` |

## Complexity
- Time Complexity: `O(n)`
- Space Complexity: `O(n)`
- Auxiliary Space: `O(n)`

## Tiny Dry Run
For `n = 5`:
- `dp[1] = 1`
- `dp[2] = 2`
- `dp[3] = 1 + 2 = 3`
- `dp[4] = 3 + 2 - 1 = 4`
- `dp[5] = 4 + 3 = 7`
- Answer = `7`

## Notes / Pitfalls
- Do not subtract on every day.
- Apply the penalty after adding the previous two values.
- Handle `n = 1` before accessing day `2`.
- Use `BigInteger` because exact values can grow large.
