## quick-solution.md

### Idea (Simple)

We must count jump sequences that reach step `n`.

The extra rule is the important part: a `2`-step jump cannot come right after another `2`-step jump.

So we should remember not only the step, but also how we arrived there.

### Pattern

Bottom-Up DP with last-move state

### Core Insight

- Split ways by the last jump used.

- If the last jump is `1`, the previous path can be anything valid.

- If the last jump is `2`, the previous jump must not be `2`.

### Invariant

At every step `i`:

- `endWithOne[i]` = number of valid ways to reach `i` with last jump `1`

- `endWithTwo[i]` = number of valid ways to reach `i` with last jump `2`

These two values together contain all valid histories needed for future transitions.

### Algorithm (with one-line why at important steps)

1. Create two DP arrays: `endWithOne` and `endWithTwo`.

- Why: the constraint depends on the last jump.

1. Set `endWithOne[1] = 1`.

- Why: step `1` can only be reached by one jump of size `1`.

1. Set `endWithTwo[2] = 1` when `n >= 2`.

- Why: step `2` can be reached directly by one jump of size `2`.

1. For each step `i` from `2` to `n`, compute `endWithOne[i] = endWithOne[i - 1] + endWithTwo[i - 1]`.

- Why: a final `1` can follow any valid path to `i - 1`.

1. For each step `i` from `3` to `n`, compute `endWithTwo[i] = endWithOne[i - 2]`.

- Why: a final `2` can only follow a path whose previous last jump was `1`.

1. Return `endWithOne[n] + endWithTwo[n]`.

- Why: every valid answer must end with either `1` or `2`.

### Code Cheat Sheet

| Code / Built-in | Meaning / Role |

|---|---|

| `endWithOne[i]` | valid ways to reach step `i` with final jump `1` |

| `endWithTwo[i]` | valid ways to reach step `i` with final jump `2` |

| `BigInteger` | stores exact large counts safely |

| `answer` | total valid ways for step `n` |

### Complexity

- Time: `O(n)`

- Space: `O(n)`

- Auxiliary Space: `O(n)`

### Tiny Dry Run

For `n = 4`:

- `i = 1`: `endWithOne[1] = 1`

- `i = 2`: `endWithOne[2] = 1`, `endWithTwo[2] = 1`

- `i = 3`: `endWithOne[3] = 2`, `endWithTwo[3] = 1`

- `i = 4`: `endWithOne[4] = 3`, `endWithTwo[4] = 1`

- Total = `4`

### Notes / Pitfalls

- Do not use one single DP array without tracking the last jump.

- The direct jump `2` to reach step `2` is valid.

- Exact counts grow fast, so fixed-size integer types may overflow.

- The answer is the sum of both ending states, not just one of them.
