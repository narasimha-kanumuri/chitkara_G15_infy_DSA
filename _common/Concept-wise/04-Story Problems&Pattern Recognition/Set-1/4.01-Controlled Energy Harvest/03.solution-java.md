## Plain-English Implementation Walkthrough
1. We need an impossible value because some DP states are not real at the start. So we create `NEG`.
2. We read `n` because the number of energy values controls the loop.
3. We read the energy values into a `long[]`. We use `long` because the total can be larger than `int`.
4. We create a `dp` table with two dimensions. One dimension tells whether the override was used. The other tells whether the previous node was taken.
5. We set `dp[0][0] = 0`. This means no node has been processed, no override is used, and there is no previous taken node.
6. For each energy value, we create a new table called `next`. This avoids mixing old-prefix states with new-prefix states.
7. Skipping the current node is always allowed. So we keep the same override status and set `prevTaken` to `0`.
8. Taking the current node after a skipped previous node is always allowed. So if `prevTaken == 0`, we add the current value and set `prevTaken` to `1`.
9. Taking the current node after a taken previous node creates an adjacent pair. This is allowed only when `used == 0`. After this, the override becomes used.
10. After processing all nodes, any final state may contain the answer. So we return the maximum among all four final states.

## Complete Solution Code
```java
import java.io.BufferedReader;
import java.io.InputStreamReader;
import java.util.StringTokenizer;

public class Main {
    private static final long NEG = Long.MIN_VALUE / 4;

    private static long maxEnergy(long[] energy) {
        long[][] dp = new long[2][2];

        for (int used = 0; used < 2; used++) {
            for (int prevTaken = 0; prevTaken < 2; prevTaken++) {
                dp[used][prevTaken] = NEG;
            }
        }

        dp[0][0] = 0L;

        for (long value : energy) {
            long[][] next = new long[2][2];

            for (int used = 0; used < 2; used++) {
                for (int prevTaken = 0; prevTaken < 2; prevTaken++) {
                    next[used][prevTaken] = NEG;
                }
            }

            for (int used = 0; used < 2; used++) {
                for (int prevTaken = 0; prevTaken < 2; prevTaken++) {
                    if (dp[used][prevTaken] == NEG) {
                        continue;
                    }

                    next[used][0] = Math.max(next[used][0], dp[used][prevTaken]);

                    if (prevTaken == 0) {
                        next[used][1] = Math.max(next[used][1], dp[used][prevTaken] + value);
                    }

                    if (prevTaken == 1 && used == 0) {
                        next[1][1] = Math.max(next[1][1], dp[used][prevTaken] + value);
                    }
                }
            }

            dp = next;
        }

        long answer = 0L;
        for (int used = 0; used < 2; used++) {
            for (int prevTaken = 0; prevTaken < 2; prevTaken++) {
                answer = Math.max(answer, dp[used][prevTaken]);
            }
        }

        return answer;
    }

    public static void main(String[] args) throws Exception {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));

        int n = Integer.parseInt(br.readLine().trim());
        long[] energy = new long[n];

        StringTokenizer st = new StringTokenizer(br.readLine());
        for (int i = 0; i < n; i++) {
            energy[i] = Long.parseLong(st.nextToken());
        }

        System.out.println(maxEnergy(energy));
    }
}
```

## Code Deconstruction
- `NEG` is used for impossible states. We do not use `Long.MIN_VALUE` directly because adding to it can overflow.
- `dp[used][prevTaken]` stores the best energy for the processed prefix.
- `used` tells whether the override is already spent.
- `prevTaken` tells whether the previous node was collected.
- The skip transition keeps the same total and sets `prevTaken` to `0`.
- The normal take transition is allowed when `prevTaken == 0`.
- The override take transition is allowed when `prevTaken == 1 && used == 0`.
- The final answer checks all four states because the best result does not need to end in a specific state.

## Language-Specific Notes
- `long` is used because energy sums can be large.
- The DP table is only `2 x 2`, so memory stays constant.
- The code treats the override as optional.
- The code allows skipping nodes naturally because the initial answer is `0`.

## Complexity Summary
- Time Complexity: `O(n)`
- Space Complexity: `O(1)`
- Auxiliary Space: `O(1)`
