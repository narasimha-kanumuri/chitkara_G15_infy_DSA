## Plain-English Implementation Walkthrough
1. We need exact brightness values.  
   So the code uses `BigInteger` instead of primitive integer types.

2. The answer for each day depends only on earlier days.  
   So the code stores all day values in a `dp` array.

3. Java object arrays contain `null` by default.  
   So the code fills every `dp` cell with `BigInteger.ZERO` before using it.

4. Day `1` and day `2` are fixed starting values.  
   So the code assigns `dp[1] = 1` and, when possible, `dp[2] = 2`.

5. If `n` is `1`, day `2` does not exist in the needed computation.  
   So the method returns `dp[1]` immediately.

6. For each later day, the code adds the previous two day values.  
   This directly matches the recurrence rule.

7. The maintenance drain depends only on the day number.  
   So the code checks `day % 4 == 0` inside the loop.

8. If the day is a multiple of `4`, the code subtracts `1` from that day’s computed value.  
   This keeps the penalty local to that day.

9. After the loop finishes, `dp[n]` stores the final brightness.  
   So the method returns `dp[n]`.

10. The `main` method only reads input and prints the result.  
    This keeps the DP logic inside a separate method.

## Complete Solution Code
```java
import java.io.BufferedReader;
import java.io.InputStreamReader;
import java.math.BigInteger;

public class Main {
    private static BigInteger beaconBrightness(int n) {
        BigInteger[] dp = new BigInteger[n + 1];

        for (int i = 0; i <= n; i++) {
            dp[i] = BigInteger.ZERO;
        }

        dp[1] = BigInteger.ONE;

        if (n == 1) {
            return dp[1];
        }

        dp[2] = BigInteger.valueOf(2);

        for (int day = 3; day <= n; day++) {
            dp[day] = dp[day - 1].add(dp[day - 2]);

            if (day % 4 == 0) {
                dp[day] = dp[day].subtract(BigInteger.ONE);
            }
        }

        return dp[n];
    }

    public static void main(String[] args) throws Exception {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        int n = Integer.parseInt(br.readLine().trim());
        System.out.println(beaconBrightness(n));
    }
}
```

## Code Deconstruction
- `beaconBrightness(int n)`
  - Computes the brightness value for day `n`.
- `BigInteger[] dp`
  - Stores brightness values for all days from `1` to `n`.
- `dp[1] = BigInteger.ONE`
  - Stores the fixed brightness for day `1`.
- `if (n == 1)`
  - Prevents unnecessary access to day `2` logic when only day `1` is needed.
- `dp[2] = BigInteger.valueOf(2)`
  - Stores the fixed brightness for day `2`.
- `dp[day - 1].add(dp[day - 2])`
  - Builds the current day from the previous two days.
- `day % 4 == 0`
  - Checks whether the maintenance drain applies.
- `.subtract(BigInteger.ONE)`
  - Applies the penalty for multiples of `4`.
- `return dp[n]`
  - Returns the final value requested by the problem.

## Language-Specific Notes
- `BigInteger` is used because the exact brightness value may become larger than `long`.
- Use `.add(...)` and `.subtract(...)`; do not use `+` or `-` with `BigInteger` objects.
- Initialize every `BigInteger[]` cell before using it.
- The solution is iterative, so it avoids recursion stack issues.
- The DP array uses day numbers directly, so it is easy to match code with the recurrence.
