# Frog Memo Jump — Quick Solution

## Idea (Simple)

The frog wants the minimum energy to reach the last stone.
From each stone, it may jump `1` or `2` positions.
The energy depends on height difference.
If two `2-step` jumps happen consecutively, the second one adds penalty `p`.

So we use recursion with memoization and remember whether the previous jump was `2`.

## Pattern Recognition Clues

- The task asks for minimum total energy.
- Future cost depends on the current stone.
- A penalty depends on the previous jump.
- The same `(stone, previous jump type)` state can repeat.

## State Discovery

The first simple state is `dp[i]`: minimum energy from stone `i`.

But `dp[i]` is not enough.
It does not remember whether the previous jump was a `2-step` jump.

That missing information is important because the next `2-step` jump may add penalty `p`.

So we use:

```text
solve(i, prevTwo)
```

Meaning:

```text
minimum energy needed to reach the last stone from stone i,
where prevTwo tells whether the previous jump was 2-step
```

## Pattern

Top-Down DP with Previous Choice State

## Core Insight

- A `1-step` jump never creates a consecutive `2-step` penalty.
- A `2-step` jump adds penalty only when `prevTwo = 1`.
- Cache both state dimensions: stone index and previous jump type.

## Invariant

For every state `solve(i, prevTwo)`:

- The frog is currently on stone `i`.
- `prevTwo` correctly tells whether the jump used to reach `i` was length `2`.
- The function returns the minimum valid energy from `i` to the last stone.

## Algorithm (with one-line why at important steps)

1. Start with `solve(0, 0)`.
   - Why: the frog starts on stone `0` and no previous jump exists.
2. If `i == n - 1`, return `0`.
   - Why: the frog already reached the target.
3. If the state is cached, return it.
   - Why: repeated subproblems should not be recomputed.
4. Try a `1-step` jump if `i + 1 < n`.
   - Why: this is one allowed move and resets `prevTwo` to `0`.
5. Try a `2-step` jump if `i + 2 < n`.
   - Why: this is the second allowed move.
6. Add penalty `p` to the `2-step` option only if `prevTwo = 1`.
   - Why: penalty applies only on consecutive `2-step` jumps.
7. Store and return the smaller option.
   - Why: we need minimum energy.

## Mental Cheat Sheet

| Item | Meaning |
|---|---|
| Pattern | Memoized recursion with previous choice state |
| State | `(i, prevTwo)` |
| `i` | current stone index |
| `prevTwo` | whether previous jump was length `2` |
| Transition | try jump `1` and jump `2` |
| Answer | `solve(0, 0)` |

## Complexity

- Time Complexity: `O(n)`
- Space Complexity: `O(n)`
- Auxiliary Space: `O(n)` for memo table and recursion stack

There are at most `2 * n` useful states.
Each state checks at most two jumps.

## Tiny Dry Run

For:

```text
height = [10, 30, 40, 20], p = 10
```

Start at `solve(0, 0)`.

Possible early choices:

```text
0 -> 1 cost = 20, state solve(1, 0)
0 -> 2 cost = 30, state solve(2, 1)
```

From stone `1`, jump to stone `3` costs `10`.
Total = `20 + 10 = 30`.

So the answer is `30`.

## Notes / Pitfalls

- Do not cache only by index `i`.
- Add penalty only when the current jump is `2` and the previous jump was also `2`.
- A `1-step` jump resets `prevTwo` to `0`.
- Use `long` because height values and penalties can be large.
- For `n = 1`, the answer is `0`.
