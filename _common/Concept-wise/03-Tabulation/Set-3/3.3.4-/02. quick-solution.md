## Idea (Simple)
We count ways to tile a corridor of length `n` using tiles of length `1` and `2`.
Long tiles cannot be adjacent.
The final tile must be length `1`.
So we track layouts by the last tile used.

## Pattern Recognition Clues
- We are counting all valid arrangements.
- Each longer layout is built from smaller lengths.
- The next tile depends on the previous tile length.
- This suggests bottom-up DP with last-choice state.

## State Discovery
The first idea is `dp[i]`: number of ways to tile length `i`.
But `dp[i]` is not enough.
It does not tell whether the last tile was length `1` or length `2`.
That matters because a new length-`2` tile cannot follow another length-`2` tile.
So we split the state:
- `endWithOne[i]`: layouts of length `i` ending with tile `1`
- `endWithTwo[i]`: layouts of length `i` ending with tile `2`

## Pattern
Bottom-Up Combinatorial DP with Last Tile State

## Core Insight
- A tile `1` can be placed after any valid layout.
- A tile `2` can be placed only after a layout ending with tile `1`.
- Since the final tile must be `1`, the answer is `endWithOne[n]`.

## Invariant
For every length `i`:
- `endWithOne[i]` stores all valid layouts of length `i` ending with tile `1`.
- `endWithTwo[i]` stores all valid layouts of length `i` ending with tile `2`.
No counted layout has two adjacent long tiles.

## Algorithm (with one-line why at important steps)
1. Keep states for layouts ending with tile `1` and tile `2`.
   - Why: adjacency depends on the previous tile.
2. Treat the empty layout as one safe starting prefix.
   - Why: it allows the first long tile to be placed when needed.
3. For each length, compute ways ending with tile `1` from both previous endings.
   - Why: tile `1` can follow either tile type.
4. Compute ways ending with tile `2` only from layouts ending with tile `1` two units earlier.
   - Why: tile `2` cannot follow tile `2`.
5. Return the state ending with tile `1` at length `n`.
   - Why: the last tile must be small.

## Mental Cheat Sheet
| Item | Meaning |
|---|---|
| Pattern | Counting DP with last tile state |
| Simple failed state | `dp[i]` forgets the last tile |
| Tile `1` transition | can follow both ending states |
| Tile `2` transition | can follow only a previous tile `1` state |
| Final condition | answer must end with tile `1` |
| Answer | `endWithOne[n]` |

## Complexity
- Time Complexity: `O(n)`
- Space Complexity: `O(1)`
- Auxiliary Space: `O(1)`

## Tiny Dry Run
For `n = 4`:
- Length `1`: valid ending with `1` = `1`
- Length `2`: valid final layouts ending with `1` = `1`
- Length `3`: valid final layouts ending with `1` = `2`
- Length `4`: valid final layouts ending with `1` = `3`
- Answer = `3`

## Notes / Pitfalls
- Do not return layouts ending with tile `2`.
- Do not allow two length-`2` tiles to touch.
- A layout of length `2` using only one long tile is not a valid final answer because it ends with `2`.
- Use `BigInteger` because exact counts can become large.
