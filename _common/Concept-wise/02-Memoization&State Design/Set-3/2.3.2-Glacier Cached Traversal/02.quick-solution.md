# Glacier Cached Traversal — Quick Solution

## Idea (Simple)

We need the minimum energy to reach the last icy platform.
The climber can jump `1` or `2` platforms.
Landing on a blue platform adds cold cost.
Two `2-step` jumps cannot happen consecutively.

So we use memoized recursion and remember whether the previous jump was `2`.

## Pattern Recognition Clues

- The problem asks for minimum total cost.
- The same platform can be reached by different routes.
- Future choices depend on the previous jump length.
- Repeated recursive calls suggest memoization.

## State Discovery

The first simple state is `solve(i)`.

But `solve(i)` is not enough.
It only tells where the climber is standing.
It does not tell whether the previous jump was a `2-step` jump.

That missing information decides whether the next `2-step` jump is allowed.

So we use:

```text
solve(i, prevTwo)
```

Meaning:

```text
minimum energy to reach the final platform from platform i,
where prevTwo tells whether the previous jump was length 2
```

## Pattern

Top-Down DP with Previous Jump State

## Core Insight

- A `1-step` jump is always allowed.
- A `2-step` jump is allowed only when `prevTwo = 0`.
- Landing cost is added for the destination platform, not the current platform.
- Cache every `(i, prevTwo)` state.

## Invariant

For every state `solve(i, prevTwo)`:

- The climber is currently on platform `i`.
- `prevTwo` correctly stores whether the jump used to reach `i` was length `2`.
- The function returns the minimum valid remaining energy from that state.

## Algorithm (with one-line why at important steps)

1. Start with `solve(0, 0)`.
   - Why: the climber starts on platform `0`, and no previous jump exists.
2. If `i == n - 1`, return `0`.
   - Why: the final platform has already been reached.
3. If the state is cached, return it.
   - Why: repeated states should be solved once.
4. Try jump `1` if `i + 1 < n`.
   - Why: a `1-step` jump is always valid and resets `prevTwo` to `0`.
5. Try jump `2` only if `prevTwo == 0` and `i + 2 < n`.
   - Why: this blocks consecutive `2-step` jumps.
6. Add height-difference cost and blue-platform landing cost.
   - Why: both are part of the jump cost.
7. Store and return the minimum value.
   - Why: the task asks for minimum energy.

## Mental Cheat Sheet

| Item | Meaning |
|---|---|
| Pattern | Memoized min-cost jump DP |
| State | `(i, prevTwo)` |
| `i` | current platform |
| `prevTwo` | previous jump was length `2` or not |
| Blue cost | added when landing on a blue platform |
| Answer | `solve(0, 0)` |

## Complexity

- Time Complexity: `O(n)`
- Space Complexity: `O(n)`
- Auxiliary Space: `O(n)` for memo table and recursion stack

There are at most `2 * n` states.
Each state checks at most two jumps.

## Tiny Dry Run

For:

```text
height = [3, 8, 2, 6], c = 5, blue = NBNN
```

Start at `solve(0, 0)`.

Possible moves:

```text
0 -> 1 cost = 5 + 5 blue cost = 10
0 -> 2 cost = 1
```

From platform `2`, only jump `1` is needed to reach platform `3`:

```text
2 -> 3 cost = 4
```

Total = `1 + 4 = 5`.

## Notes / Pitfalls

- Do not cache only by index.
- Add cold cost for the platform being landed on.
- Do not add cold cost for the starting platform.
- Do not allow a `2-step` jump after another `2-step` jump.
- Use `long` because heights and costs can be large.
