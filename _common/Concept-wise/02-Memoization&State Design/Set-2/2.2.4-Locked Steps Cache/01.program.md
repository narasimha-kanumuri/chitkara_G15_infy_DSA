# Locked Steps Cache

## Concepts Involved

Memoization, State Design, Constraint Handling, Counting DP, Prime Rule

## Problem Statement

A student wants to climb from step `0` to step `n`.

From any step, the student may normally move either:

- `1` step forward
- `2` steps forward

Some steps are locked and cannot be landed on.
Step `0` is the starting step and is never locked.

There is one extra movement rule:

- If the student is currently standing on a prime-numbered step, the next move must be exactly `1` step.

Many recursive paths may again ask the same question:

```text
How many valid ways are there from this step to n?
```

Use memoization to avoid recomputing the same step again and again.

Return the total number of valid ways to reach exactly step `n`.

A valid way is defined by the sequence of moves used.
Two ways are different if at least one move length differs.

## Input Format

- The first line contains two integers `n` and `m`.
- The second line contains `m` distinct integers, representing locked steps.
- If `m = 0`, the second line may be empty or absent.

## Output Format

- Print the total number of valid ways to reach exactly step `n`.

## Constraints

- `1 <= n <= 2000`
- `0 <= m <= n`
- Locked steps are in the range `1` to `n`
- Step `0` is never locked
- The answer may be very large
- Use exact large number arithmetic

## Examples

### Example 1

Input:

```text
4 0
```

Output:

```text
3
```

Explanation:

The valid ways are:

```text
0 -> 1 -> 2 -> 3 -> 4
0 -> 1 -> 3 -> 4
0 -> 2 -> 3 -> 4
```

Step `2` is prime, so from step `2` the student must move exactly `1` step.
That is why `0 -> 2 -> 4` is not valid.

### Example 2

Input:

```text
5 1
3
```

Output:

```text
0
```

Explanation:

Step `3` is locked.
If the student reaches prime step `2`, the next move must be to step `3`, but it is locked.
Other possible routes also cannot reach the top without landing on the locked step.

### Example 3

Input:

```text
5 0
```

Output:

```text
3
```

Explanation:

The valid ways are:

```text
0 -> 1 -> 2 -> 3 -> 4 -> 5
0 -> 1 -> 3 -> 4 -> 5
0 -> 2 -> 3 -> 4 -> 5
```

Prime steps force the next move to be exactly `1` step.

## Edge-Case Thinking Prompts

- What should happen if step `n` itself is locked?
- Why is step `1` not treated as prime?
- Why is the current step enough as a memoization state here?
- When should a path return `0`?
- Why should a move beyond `n` not be counted?
