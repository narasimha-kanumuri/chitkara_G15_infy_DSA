## Idea (Simple)
We have garden plots in a row.
Each plot has a reward value.
We may choose a plot or skip it.

The rule is: two adjacent plots cannot both be chosen.
Some rewards may be negative, so choosing no plot is allowed.
This means the answer should never be forced below `0`.

## Pattern Recognition Clues
- The problem asks for the maximum total reward, so this is optimization DP.
- Every plot gives a choice: take it or skip it.
- Taking the current plot blocks the previous plot.
- Negative rewards mean we must keep the option of taking nothing.

## State Discovery
The first idea is to greedily take every positive reward.
That is not enough because adjacent positive plots may conflict.
For example, choosing a small positive plot can block a larger reward next to it.

The next simple idea is `dp[i]`.
This works here because the only needed history is the best answer before the previous plot.
So we define:

- `dp[i]`: maximum valid reward using plots from index `0` to `i`

This state is enough because each plot only conflicts with its immediate neighbor.

## Pattern
Take / Skip Dynamic Programming

## Core Insight
- If we skip plot `i`, the answer stays `dp[i - 1]`.
- If we take plot `i`, the previous plot cannot be taken, so we use `dp[i - 2] + reward[i]`.
- Since choosing no plot is allowed, keep `0` as a valid answer.

## Invariant
After processing plot `i`, `dp[i]` stores the maximum valid reward obtainable from plots `0...i`, without choosing two adjacent plots and while allowing the empty selection.

## Algorithm (with one-line why at important steps)
1. If `n = 0`, return `0`.
   - Why: no plots means no reward.
2. Set `dp[0] = max(0, reward[0])`.
   - Why: the first plot can be taken or skipped.
3. If `n = 1`, return `dp[0]`.
   - Why: only one decision exists.
4. Set `dp[1] = max(dp[0], reward[1], 0)`.
   - Why: the first two plots are adjacent, so we can take at most one of them.
5. For every plot `i` from `2` to `n - 1`, compute take and skip.
   - Why: every optimal answer either includes the current plot or does not.
6. Use `dp[i] = max(dp[i - 1], dp[i - 2] + reward[i], 0)`.
   - Why: this keeps the best valid choice and protects all-negative cases.
7. Return `dp[n - 1]`.
   - Why: it stores the best answer for the full row.

## Mental Cheat Sheet
| Item | Meaning |
|---|---|
| Pattern | Take / Skip DP |
| State | `dp[i]` = best valid reward up to plot `i` |
| Skip current | `dp[i - 1]` |
| Take current | `dp[i - 2] + reward[i]` |
| Empty selection | keep answer at least `0` |
| Answer | `dp[n - 1]` |

## Complexity
- Time Complexity: `O(n)`
- Space Complexity: `O(n)`
- Auxiliary Space: `O(n)`

## Tiny Dry Run
For `reward = [3, 8, 4, 9, 2]`:

- `dp[0] = 3`
- `dp[1] = 8`
- `dp[2] = max(8, 3 + 4) = 8`
- `dp[3] = max(8, 8 + 9) = 17`
- `dp[4] = max(17, 8 + 2) = 17`

Answer = `17`.

## Notes / Pitfalls
- Do not greedily take all positive values.
- Do not use `dp[i - 1] + reward[i]`; that may take adjacent plots.
- Handle all-negative arrays by returning `0`.
- Use `long` in Java because totals can exceed `int` range.
