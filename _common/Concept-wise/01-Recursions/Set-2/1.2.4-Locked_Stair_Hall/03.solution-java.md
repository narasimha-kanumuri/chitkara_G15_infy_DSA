# Locked Stair Hall — Java Solution

## Language
Java

## Plain-English Implementation Walkthrough
1. We read the target step `n` and number of locked steps `m`.
   This tells us the stair size and how many blocked positions exist.

2. We create a `boolean` array named `locked`.
   This lets us check in constant time whether a step is blocked.

3. We mark every locked step as `true`.
   Then the recursive method can reject it immediately.

4. We create `countWays(current, n, locked)`.
   The parameter `current` tells where the student is standing now.

5. If `current > n`, we return `0`.
   This path crossed the target and cannot be counted.

6. If `locked[current]` is true, we return `0`.
   The student is not allowed to stand on that step.

7. If `current == n`, we return `1`.
   This means one valid path has reached the target exactly.

8. If the current step is prime, we only move to `current + 1`.
   This directly implements the hall rule.

9. If the current step is not prime, we try both `current + 1` and `current + 2`.
   We add both results because both branches create different paths.

10. We print `countWays(0, n, locked)`.
   The climb starts from step `0`.

## Complete Solution Code
```java
import java.util.Scanner;

public class Main {
    static boolean isPrime(int x) {
        if (x < 2) {
            return false;
        }

        for (int d = 2; d * d <= x; d++) {
            if (x % d == 0) {
                return false;
            }
        }

        return true;
    }

    static long countWays(int current, int n, boolean[] locked) {
        if (current > n) {
            return 0;
        }

        if (locked[current]) {
            return 0;
        }

        if (current == n) {
            return 1;
        }

        if (isPrime(current)) {
            return countWays(current + 1, n, locked);
        }

        long total = 0;
        total += countWays(current + 1, n, locked);
        total += countWays(current + 2, n, locked);

        return total;
    }

    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);

        int n = sc.nextInt();
        int m = sc.nextInt();

        boolean[] locked = new boolean[n + 1];

        for (int i = 0; i < m; i++) {
            int step = sc.nextInt();
            if (step >= 1 && step <= n) {
                locked[step] = true;
            }
        }

        System.out.println(countWays(0, n, locked));
        sc.close();
    }
}
```

## Code Deconstruction
- `isPrime(int x)` checks if a step number is prime.
  - It returns `false` for `0` and `1`.
  - It tries divisors only up to `sqrt(x)` using `d * d <= x`.

- `countWays(int current, int n, boolean[] locked)` is the recursive function.
  - `current` is the recursive state.
  - `n` is the target.
  - `locked` stores blocked steps.

- `current > n` is an invalid base case.
  - The move went beyond the target.

- `locked[current]` is another invalid base case.
  - The path landed on a locked step.

- `current == n` is the success base case.
  - One valid path is complete.

- `isPrime(current)` restricts the next move.
  - Prime-numbered steps allow only `+1`.

- The final `total` adds both normal choices.
  - This is counting recursion, so valid branch counts are added.

## Test Cases

| `4 0` | `3` | Three valid paths under prime rule |
| `6 1` / `5` | `3` | Step `5` is locked |
| `3 1` / `3` | `0` | Target is locked |
| `2 0` | `2` | Paths are `0 -> 2` and `0 -> 1 -> 2` |

## Language-Specific Notes
- `boolean[]` is efficient for checking locked steps.
- The code assumes input follows the constraints.
- `long` is safer than `int` for counting paths.
- This is a plain recursive solution and may recompute states.
- A later memoized version can store the result for each `current` step.

## Complexity Summary
- Time Complexity: `O(2^n * sqrt(n))` worst case because each call may check primality.
- Space Complexity: `O(n)` for recursion stack and locked array.
- Auxiliary Space: `O(n)` recursion stack plus locked array.
