# Binary Memory Builder — Quick Solution

## Idea (Simple)

We build the binary sequence from left to right.
At each position, we may place `0` or `1`.
But we cannot place `1` immediately after another `1`.
Also, the final sequence must end with `0`.

So we use memoized recursion with state `(position, lastBit)`.

## Pattern Recognition Clues

- We are counting all valid sequences.
- Choices repeat at each position: place `0` or place `1`.
- The next choice depends on the previous bit.
- Repeated recursive states suggest memoization.

## State Discovery

The first simple state is `ways(pos)`.

But `ways(pos)` is not enough.
It does not know whether the previous bit was `0` or `1`.

That matters because after `1`, we cannot place another `1`.

So we use:

```text
solve(pos, lastBit)
```

Meaning:

```text
number of valid completions from position pos,
when the previous bit is lastBit
```

## Pattern

Top-Down DP with Last Bit State

## Core Insight

- Placing `0` is always allowed.
- Placing `1` is allowed only if the previous bit is not `1`.
- At the end, count only sequences whose last placed bit is `0`.

## Invariant

For every state `solve(pos, lastBit)`:

- Positions before `pos` are already fixed.
- No consecutive `1`s exist in the prefix built so far.
- `lastBit` correctly stores the bit placed at `pos - 1`.
- The function counts only completions that can still become valid final sequences.

## Algorithm (with one-line why at important steps)

1. Start with `solve(0, 0)`.
   - Why: no bit is placed yet, so treating previous bit as `0` allows either first bit.
2. If `pos == n`, return `1` only when `lastBit == 0`.
   - Why: the sequence must end with `0`.
3. If the state is cached, return it.
   - Why: the same `(pos, lastBit)` appears many times.
4. Place `0` and move to `solve(pos + 1, 0)`.
   - Why: `0` can follow any previous bit.
5. If `lastBit == 0`, place `1` and move to `solve(pos + 1, 1)`.
   - Why: this prevents consecutive `1`s.
6. Save and return the total.
   - Why: future calls can reuse the result.

## Mental Cheat Sheet

| Item | Meaning |
|---|---|
| Pattern | Memoized recursion with last bit state |
| State | `(pos, lastBit)` |
| Choice | place `0` or place `1` |
| Restriction | `1` cannot follow `1` |
| End rule | final bit must be `0` |
| Answer | `solve(0, 0)` |

## Complexity

- Time Complexity: `O(n)`
- Space Complexity: `O(n)`
- Auxiliary Space: `O(n)` for memo table and recursion stack

There are only `2 * (n + 1)` useful states.

## Tiny Dry Run

For `n = 3`, valid sequences ending in `0` are:

```text
000
010
100
```

Count = `3`.

The sequence `110` is invalid because it has consecutive `1`s.
The sequence `101` is not counted because it ends in `1`.

## Notes / Pitfalls

- Do not cache only by `pos`; the previous bit changes allowed choices.
- Do not count sequences ending in `1`.
- Place `1` only when `lastBit == 0`.
- Use `BigInteger` because the answer can grow large.
- The starting previous bit can be treated as `0` to allow both first choices.
