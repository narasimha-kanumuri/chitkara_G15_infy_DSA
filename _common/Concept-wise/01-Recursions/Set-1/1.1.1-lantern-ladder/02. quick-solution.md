# Quick Solution — Lantern Ladder

## Idea (Simple)

We count all ways to climb from position `0` to rung `n`.
At each position, there are two possible choices: jump `1` rung or jump `2` rungs.
But a broken rung cannot be landed on, and two `2-rung` jumps cannot happen back-to-back.
So the recursion must remember whether the previous jump was `2`.

## Pattern Recognition Clues

- The problem asks for the number of valid ways, so this is counting recursion.
- From each position, we expand choices: `+1` or `+2`.
- A future choice depends on the previous jump.
- Broken rungs act like blocked states that return zero ways.

## State Discovery

The simple state is `f(pos)`: number of ways to reach the top from `pos`.
But `f(pos)` is not enough.
It forgets whether the previous jump was `2`.
That matters because another `2-rung` jump may be blocked.
So the final state is:

- `solve(pos, lastWasTwo)`

Here, `pos` is the current position.
`lastWasTwo` tells whether the last jump used to reach `pos` was a `2-rung` jump.

## Pattern

Recursive Exploration with Previous Choice State

## Core Insight

- A `1-rung` jump is always allowed if it lands inside the tower and not on a broken rung.
- A `2-rung` jump is allowed only when the previous jump was not `2`.
- If a position is invalid or broken, it contributes `0` ways.

## Invariant

For every call `solve(pos, lastWasTwo)`, the function returns the number of valid ways to reach rung `n` from `pos`, assuming the last jump status is correctly represented by `lastWasTwo`.

## Algorithm (with one-line why at important steps)

1. Mark all broken rungs in a boolean array.
   - Why: we need fast checking before accepting a position.
2. Start recursion from `solve(0, false)`.
   - Why: before climbing, no previous jump exists.
3. If `pos == n`, return `1`.
   - Why: one complete valid sequence has been formed.
4. If `pos > n` or `pos` is broken, return `0`.
   - Why: this path cannot be used.
5. Try a `1-rung` jump using `solve(pos + 1, false)`.
   - Why: the new last jump is not `2`.
6. Try a `2-rung` jump only when `lastWasTwo == false`.
   - Why: two consecutive `2-rung` jumps are not allowed.
7. Return the sum of all valid choices.
   - Why: each valid choice creates a separate climbing sequence.

## Mental Cheat Sheet

| Item | Meaning |
|---|---|
| Pattern | Recursive choice expansion |
| State | Current position + whether previous jump was `2` |
| Blocked state | Broken rung or position beyond `n` |
| Valid finish | `pos == n` returns `1` |
| Answer | `solve(0, false)` |

## Complexity

- Time Complexity: `O(2^n)` in the worst case without memoization
- Space Complexity: `O(n)` because of the recursion call stack
- Auxiliary Space: `O(n)` for recursion stack and broken-rung array

## Tiny Dry Run

For `n = 4` and broken rung `{2}`:

- Start at `0`.
- Jump `1` to `1`.
- From `1`, jump `1` lands on broken rung `2`, so it gives `0`.
- From `1`, jump `2` goes to `3`.
- From `3`, jump `1` reaches `4`, so it gives `1`.

Answer = `1`.

## Notes / Pitfalls

- Do not allow landing on broken rungs.
- Jumping over a broken rung is allowed if you do not land on it.
- Do not use only `pos` as state because the previous jump affects the next choice.
- The starting call should use `lastWasTwo = false`.
