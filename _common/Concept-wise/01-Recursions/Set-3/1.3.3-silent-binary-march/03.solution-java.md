## Plain-English Implementation Walkthrough

1. We create a `restricted` array with size `n + 1`.  
   This lets us use positions from `1` to `n` directly.

2. We mark every restricted position as `true`.  
   A `true` value means `1` cannot be placed there.

3. We start recursion from position `1`.  
   The code is built from left to right.

4. We pass `prevBit` into the recursive method.  
   This is needed because the next `1` depends on the previous bit.

5. If `pos > n`, we return `1`.  
   This means all positions were filled without breaking any rule.

6. We first place `0`.  
   This is always safe because restricted positions only block `1`.

7. We place `1` only when three checks pass.  
   The previous bit must be `0`, the current position must not be restricted, and the current position must not be the final position.

8. Each recursive branch returns a number of valid completions.  
   We add these numbers because the problem asks for a count.

9. The final returned value is printed as the answer.

## Complete Solution Code

```java
import java.util.*;

public class Main {
    static long solve(int pos, int prevBit, int n, boolean[] restricted) {
        if (pos > n) {
            return 1;
        }

        long ways = 0;

        // Choice 1: place 0 at this position.
        ways += solve(pos + 1, 0, n, restricted);

        // Choice 2: place 1 if all rules allow it.
        if (prevBit == 0 && !restricted[pos] && pos != n) {
            ways += solve(pos + 1, 1, n, restricted);
        }

        return ways;
    }

    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);

        int n = sc.nextInt();
        int k = sc.nextInt();

        boolean[] restricted = new boolean[n + 1];

        for (int i = 0; i < k; i++) {
            int position = sc.nextInt();
            restricted[position] = true;
        }

        long answer = solve(1, 0, n, restricted);
        System.out.println(answer);
    }
}
```

## Code Deconstruction

### `restricted` array

```java
boolean[] restricted = new boolean[n + 1];
```

This array marks positions where `1` cannot be placed.  
Index `0` is unused so the code can use natural position numbers.

### `solve(pos, prevBit, n, restricted)`

This method returns the number of valid ways to fill positions from `pos` to `n`.

- `pos` is the current position being filled.
- `prevBit` is the bit placed at the previous position.
- `restricted` tells whether `1` is blocked at a position.

### Base Case

```java
if (pos > n) {
    return 1;
}
```

If all positions are filled, one valid code has been completed.  
The final-position rule is already enforced before placing `1`.

### Placing `0`

```java
ways += solve(pos + 1, 0, n, restricted);
```

`0` is always allowed.  
After placing `0`, the previous bit for the next call becomes `0`.

### Placing `1`

```java
if (prevBit == 0 && !restricted[pos] && pos != n) {
    ways += solve(pos + 1, 1, n, restricted);
}
```

This condition checks all three rules for placing `1`:

- It must not create consecutive `1`s.
- The position must not be restricted.
- The final position must not be `1`.

## Language-Specific Notes

- `long` is used instead of `int` because counts can grow quickly.
- Java initializes all values in a `boolean[]` to `false` by default.
- Pure recursion repeats many subproblems. This is intentional for Day 1 recursion practice.
- On Day 2, the same state can be memoized as `dp[pos][prevBit]`.

## Complexity Summary

- Time Complexity: `O(2^n)` for pure recursion.
- Space Complexity: `O(n)` due to recursion stack.
- Auxiliary Space: `O(n)` due to recursion stack.
