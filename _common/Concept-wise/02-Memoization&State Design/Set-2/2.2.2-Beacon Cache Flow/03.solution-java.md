# Beacon Cache Flow — Java Solution

## Language

Java

## Plain-English Implementation Walkthrough

1. We use `BigInteger` because the recurrence grows quickly.
   Normal integer types may overflow.

2. We create a method `solve(day, memo)`.
   This method returns the brightness value for one day.

3. The parameter `day` is the state.
   Once `day` is known, the answer for that day is fixed.

4. The memo array stores already computed brightness values.
   `memo[day]` stores the answer for that day.

5. If `day == 0`, we return `BigInteger.ONE`.
   This matches the first base rule.

6. If `day == 1`, we return `BigInteger.valueOf(2)`.
   This matches the second base rule.

7. If `memo[day]` is not `null`, we return it.
   This means the same day was already calculated earlier.

8. For other days, we add the brightness of the previous two days.
   This directly follows the recurrence.

9. If `day % 4 == 0`, we subtract `1` from the computed value.
   This applies the periodic correction only on the required days.

10. We store the final value in `memo[day]`.
    Then we return it.

11. The final answer is `solve(n, memo)`.
    This computes only the values needed to reach day `n`.

## Complete Solution Code

```java
import java.io.BufferedReader;
import java.io.InputStreamReader;
import java.math.BigInteger;

public class Main {

    private static final BigInteger TWO = BigInteger.valueOf(2);

    private static BigInteger solve(int day, BigInteger[] memo) {
        if (day == 0) {
            return BigInteger.ONE;
        }

        if (day == 1) {
            return TWO;
        }

        if (memo[day] != null) {
            return memo[day];
        }

        BigInteger value = solve(day - 1, memo).add(solve(day - 2, memo));

        if (day % 4 == 0) {
            value = value.subtract(BigInteger.ONE);
        }

        memo[day] = value;
        return value;
    }

    private static BigInteger brightnessOnDay(int n) {
        BigInteger[] memo = new BigInteger[n + 1];
        return solve(n, memo);
    }

    public static void main(String[] args) throws Exception {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        int n = Integer.parseInt(br.readLine().trim());
        System.out.println(brightnessOnDay(n));
    }
}
```

## Code Deconstruction

### Constant `TWO`

```java
private static final BigInteger TWO = BigInteger.valueOf(2);
```

This stores the base value for day `1`.
It avoids recreating the same `BigInteger` value repeatedly.

### `solve` method

```java
private static BigInteger solve(int day, BigInteger[] memo)
```

This method computes the brightness for one day.
The only changing state is `day`.

### Base cases

```java
if (day == 0) return BigInteger.ONE;
if (day == 1) return TWO;
```

These two values start the recurrence.
Without these base cases, recursion would not stop correctly.

### Memo check

```java
if (memo[day] != null) {
    return memo[day];
}
```

This prevents repeated computation of the same day.

### Recurrence computation

```java
BigInteger value = solve(day - 1, memo).add(solve(day - 2, memo));
```

The current day's brightness starts as the sum of the previous two days.

### Periodic correction

```java
if (day % 4 == 0) {
    value = value.subtract(BigInteger.ONE);
}
```

This applies the correction only when the day number is divisible by `4`.

### Saving result

```java
memo[day] = value;
return value;
```

The final brightness for this day is saved and returned.

## Language-Specific Notes

- `BigInteger` is immutable. Methods like `.add()` and `.subtract()` return a new value.
- Do not use `+` or `-` for `BigInteger` arithmetic.
- `memo[day] == null` means the day has not been computed yet.
- The recursion depth can reach `n`; this is acceptable for memoization teaching, but tabulation can avoid stack depth.
- The `%` operator works on `int day`, not on `BigInteger`, because the divisibility check is for the day index.

## Complexity Summary

- Time Complexity: `O(n)`
- Space Complexity: `O(n)`
- Auxiliary Space: `O(n)`

Each day from `0` to `n` is computed once.
BigInteger arithmetic also depends on the number of digits in the brightness value.
