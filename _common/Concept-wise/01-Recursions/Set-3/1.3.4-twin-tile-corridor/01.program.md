# Twin Tile Corridor

## Concepts Involved

- Base Case Design
- Return Meaning
- Choice Expansion
- Recursive State Definition
- Recursion Tree Analysis
- Constraint Propagation
- Counting Recursion
- Optimization vs Counting Distinction

## Problem Statement

A corridor of length `n` must be fully covered using tiles of length `1` and length `2`.

The corridor must follow two rules:

1. Two length-`2` tiles cannot be placed next to each other.
2. The final tile placed at the end of the corridor must be a length-`1` tile.

Return the total number of valid tiling arrangements.

The corridor is covered from left to right.  
At each step, you may place a length-`1` tile or a length-`2` tile if the placement is valid.

## Input Format

- The first line contains an integer `n`.

## Output Format

Print one integer: the number of valid ways to cover the corridor.

## Constraints

- `1 <= n <= 30`
- The intended solution for this file is pure recursion, so the input size is kept small for Day 1 teaching.

## Examples

### Example 1

**Input**

```text
4
```

**Output**

```text
3
```

**Explanation**

The valid tilings are:

```text
1 + 1 + 1 + 1
2 + 1 + 1
1 + 2 + 1
```

The tiling `1 + 1 + 2` is invalid because the final tile is length `2`.  
The tiling `2 + 2` is invalid because two length-`2` tiles are adjacent and the final tile is also length `2`.

### Example 2

**Input**

```text
5
```

**Output**

```text
4
```

**Explanation**

The valid tilings are:

```text
1 + 1 + 1 + 1 + 1
2 + 1 + 1 + 1
1 + 2 + 1 + 1
1 + 1 + 2 + 1
```

Every valid tiling ends with a length-`1` tile.

### Example 3

**Input**

```text
1
```

**Output**

```text
1
```

**Explanation**

Only one tile of length `1` is needed.

## Edge-Case Thinking Prompts

- What should the recursion return when the remaining corridor length becomes `0`?
- Why is placing a length-`2` tile not allowed when exactly `2` length remains?
- Why does the function need to remember whether the previous tile was length `2`?
- Is this problem asking for a count, or for a minimum / maximum value?
- What should happen if a tile is larger than the remaining length?

## Thinking Like a Programmer

Before writing code, define:

```text
State:
Choice:
Transition:
Base case:
Order:
Answer:
```

For this problem:

```text
State: remaining corridor length and whether the previous tile was length 2
Choice: place a length-1 tile, or place a valid length-2 tile
Transition: reduce remaining length by tile size and update previous tile information
Base case: if remaining length is 0, one valid tiling is completed
Order: recursively cover the corridor from left to right
Answer: solve(n, false)
```
