# Silent Sequence Cache — Quick Solution

## Idea (Simple)

We build the binary sequence from left to right.
At each position, we can try placing `0` or `1`.
But `1` cannot follow `1`, restricted positions cannot contain `1`, and the final sequence must end with `0`.

So we use memoized recursion with state `(pos, lastBit)`.

## Pattern Recognition Clues

- We are asked to count all valid sequences.
- Choices repeat at every position.
- The next choice depends on the previous bit.
- Repeated recursive suffixes suggest memoization.

## State Discovery

The first simple state is `solve(pos)`.

But `solve(pos)` is not enough.
It does not know whether the previous bit was `1`.

That missing information decides whether we may place `1` now.

So we use:

```text
solve(pos, lastBit)
```

Meaning:

```text
number of valid ways to fill positions from pos to n - 1,
when the previous placed bit is lastBit
```

## Pattern

Top-Down Counting DP with Last Bit State

## Core Insight

- Placing `0` is always allowed.
- Placing `1` is allowed only if the previous bit is `0` and the current position is not restricted.
- At the end, count only sequences whose last bit is `0`.

## Invariant

For every state `solve(pos, lastBit)`:

- All positions before `pos` have already been filled validly.
- `lastBit` correctly stores the bit placed at `pos - 1`.
- No prefix counted so far contains consecutive `1`s.
- Restricted positions before `pos` have been respected.

## Algorithm (with one-line why at important steps)

1. Start with `solve(0, 0)`.
   - Why: no previous bit exists, so we treat it as `0`.
2. If `pos == n`, return `1` only when `lastBit == 0`.
   - Why: the completed sequence must end with `0`.
3. If the state is cached, return it.
   - Why: the same suffix state can repeat.
4. Try placing `0` and move to `solve(pos + 1, 0)`.
   - Why: `0` is always valid.
5. Try placing `1` only if `lastBit == 0` and `blocked[pos] == '0'`.
   - Why: this enforces both rules for placing `1`.
6. Store and return the total.
   - Why: future calls can reuse the same answer.

## Mental Cheat Sheet

| Item | Meaning |
|---|---|
| Pattern | Memoized binary sequence counting |
| State | `(pos, lastBit)` |
| `pos` | current position to fill |
| `lastBit` | bit placed just before `pos` |
| Restricted position | cannot place `1` |
| Answer | `solve(0, 0)` |

## Complexity

- Time Complexity: `O(n)`
- Space Complexity: `O(n)`
- Auxiliary Space: `O(n)` for memo table and recursion stack

There are at most `2 * (n + 1)` states.
Each state has at most two choices.

## Tiny Dry Run

For:

```text
n = 3, blocked = 000
```

Valid sequences ending with `0` are:

```text
000
010
100
```

Count = `3`.

## Notes / Pitfalls

- Do not cache only by position.
- Do not place `1` at a restricted position.
- Do not count sequences ending with `1`.
- Use `BigInteger` because the count can grow large.
- Treat the starting previous bit as `0` so the first position may be `1` if unrestricted.
