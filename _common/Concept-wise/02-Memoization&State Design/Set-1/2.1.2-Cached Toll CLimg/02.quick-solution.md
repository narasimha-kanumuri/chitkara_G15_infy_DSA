# Cached Toll Climb — Quick Solution

## Idea (Simple)

We need the minimum toll to cross all steps.
At each landed step, we can pay the toll or use the one coupon if it is still unused.
Then we may jump `1` or `2` steps forward.

Because the same step can be reached with the coupon used or unused, we use memoization with coupon state.

## Pattern Recognition Clues

- The problem asks for a minimum cost.
- From each step, the same choices repeat: jump `1` or jump `2`.
- A one-time coupon affects future decisions.
- Repeated recursive calls suggest top-down DP with memoization.

## State Discovery

The first simple state is `dp[i]`: minimum cost from step `i`.

But `dp[i]` is not enough.
It forgets whether the coupon is still available.

That matters because reaching step `i` with the coupon unused is better than reaching it with the coupon already used.

So we use this state:

```text
solve(i, used)
```

Meaning:

```text
minimum cost to finish from step i,
where used tells whether the coupon has already been used
```

## Pattern

Top-Down DP with One-Time Coupon State

## Core Insight

- At each step, there are two cost choices: pay or use coupon.
- After handling the current step, move to `i + 1` or `i + 2`.
- The coupon state must be part of the cache key.

## Invariant

For every state `solve(i, used)`:

- All costs before step `i` have already been decided.
- `used = 0` means the coupon is still available.
- `used = 1` means the coupon cannot be used again.
- The function returns the minimum valid remaining cost from step `i`.

## Algorithm (with one-line why at important steps)

1. Start with `solve(0, 0)`.
   - Why: the courier must first land on step `0`, and the coupon is unused.
2. If `i >= n`, return `0`.
   - Why: moving beyond the last step completes the climb.
3. If the state is already cached, return it.
   - Why: this avoids recomputing the same subproblem.
4. Compute the cost when paying `cost[i]`.
   - Why: paying is always allowed.
5. If the coupon is unused, compute the cost when using it on step `i`.
   - Why: the current toll becomes `0`, and coupon becomes used.
6. Store and return the smaller value.
   - Why: the task asks for minimum total toll.

## Mental Cheat Sheet

| Item | Meaning |
|---|---|
| Pattern | Memoization with extra state |
| State | `(i, used)` |
| `i` | current landed step |
| `used` | whether coupon has been used |
| Choices | pay toll or use coupon if available |
| Answer | `solve(0, 0)` |

## Complexity

- Time Complexity: `O(n)`
- Space Complexity: `O(n)`
- Auxiliary Space: `O(n)` for memo table and recursion stack

There are `2 * n` useful states.
Each state does constant work.

## Tiny Dry Run

For:

```text
cost = [5, 100, 5]
```

Start: `solve(0, 0)`

Key choices:

```text
Pay step 0: 5 + min(solve(1,0), solve(2,0))
Use coupon on step 0: 0 + min(solve(1,1), solve(2,1))
```

The best final cost is `5`.

## Notes / Pitfalls

- Do not cache only by step index.
- `used` must be part of the state.
- Moving beyond the last step adds `0` cost.
- Use `long`, not `int`, because toll sums can be large.
- The coupon is used at most once, not once per path segment.
