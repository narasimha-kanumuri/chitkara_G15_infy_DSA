# 12-Day Plan — DP-Heavy, Exam-Oriented

| Day | Topic | Core Skill | Main Pattern | Difficulty | Why This Day Exists |
| :--- | :--- | :--- | :--- | :--- | :--- |
| 1 | Recursion for DP | Base case, call stack, recursion tree, return meaning | Shrink input / choose next step | Medium | Build only the recursion needed for DP. No deep recursion theory. |
| 2 | Memoization Bridge + Light Backtracking | State = what changes, overlapping subproblems, cache key | Recursion → memoization | Medium | Move fast from recursion to DP thinking. Backtracking only as one “choice tree” example. |
| 3 | DP I — 1D Linear DP | Define dp[i], base cases, recurrence | Climbing stairs / min cost / frog jump | Medium | First real DP day. Students learn state, transition, answer location. |
| 4 | DP II — Take / Skip DP | Choice-based recurrence | House robber / max non-adjacent / delete-and-earn style | Medium | Separates “linear forward DP” from “decision DP”. Very important for story problems. |
| 5 | DP III — Count vs Minimize | Identify whether answer is count, min, max, or possible | Coin min vs coin ways / decode ways | Medium-Hard | Prevents the most common DP mistake: mixing counting and optimization transitions. |
| 6 | Interleave + Retrieval Day 1 | Recall 1D patterns under pressure | Mixed 1D DP timed drills | Medium-Hard | Forces pattern discrimination before moving to 2D. |
| 7 | DP IV — 2D Sequence DP | Table meaning, two-index state | LCS / edit distance / palindromic subsequence | Hard | Students learn dp[i][j] as meaning, not as a template. |
| 8 | DP V — Grid / Path DP | Coordinate transitions | Unique paths / min path / obstacles | Medium-Hard | Easier 2D than strings, but reinforces table direction and base rows/cols. |
| 9 | DP VI — Knapsack I | Include/exclude under constraint | 0/1 knapsack / subset sum / equal partition | Hard | Core choice-optimization family. Needs separate focus. |
| 10 | DP VII — Knapsack II + Counting DP | Bounded vs unbounded, combinations vs permutations | Unbounded coin / target sum / count ways | Hard | Builds the hardest distinction: reuse allowed? order matters? count or optimize? |
| 11 | Tree DP Basics + Hybrid Recognition | Postorder state aggregation | Diameter / House Robber III / subtree answer | Hard | Only basics. No rerooting. Use as stretch + hybrid thinking. |
| 12 | Story Problems + Mock Strategy | Story → pattern → state → recurrence → code | Mixed exam-style problems | Mixed | Final conversion day: pattern recognition under exam pressure. |
