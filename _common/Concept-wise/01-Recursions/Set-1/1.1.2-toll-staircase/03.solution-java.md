# Java Solution — Toll Staircase

## Language

Java

## Plain-English Implementation Walkthrough

1. We store all toll values in an integer array.
   The index of the array is the step number.
   Related code: `int[] cost = new int[n];`

2. The courier may start from step `0` or step `1`.
   So we compute both starts and keep the smaller answer.
   Related code: `Math.min(solve(0, false, cost), solve(1, false, cost))`

3. The recursive method receives the current step and pass status.
   We need both because the free pass changes the cost decision.
   Related code: `solve(int i, boolean usedPass, int[] cost)`

4. If `i >= cost.length`, the courier has crossed the staircase.
   No more toll is needed, so we return `0`.
   Related code: `if (i >= cost.length) return 0;`

5. First, we handle the normal choice of paying for the current step.
   After paying, the courier may jump one or two steps.
   Related code: `payCurrent = cost[i] + Math.min(...)`

6. If the free pass is still unused, we also try making the current step free.
   After using the pass, future calls must know it is already used.
   Related code: `if (!usedPass) { usePassCurrent = Math.min(... true ...); }`

7. We return the smaller value between paying and using the pass.
   This matches the goal of minimum total toll.
   Related code: `return Math.min(payCurrent, usePassCurrent);`

8. The code uses plain recursion for Day 1.
   This shows the choice tree clearly before memoization is introduced later.

## Complete Solution Code

```java
import java.util.Scanner;

public class Main {
    private static final int INF = 1_000_000_000;

    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);

        int n = sc.nextInt();
        int[] cost = new int[n];

        for (int i = 0; i < n; i++) {
            cost[i] = sc.nextInt();
        }

        int answer = Math.min(
            solve(0, false, cost),
            solve(1, false, cost)
        );

        System.out.println(answer);
        sc.close();
    }

    private static int solve(int i, boolean usedPass, int[] cost) {
        if (i >= cost.length) {
            return 0;
        }

        int payCurrent = cost[i] + Math.min(
            solve(i + 1, usedPass, cost),
            solve(i + 2, usedPass, cost)
        );

        int usePassCurrent = INF;
        if (!usedPass) {
            usePassCurrent = Math.min(
                solve(i + 1, true, cost),
                solve(i + 2, true, cost)
            );
        }

        return Math.min(payCurrent, usePassCurrent);
    }
}
```

## Code Deconstruction

- `cost` stores the toll for each step.
- `solve(i, usedPass, cost)` returns the minimum toll needed from step `i` onward.
- `i >= cost.length` means the courier has crossed the staircase.
- `payCurrent` represents paying the toll at step `i` and then choosing the cheaper jump.
- `usePassCurrent` represents using the free pass on step `i`.
- `usedPass` becomes `true` after the free pass is used.
- `Math.min(payCurrent, usePassCurrent)` keeps the cheaper valid decision.
- `Math.min(solve(0, false, cost), solve(1, false, cost))` handles the two allowed starting positions.

## Language-Specific Notes

- `int` is enough for the given constraints because `n <= 30` and each cost is at most `10000`.
- `INF` is used only as a safe initial value for the pass option when it is not available.
- This recursive solution repeats many states. Day 2 can improve it using memoization with state `(i, usedPass)`.
- `Scanner` is used for readability in classroom code.

## Complexity Summary

- Time Complexity: `O(2^n)` in the worst case without memoization
- Space Complexity: `O(n)` because recursive calls can go up to depth `n`
- Auxiliary Space: `O(n)` for the recursion call stack
