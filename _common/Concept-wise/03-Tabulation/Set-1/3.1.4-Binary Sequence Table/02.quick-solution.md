## Day 03 — Tabulation (Bottom-Up DP) — Problem 4

## quick-solution.md

### Idea (Simple)
We need to count binary strings of length `n`.
The string cannot contain consecutive `1`s and must end with `0`.
So at every position, we track how many valid strings end with `0` and how many end with `1`.

### Pattern
Bottom-Up DP with last-bit state

### Core Insight
- A `0` can be placed after both `0` and `1`.
- A `1` can be placed only after `0`.
- Since the final string must end with `0`, the answer is only the `endWithZero[n]` state.

### Invariant
At every length `i`:
- `endWithZero[i]` = number of valid binary strings of length `i` ending with `0`
- `endWithOne[i]` = number of valid binary strings of length `i` ending with `1`

These states always contain only strings with no consecutive `1`s.

### Algorithm (with one-line why at important steps)
1. Create two DP arrays: `endWithZero` and `endWithOne`.
   - Why: the validity of placing `1` depends on the previous bit.
2. Initialize `endWithZero[1] = 1` and `endWithOne[1] = 1`.
   - Why: both `0` and `1` are valid one-bit strings.
3. For every length `i` from `2` to `n`, compute `endWithZero[i] = endWithZero[i - 1] + endWithOne[i - 1]`.
   - Why: `0` can follow any valid previous ending.
4. Compute `endWithOne[i] = endWithZero[i - 1]`.
   - Why: `1` can only follow a string ending in `0`.
5. Print `endWithZero[n]`.
   - Why: the problem requires the final bit to be `0`.

### Code Cheat Sheet
| Code / Built-in | Meaning / Role |
|---|---|
| `endWithZero[i]` | valid strings of length `i` ending with `0` |
| `endWithOne[i]` | valid strings of length `i` ending with `1` |
| `BigInteger` | stores exact large counts safely |
| `.add(...)` | adds two `BigInteger` values |

### Complexity
- Time: `O(n)`
- Space: `O(n)`
- Auxiliary Space: `O(n)`

### Tiny Dry Run
For `n = 4`:
- length `1`: `endWithZero = 1`, `endWithOne = 1`
- length `2`: `endWithZero = 2`, `endWithOne = 1`
- length `3`: `endWithZero = 3`, `endWithOne = 2`
- length `4`: `endWithZero = 5`, `endWithOne = 3`
- Answer = `5`

### Notes / Pitfalls
- Do not return total valid strings; return only strings ending with `0`.
- Do not allow `endWithOne[i]` to come from `endWithOne[i - 1]`.
- Use `BigInteger` because exact counts grow quickly.
- The DP is iterative, so there is no recursion depth issue.
