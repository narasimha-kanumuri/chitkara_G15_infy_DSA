## Plain-English Implementation Walkthrough

1. We use `BigInteger` because the number of valid binary sequences can become very large.

2. We use the method `solve(pos, lastBit, n, memo)` to represent one recursion state.
   It means: “how many valid sequences can be completed from position `pos`?”

3. The parameter `pos` tells which position we are about to fill.
   Positions before `pos` are already chosen.

4. The parameter `lastBit` tells what bit was placed just before `pos`.
   This is needed because `1` cannot follow `1`.

5. If `pos == n`, all positions are filled.
   At this point, we return `1` only if `lastBit == 0`.
   This checks the final rule that the sequence must end with `0`.

6. If `memo[pos][lastBit]` is not `null`, we return it.
   This means the state was already solved earlier.

7. We always try placing `0`.
   After placing `0`, the next state has `lastBit = 0`.

8. We try placing `1` only when `lastBit == 0`.
   This prevents two consecutive `1`s.

9. We add the counts from valid choices.
   This is correct because we are counting all possible valid sequences.

10. We save the total in `memo[pos][lastBit]` and return it.
    This completes memoization for the current state.

11. The final answer is `solve(0, 0, n, memo)`.
    We use `lastBit = 0` at the start because no previous `1` exists.

## Complete Solution Code

```java
import java.io.BufferedReader;
import java.io.InputStreamReader;
import java.math.BigInteger;

public class Main {

    private static BigInteger solve(int pos, int lastBit, int n, BigInteger[][] memo) {
        if (pos == n) {
            return lastBit == 0 ? BigInteger.ONE : BigInteger.ZERO;
        }

        if (memo[pos][lastBit] != null) {
            return memo[pos][lastBit];
        }

        BigInteger total = BigInteger.ZERO;

        // Place 0. This is always allowed.
        total = total.add(solve(pos + 1, 0, n, memo));

        // Place 1 only if the previous bit was not 1.
        if (lastBit == 0) {
            total = total.add(solve(pos + 1, 1, n, memo));
        }

        memo[pos][lastBit] = total;
        return total;
    }

    private static BigInteger countValidSequences(int n) {
        BigInteger[][] memo = new BigInteger[n + 1][2];
        return solve(0, 0, n, memo);
    }

    public static void main(String[] args) throws Exception {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        int n = Integer.parseInt(br.readLine().trim());
        System.out.println(countValidSequences(n));
    }
}
```

## Code Deconstruction

### `solve` method

```java
private static BigInteger solve(int pos, int lastBit, int n, BigInteger[][] memo)
```

This method represents the DP state.

- `pos` is the next position to fill.
- `lastBit` is the bit placed before this position.
- `n` is the required sequence length.
- `memo` stores answers for already solved states.

### Completed sequence check

```java
if (pos == n) {
    return lastBit == 0 ? BigInteger.ONE : BigInteger.ZERO;
}
```

This checks the final rule.
If the sequence ends with `0`, it is counted.
If it ends with `1`, it is rejected.

### Memoization check

```java
if (memo[pos][lastBit] != null) {
    return memo[pos][lastBit];
}
```

This avoids repeated recursion for the same state.

### Placing `0`

```java
total = total.add(solve(pos + 1, 0, n, memo));
```

A `0` can be placed after either `0` or `1`.
So this move is always valid.

### Placing `1`

```java
if (lastBit == 0) {
    total = total.add(solve(pos + 1, 1, n, memo));
}
```

A `1` can be placed only after `0`.
This prevents consecutive `1`s.

### Saving result

```java
memo[pos][lastBit] = total;
return total;
```

The result is saved before returning, so future calls reuse it.

## Language-Specific Notes

- `BigInteger` is immutable. Use `.add()` to add two values.
- Do not use `+` for `BigInteger` addition.
- `memo[pos][lastBit] == null` means the state has not been solved yet.
- The recursion depth can reach `n`, which is part of the memoization teaching style.
- `lastBit` is represented as `0` or `1` so it can directly index the memo table.

## Complexity Summary

- Time Complexity: `O(n)`
- Space Complexity: `O(n)`
- Auxiliary Space: `O(n)`

There are at most `2 * (n + 1)` states.
Each state does constant work, except for `BigInteger` addition cost, which depends on the number of digits in the answer.
