# Quick Solution — Toll Staircase

## Idea (Simple)

We need the minimum toll to cross the staircase.
At each step, the courier can move `1` or `2` steps.
If the courier lands on a step, that step's cost is paid unless the one free pass is used there.
So recursion must try both movement choices and both pass decisions when the pass is still available.

## Pattern Recognition Clues

- The problem asks for minimum cost, not number of ways.
- Each step creates choices: move `1` or move `2`.
- A one-time free pass changes future decisions.
- This suggests optimization recursion with an extra state for pass usage.

## State Discovery

The simple state is `solve(i)`: minimum cost from step `i` to the end.
But `solve(i)` is not enough.
It forgets whether the free pass is still available.
The best choice at step `i` changes if the pass is unused.
So the final state is:

- `solve(i, usedPass)`

Here, `i` is the current landed step.
`usedPass` tells whether the free pass has already been used.

## Pattern

Optimization Recursion with One-Time Resource State

## Core Insight

- When we land on a step, we either pay its cost or use the free pass if available.
- After handling the current step cost, we choose the cheaper future path from `i + 1` or `i + 2`.
- The answer is the better start between step `0` and step `1`.

## Invariant

For every call `solve(i, usedPass)`, the function returns the minimum extra toll needed to cross the staircase starting from landed step `i`, assuming `usedPass` correctly describes whether the free pass is already spent.

## Algorithm (with one-line why at important steps)

1. Define a recursive function `solve(i, usedPass)`.
   - Why: the current index and pass status decide the best future cost.
2. If `i >= n`, return `0`.
   - Why: the courier has already crossed the staircase.
3. Compute the cost if the courier pays for step `i`.
   - Why: paying is always one possible choice.
4. From this paid state, take the minimum of moving to `i + 1` or `i + 2`.
   - Why: the courier wants the cheaper continuation.
5. If the pass is unused, compute the cost if the current step is free.
   - Why: the pass can cancel exactly one landed step.
6. Return the minimum among all valid choices.
   - Why: this is an optimization problem.
7. Start from both `0` and `1`, and take the minimum.
   - Why: the courier may start by landing on either step.

## Mental Cheat Sheet

| Item | Meaning |
|---|---|
| Pattern | Optimization recursion |
| State | Current step + free-pass status |
| Choice | Pay current cost or use pass, then jump 1 or 2 |
| Base case | `i >= n` means crossing is complete |
| Answer | `min(solve(0, false), solve(1, false))` |

## Complexity

- Time Complexity: `O(2^n)` in the worst case without memoization
- Space Complexity: `O(n)` because of recursion depth
- Auxiliary Space: `O(n)` for the recursion call stack

## Tiny Dry Run

For `cost = [5, 100, 5, 5]`:

- Start at step `0` or `1`.
- Starting at step `1`, use the free pass on cost `100`.
- Jump from `1` to `3`, pay `5`, then cross.
- Total = `5`.

Answer = `5`.

## Notes / Pitfalls

- Do not add costs after crossing beyond the last step.
- Do not use the free pass more than once.
- Do not treat this as counting recursion. Use `min`, not `+` for alternatives.
- The pass can only cancel a step that is actually landed on.
