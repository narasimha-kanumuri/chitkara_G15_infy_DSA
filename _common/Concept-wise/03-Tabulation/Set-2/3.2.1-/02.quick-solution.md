## Idea (Simple)
We need to count ways to reach step `n` using jumps of `1`, `2`, or `3`.
The rule is that the same jump size cannot be used twice in a row.
So the answer depends not only on the current step, but also on the last jump used to reach that step.

## Pattern Recognition Clues
- We are counting all valid jump sequences.
- Each answer is built from smaller step numbers.
- The next jump depends on the previous jump size.
- This suggests dynamic programming with a last-choice state.

## State Discovery
The first idea is `dp[i]`: number of ways to reach step `i`.
But `dp[i]` is not enough.
It tells how many ways reach `i`, but it forgets the last jump size.
The next jump is valid only if it is different from the previous jump.
So we split the state:
- `endWithOne[i]`: ways to reach `i` ending with jump `1`
- `endWithTwo[i]`: ways to reach `i` ending with jump `2`
- `endWithThree[i]`: ways to reach `i` ending with jump `3`

## Pattern
Bottom-Up DP with Last Jump State

## Core Insight
- A final jump of `1` can come after jumps `2` or `3`.
- A final jump of `2` can come after jumps `1` or `3`.
- A final jump of `3` can come after jumps `1` or `2`.

## Invariant
For every step `i`:
- `endWithOne[i]` stores all valid routes to `i` whose last jump is `1`.
- `endWithTwo[i]` stores all valid routes to `i` whose last jump is `2`.
- `endWithThree[i]` stores all valid routes to `i` whose last jump is `3`.
No counted route has two consecutive equal jumps.

## Algorithm (with one-line why at important steps)
1. Create three arrays for routes ending with jumps `1`, `2`, and `3`.
   - Why: the last jump controls which next jump is allowed.
2. Set base routes for direct jumps from the ground.
   - Why: `1`, `2`, and `3` can each start a route if the target step exists.
3. For each step `i`, compute routes ending with `1` from step `i - 1` states ending with `2` or `3`.
   - Why: jump `1` cannot follow another jump `1`.
4. Compute routes ending with `2` from step `i - 2` states ending with `1` or `3`.
   - Why: jump `2` cannot follow another jump `2`.
5. Compute routes ending with `3` from step `i - 3` states ending with `1` or `2`.
   - Why: jump `3` cannot follow another jump `3`.
6. Return the sum of all three ending states at step `n`.
   - Why: a valid route may end with any allowed jump size.

## Mental Cheat Sheet
| Item | Meaning |
|---|---|
| Pattern | Counting DP with last jump state |
| Simple failed state | `dp[i]` forgets the last jump |
| Needed memory | last jump size: `1`, `2`, or `3` |
| Transition idea | current jump must differ from previous jump |
| Answer | `endWithOne[n] + endWithTwo[n] + endWithThree[n]` |

## Complexity
- Time Complexity: `O(n)`
- Space Complexity: `O(n)`
- Auxiliary Space: `O(n)`

## Tiny Dry Run
For `n = 4`:
- Valid routes include `1, 3`, `3, 1`, and `1, 2, 1`.
- Route `2, 2` is invalid because the same jump repeats.
- Route `1, 1, 1, 1` is invalid because adjacent jumps are equal.
- Answer = `3`.

## Notes / Pitfalls
- Do not use only one `dp[i]` value.
- Do not allow a state to transition from the same last jump.
- Direct jumps from the ground are valid starting routes.
- Use `BigInteger` for exact large counts.
