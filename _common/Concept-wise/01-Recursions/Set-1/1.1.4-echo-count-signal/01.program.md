# Echo Count Signal

## Concepts Involved

- Base Case Design
- Return Meaning
- Choice Expansion
- Recursive State Definition
- Recursion Tree Analysis
- Constraint Propagation
- Counting Recursion
- End-State Validation

## Problem Statement

A signal machine grows from level `0` to level `n`.
At every step, the machine can increase the signal level by either `+1` or `+2`.

A construction sequence is a list of increases that starts from level `0` and reaches exactly level `n`.

There is one final stability rule:

A construction is invalid if it ends with two consecutive `+1` increases.

Your task is to count the number of valid construction sequences that reach exactly level `n`.

A sequence is valid only if:

- Every increase is either `+1` or `+2`.
- The final level is exactly `n`.
- The last two increases must not both be `+1`.
- Two consecutive `+1` increases may appear earlier in the sequence, but not at the end.

## Input Format

```text
n
```

Where:

- `n` is the target signal level.

## Output Format

Print one integer: the number of valid construction sequences.

## Constraints

```text
1 <= n <= 35
```

## Examples

### Example 1

Input:

```text
3
```

Output:

### Example 2

### Example 3

Input:

```text
2
```

Output:

```text

```

## Edge-Case Thinking Prompts

- What should happen when the current level becomes greater than `n`?
- Why is reaching exactly `n` different from crossing `n`?
- Why do we need to remember how the sequence ends?
- Are two consecutive `+1` increases always forbidden, or only at the end?
- Why is this a counting recursion problem?
