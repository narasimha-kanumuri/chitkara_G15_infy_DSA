## Plain-English Implementation Walkthrough

1. We store stone heights in a `long[]` array because height differences can be large.

2. We use a recursive method named `solve`.
   It means: “minimum energy needed from the current stone to the last stone.”

3. The parameter `i` stores the current stone index.
   This is needed because future jumps depend on where the frog is standing.

4. The parameter `prevTwo` stores whether the previous jump was length `2`.
   This is needed because the fatigue penalty depends on the previous jump.

5. If `i == height.length - 1`, the frog has reached the last stone.
   So we return `0` because no more jumps are needed.

6. We check `seen[i][prevTwo]` before solving the state.
   If it is already solved, we return the saved value from `memo`.

7. We set `best` to a very large value first.
   This lets us safely take the minimum among valid jump options.

8. If `i + 1` is inside the array, we try a `1-step` jump.
   This jump cost is the height difference between stone `i` and stone `i + 1`.
   After this jump, `prevTwo` becomes `0`.

9. If `i + 2` is inside the array, we try a `2-step` jump.
   This jump cost is the height difference between stone `i` and stone `i + 2`.

10. If `prevTwo == 1`, we add the penalty to the `2-step` jump.
    This is exactly the consecutive `2-step` rule.

11. We save the cheapest value in `memo[i][prevTwo]`.
    Then we return it.

12. The final answer starts from `solve(0, 0, ...)`.
    There is no previous jump at the start, so `prevTwo` is `0`.

## Complete Solution Code

```java
import java.io.BufferedReader;
import java.io.InputStreamReader;
import java.util.StringTokenizer;

public class Main {

    private static final long INF = Long.MAX_VALUE / 4;

    private static long solve(int i, int prevTwo, long[] height, long penalty,
                              long[][] memo, boolean[][] seen) {
        if (i == height.length - 1) {
            return 0L;
        }

        if (seen[i][prevTwo]) {
            return memo[i][prevTwo];
        }

        long best = INF;

        if (i + 1 < height.length) {
            long jumpCost = Math.abs(height[i] - height[i + 1]);
            long total = jumpCost + solve(i + 1, 0, height, penalty, memo, seen);
            best = Math.min(best, total);
        }

        if (i + 2 < height.length) {
            long jumpCost = Math.abs(height[i] - height[i + 2]);
            if (prevTwo == 1) {
                jumpCost += penalty;
            }
            long total = jumpCost + solve(i + 2, 1, height, penalty, memo, seen);
            best = Math.min(best, total);
        }

        seen[i][prevTwo] = true;
        memo[i][prevTwo] = best;
        return best;
    }

    private static long minimumEnergy(long[] height, long penalty) {
        int n = height.length;
        long[][] memo = new long[n][2];
        boolean[][] seen = new boolean[n][2];
        return solve(0, 0, height, penalty, memo, seen);
    }

    public static void main(String[] args) throws Exception {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));

        StringTokenizer first = new StringTokenizer(br.readLine());
        int n = Integer.parseInt(first.nextToken());
        long penalty = Long.parseLong(first.nextToken());

        long[] height = new long[n];
        StringTokenizer second = new StringTokenizer(br.readLine());
        for (int i = 0; i < n; i++) {
            height[i] = Long.parseLong(second.nextToken());
        }

        System.out.println(minimumEnergy(height, penalty));
    }
}
```

## Code Deconstruction

### `INF`

```java
private static final long INF = Long.MAX_VALUE / 4;
```

This is a very large value.
It is used as the starting value for minimum comparison.
Dividing by `4` reduces overflow risk when adding jump costs.

### `solve` method

```java
private static long solve(int i, int prevTwo, long[] height, long penalty,
                          long[][] memo, boolean[][] seen)
```

This method represents one DP state.

- `i` is the current stone.
- `prevTwo` stores whether the previous jump was `2`.
- `height` stores all stone heights.
- `penalty` stores fatigue cost.
- `memo` stores answers.
- `seen` marks solved states.

### Target base case

```java
if (i == height.length - 1) {
    return 0L;
}
```

The frog is already on the last stone.
No more energy is needed.

### Memoization check

```java
if (seen[i][prevTwo]) {
    return memo[i][prevTwo];
}
```

This prevents repeated solving of the same state.

### One-step jump

```java
long total = jumpCost + solve(i + 1, 0, height, penalty, memo, seen);
```

A `1-step` jump makes the new previous jump not equal to `2`.
So the next state uses `prevTwo = 0`.

### Two-step jump

```java
if (prevTwo == 1) {
    jumpCost += penalty;
}
```

This adds fatigue only when the previous jump was also `2`.
That is the main constraint.

### Saving result

```java
seen[i][prevTwo] = true;
memo[i][prevTwo] = best;
return best;
```

The best answer for this state is saved before returning.

## Language-Specific Notes

- Use `long` for height values, penalty, and energy totals.
- `Math.abs(height[i] - height[j])` is safe here because the values are stored in `long`.
- A `seen` table is used because `0` can be a valid answer.
- The recursion depth can reach `n`; this matches the memoization teaching goal.
- `prevTwo` is represented as `0` or `1` so it can directly index the memo table.

## Complexity Summary

- Time Complexity: `O(n)`
- Space Complexity: `O(n)`
- Auxiliary Space: `O(n)`

There are at most `2 * n` states.
Each state tries at most two jumps.
