## Idea (Simple)
We have a circular gallery with `n` exhibits.
Each exhibit has a value.
We must select exhibits so that no two selected exhibits are adjacent.

The twist is circular adjacency:
- exhibit `1` is adjacent to exhibit `2`
- exhibit `n` is adjacent to exhibit `n - 1`
- exhibit `1` is also adjacent to exhibit `n`

So we cannot solve it as only one straight line.
We split the problem into two safe linear cases:
- exclude the first exhibit
- exclude the last exhibit

The best of these two cases is the answer.

## Pattern Recognition Clues
- The problem asks for a maximum total value, so this is optimization DP.
- Each exhibit creates a take-or-skip decision.
- Taking one exhibit blocks its adjacent exhibits.
- The circular rule means the first and last exhibit conflict, so we need case splitting.

## State Discovery
For a straight row, the first idea is `dp[i]`: maximum value using exhibits from `1` to `i`.
This works for a straight row because the only conflict is with the previous exhibit.

But the gallery is circular.
`dp[i]` alone does not remember whether exhibit `1` was selected.
That missing information matters at the last exhibit, because exhibit `n` conflicts with exhibit `1`.

Instead of adding a larger state, we split the problem:
- Case 1: solve exhibits `1` to `n - 1`
- Case 2: solve exhibits `2` to `n`

Each case is now a normal straight-line take/skip DP.

## Pattern
Take / Skip DP with Circular Case Split

## Core Insight
- In any valid answer, we cannot take both the first and last exhibits.
- So every valid answer belongs to at least one of these cases: exclude first or exclude last.
- Each case becomes the classic linear non-adjacent maximum problem.

## Invariant
During the linear helper computation:
- `prevOne` stores the best answer up to the previous exhibit.
- `prevTwo` stores the best answer up to the exhibit before the previous one.

After each step, the best value respects the no-adjacent-selection rule.

## Algorithm (with one-line why at important steps)
1. If `n = 1`, return the only exhibit value.
   - Why: there are no adjacent pairs to compare.
2. If `n = 2`, return the maximum of the two values.
   - Why: both exhibits are adjacent in the circle.
3. Compute the best value from index `0` to `n - 2`.
   - Why: this excludes the last exhibit.
4. Compute the best value from index `1` to `n - 1`.
   - Why: this excludes the first exhibit.
5. In each linear range, for every exhibit, choose take or skip.
   - Why: taking current uses `prevTwo + value`, skipping uses `prevOne`.
6. Return the maximum of the two case answers.
   - Why: all valid circular selections are covered by these two safe cases.

## Mental Cheat Sheet
| Item | Meaning |
|---|---|
| Pattern | Take / Skip DP |
| Twist | first and last exhibits are adjacent |
| Case 1 | allow first side, exclude last exhibit |
| Case 2 | exclude first exhibit, allow last side |
| Linear transition | `current = max(skip, take)` |
| Answer | `max(case1, case2)` |

## Complexity
- Time Complexity: `O(n)`
- Space Complexity: `O(1)`
- Auxiliary Space: `O(1)`

## Tiny Dry Run
For `values = [6, 2, 9, 4, 5]`:

Case 1: use exhibits `1` to `4` → `[6, 2, 9, 4]`
- best non-adjacent value = `15` from `6 + 9`

Case 2: use exhibits `2` to `5` → `[2, 9, 4, 5]`
- best non-adjacent value = `14` from `9 + 5`

Answer = `max(15, 14) = 15`.

## Notes / Pitfalls
- Do not run normal house-robber DP on the full array directly.
- The first and last exhibits are adjacent because the gallery is circular.
- Handle `n = 1` separately.
- The helper solves only a straight range, not the circular problem directly.
