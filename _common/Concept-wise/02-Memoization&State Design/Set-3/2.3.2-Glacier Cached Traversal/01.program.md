# Glacier Cached Traversal

## Concepts Involved

Memoization, State Design, Constraint-aware DP, Optimization DP, Recursion to DP

## Problem Statement

A climber has to cross `n` icy platforms.

The height of platform `i` is given as `height[i]`.
Some platforms are blue. Landing on a blue platform adds an extra cold cost `c`.

The climber starts on platform `0` and wants to reach platform `n - 1`.
The starting platform is already occupied, so its landing cold cost is not added at the beginning.

From platform `i`, the climber may jump to:

- platform `i + 1`
- platform `i + 2`

The energy cost of a jump from platform `i` to platform `j` is:

```text
abs(height[i] - height[j])
```

If the climber lands on a blue platform, add `c` to the jump cost.

There is one more restriction:

- Two `2-step` jumps cannot be taken consecutively.

A plain recursive solution repeatedly evaluates the same platform states.
Use memoization to avoid recomputation.

Return the minimum total energy required to reach the final platform.
If the final platform cannot be reached, print `-1`.

## Input Format

- The first line contains two integers `n` and `c`.
- The second line contains `n` integers, where `height[i]` is the height of platform `i`.
- The third line contains a binary string of length `n`.
  - `B` means the platform is blue.
  - `N` means the platform is normal.

## Output Format

- Print the minimum total energy required to reach platform `n - 1`.
- Print `-1` if no valid route exists.

## Constraints

- `1 <= n <= 2000`
- `0 <= c <= 10^9`
- `0 <= height[i] <= 10^9`
- The answer fits in a signed 64-bit integer.

## Examples

### Example 1

Input:

```text
5 7
10 20 15 30 25
NNBNN
```

Output:

```text
25
```

Explanation:

The route below is valid:

```text
0 -> 1 -> 3 -> 4
```

Costs:

```text
0 -> 1 = abs(10 - 20) = 10
1 -> 3 = abs(20 - 30) = 10
3 -> 4 = abs(30 - 25) = 5
```

Total cost is:

```text
10 + 10 + 5 = 25
```

The route `0 -> 2 -> 4` looks cheaper by height difference, but it uses two consecutive `2-step` jumps, so it is invalid.

### Example 2

Input:

```text
4 5
3 8 2 6
NBNN
```

Output:

```text
5
```

Explanation:

Route:

```text
0 -> 2 -> 3
```

Cost:

```text
abs(3 - 2) + abs(2 - 6) = 1 + 4 = 5
```

The route `0 -> 1 -> 3` costs:

```text
abs(3 - 8) + 5 + abs(8 - 6) = 5 + 5 + 2 = 12
```

So the minimum valid total is `5`.

### Example 3

Input:

```text
1 10
9
B
```

Output:

```text
0
```

Explanation:

The climber is already on the final platform.
No landing cost is added for the starting platform.

- When does landing cold cost get added?
- What value should represent an impossible path?
