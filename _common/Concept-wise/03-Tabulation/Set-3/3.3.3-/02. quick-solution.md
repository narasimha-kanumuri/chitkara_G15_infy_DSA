## Idea (Simple)
We need to count binary signals of length `n`.
A `1` cannot appear next to another `1`.
Also, restricted positions cannot contain `1`.
So we track how many valid signals end with `0` and how many end with `1` at each position.

## Pattern Recognition Clues
- We are counting all valid binary sequences.
- The next choice depends on the previous bit.
- Some positions block one possible choice.
- This suggests bottom-up DP with last-bit state.

## State Discovery
The first idea is `dp[i]`: number of valid signals up to position `i`.
But `dp[i]` is not enough.
It does not tell whether the previous position ended with `0` or `1`.
That matters because a new `1` can be placed only after `0`.
So we split the state:
- `endWithZero[i]`: valid signals of length `i` ending with `0`
- `endWithOne[i]`: valid signals of length `i` ending with `1`

## Pattern
Bottom-Up Counting DP with Last Bit State

## Core Insight
- `0` can be placed at any position, restricted or not.
- `1` can be placed only if the position is not restricted.
- `1` can follow only a previous `0`.

## Invariant
For every position `i`:
- `endWithZero[i]` stores all valid prefixes of length `i` ending with `0`.
- `endWithOne[i]` stores all valid prefixes of length `i` ending with `1`.
No counted prefix violates the restricted-position rule or the no-consecutive-`1`s rule.

## Algorithm (with one-line why at important steps)
1. Mark all restricted positions in a boolean array.
   - Why: we need fast checking before placing `1`.
2. Create arrays `endWithZero` and `endWithOne`.
   - Why: the last bit affects the next valid choice.
3. Initialize position `1`.
   - Why: it is the smallest complete signal prefix.
4. Always allow `endWithZero[i] = endWithZero[i - 1] + endWithOne[i - 1]`.
   - Why: `0` can follow both `0` and `1`.
5. Allow `endWithOne[i] = endWithZero[i - 1]` only when position `i` is not restricted.
   - Why: `1` cannot be placed on restricted positions and cannot follow `1`.
6. Return `endWithZero[n] + endWithOne[n]`.
   - Why: a valid final signal may end with either bit.

## Mental Cheat Sheet
| Item | Meaning |
|---|---|
| Pattern | Counting DP with last-bit state |
| Simple failed state | `dp[i]` forgets the previous bit |
| Restricted position | forced to use `0` |
| Transition for `0` | can follow both previous states |
| Transition for `1` | can follow only previous `0`, and only if unrestricted |
| Answer | `endWithZero[n] + endWithOne[n]` |

## Complexity
- Time Complexity: `O(n)`
- Space Complexity: `O(n)`
- Auxiliary Space: `O(n)`

## Tiny Dry Run
For `n = 4`, restricted position = `2`:
- Position `1`: `0` and `1` are possible.
- Position `2`: only `0` is possible.
- Positions `3` and `4`: normal no-consecutive-`1` rule applies.
- Total valid signals = `6`.

## Notes / Pitfalls
- Do not place `1` on a restricted position.
- Do not allow `1` after another `1`.
- Do not return only one ending state.
- Use `BigInteger` because exact counts can grow large.
