## Plain-English Implementation Walkthrough
1. We need one method that answers one question: “What is the brightness for this day?”
   Related code: `brightness(int day)`

2. If the day is `0`, we return `1`.
   This is a fixed starting value, so no more recursion is needed.

3. If the day is `1`, we return `2`.
   This is the second fixed starting value.

4. For any other day, we need the previous day and the day before that.
   So the method calls itself for `day - 1` and `day - 2`.

5. We multiply the second previous value by `2` because the recurrence gives it double weight.
   Related code: `previous + 2 * secondPrevious`

6. We store the combined value in `value`.
   This makes it easy to apply the penalty before returning.

7. If the current day is divisible by `4`, we subtract `1` from `value`.
   This matches the mirror energy loss rule.

8. We return `value` because it now stores the final brightness for the current day.

## Complete Solution Code
```java
import java.util.Scanner;

public class Main {
    static long brightness(int day) {
        if (day == 0) {
            return 1;
        }

        if (day == 1) {
            return 2;
        }

        long previous = brightness(day - 1);
        long secondPrevious = brightness(day - 2);

        long value = previous + 2 * secondPrevious;

        if (day % 4 == 0) {
            value -= 1;
        }

        return value;
    }

    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        int n = sc.nextInt();
        System.out.println(brightness(n));
        sc.close();
    }
}
```

## Code Deconstruction
- `brightness(int day)` is the recursive function.
  - The parameter `day` is the state.
  - It tells which day we are solving now.

- The first two `if` blocks are base cases.
  - They stop recursion at day `0` and day `1`.

- `previous` stores `brightness(day - 1)`.
  - This is the value from the previous day.

- `secondPrevious` stores `brightness(day - 2)`.
  - This is the value from two days before.

- `value = previous + 2 * secondPrevious` applies the recurrence.
  - The second previous day has double effect.

- `if (day % 4 == 0)` applies the periodic penalty.
  - It affects only the current day.

- `return value` sends the final brightness back to the caller.

## Language-Specific Notes
- `long` is used instead of `int` to reduce overflow risk.
- This is a plain recursive solution, so it recomputes many values.
- The constraint is small because this is a Day 1 recursion problem.
- In a later memoization version, we can store `brightness(day)` after computing it once.

## Complexity Summary
- Time Complexity: `O(2^n)` worst case.
- Space Complexity: `O(n)` recursion stack.
- Auxiliary Space: `O(n)` recursion stack.
