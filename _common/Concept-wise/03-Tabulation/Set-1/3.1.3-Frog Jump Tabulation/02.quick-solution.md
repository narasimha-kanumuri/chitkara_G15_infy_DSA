# quick-solution.md

## Idea (Simple)

- This is a minimum-energy frog jump problem.
- The frog can jump 1 stone or 2 stones. The twist is that two consecutive 2-stone jumps add a fatigue penalty.
- So we must remember:

- which stone the frog reached
- whether the last jump was 1 or 2

## Pattern

- Bottom-Up DP with last-jump state

## Core Insight

- A 1-jump can follow any previous jump.
- A 2-jump can also follow any previous jump, but if it follows another 2-jump, add penalty p.
- Therefore, one DP value per stone is not enough.

## Invariant

- At every stone i:

- endWithOne[i] = minimum energy to reach stone i with last jump length 1
- endWithTwo[i] = minimum energy to reach stone i with last jump length 2
- These two values preserve exactly the history needed to decide whether fatigue applies.

## Algorithm (with one-line why at important steps)

- Create arrays endWithOne and endWithTwo.
- **Why:** fatigue depends on the previous jump length.
- Initialize both states at stone 1 as 0.
- **Why:** the frog starts there before making any jump.
- To reach stone i using a 1-jump, come from stone i - 1.
- **Why:** only one previous position is possible for a 1-jump.
- Use the cheaper of endWithOne[i - 1] and endWithTwo[i - 1].
- **Why:** a 1-jump does not create fatigue.
- To reach stone i using a 2-jump, come from stone i - 2.
- **Why:** that is the only previous position for a 2-jump.
- If the previous state also ended with 2, add penalty p.
- **Why:** this forms two consecutive long jumps.
- Answer is min(endWithOne[n], endWithTwo[n]).
- **Why:** the final jump may be either length.

## Code Cheat Sheet

| Code / Built-in | Meaning / Role |
| :--- | :--- |
| endWithOne[i] | best energy reaching i with last jump 1 |
| endWithTwo[i] | best energy reaching i with last jump 2 |
| Math.abs(a - b) | energy cost between two heights |
| Math.min(a, b) | choose cheaper transition |
| INF | marks unreachable / not-yet-computed states |

## Complexity

- **Time:**
- `O(n)`
- **Space:**
- `O(n)`
- **Auxiliary Space:**
- `O(n)`

## Tiny Dry Run

- For:
- height = [1, 100, 2, 100, 3], p = 10

- Jump 1 -> 3: cost |1 - 2| = 1
- Jump 3 -> 5: cost |2 - 3| = 1, plus penalty 10
- Total = 12
- This path uses two consecutive 2-jumps, so fatigue is counted.

## Notes / Pitfalls

- Do not use plain frog-jump DP without last-jump state.
- The first 2-jump should not get penalty.
- Use long, not int, for energy values.
- For n = 1, answer is 0.
