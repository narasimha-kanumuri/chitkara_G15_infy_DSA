## Title
Locked Stair Tabulation

## Concepts Involved
Tabulation, Constraint Handling, Counting DP

## Problem Statement
A student wants to climb a staircase with `n` numbered steps.

The student starts on the ground at position `0` and wants to reach exactly step `n`.
From any position, the student may usually move:
- `1` step forward, or
- `2` steps forward.

Some steps are locked and cannot be landed on.
If a step is locked, that step cannot be part of any valid path.

There is one extra rule:
- If the student is currently standing on a prime-numbered step, the next move must be exactly `1` step.

Step `0` is the ground. It is not considered prime.
The destination step `n` must also be unlocked for the answer to be non-zero.

Count the total number of valid ways to reach exactly step `n`.
Print the exact count.

## Input Format
- First line: two integers `n` and `m`
  - `n` = target step
  - `m` = number of locked steps
- Second line: `m` space-separated integers representing the locked step numbers

If `m = 0`, the second line may be empty or absent.

## Output Format
- Print the total number of valid ways to reach exactly step `n`

## Constraints
- `1 <= n <= 5000`
- `0 <= m <= n`
- `1 <= lockedStep <= n`
- Locked step numbers are distinct

## Examples
### Example 1
Input:
`4 0`

Output:
`3`

### Example 2
Input:
`6 1`
`5`

Output:
`3`

### Example 3
Input:
`5 1`
`3`

Output:
`0`

## Edge-Case Thinking Prompts
- What if the target step itself is locked?
- What if step `2` is prime and forces the next move to be `1`?
- Is step `1` prime?
- Can the student jump over locked steps?
- Should the ground position `0` follow the prime rule?
