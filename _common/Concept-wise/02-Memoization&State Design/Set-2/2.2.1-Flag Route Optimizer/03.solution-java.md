## Plain-English Implementation Walkthrough

1. We use `BigInteger` because the number of valid routes can grow very large.

2. We use the method `solve(step, lastJump, n, memo)` to represent one DP state.
   It means: ‚Äúhow many valid routes can finish from this step?‚Äù

3. The parameter `step` tells the current position of the volunteer.
   It changes after every jump.

4. The parameter `lastJump` stores the jump size used just before this state.
   This is needed because the next jump cannot be the same size.

5. The value `0` is used as the starting `lastJump`.
   It means no real jump has happened yet.

6. If `step == n`, we return `BigInteger.ONE`.
   This means the current route reached the target exactly.

7. If `step > n`, we return `BigInteger.ZERO`.
   This route crossed the target, so it is invalid.

8. If the memo table already has this answer, we return it.
   This is the memoization step.

9. We try jump sizes from `1` to `3` using a loop.
   This avoids writing the same logic three times.

10. We skip a jump when `jump == lastJump`.
    This line enforces the no-consecutive-equal-jump rule.

11. For every allowed jump, we move to `step + jump` and set the new `lastJump` to `jump`.
    This keeps the state correct for the next decision.

12. We add all valid route counts, save the total, and return it.
    This returns the total number of valid routes from the current state.

## Complete Solution Code

```java
import java.io.BufferedReader;
import java.io.InputStreamReader;
import java.math.BigInteger;

public class Main {

    private static BigInteger solve(int step, int lastJump, int n, BigInteger[][] memo) {
        if (step == n) {
            return BigInteger.ONE;
        }

        if (step > n) {
            return BigInteger.ZERO;
        }

        if (memo[step][lastJump] != null) {
            return memo[step][lastJump];
        }

        BigInteger total = BigInteger.ZERO;

        for (int jump = 1; jump <= 3; jump++) {
            if (jump == lastJump) {
                continue;
            }

            total = total.add(solve(step + jump, jump, n, memo));
        }

        memo[step][lastJump] = total;
        return total;
    }

    private static BigInteger countRoutes(int n) {
        BigInteger[][] memo = new BigInteger[n + 1][4];
        return solve(0, 0, n, memo);
    }

    public static void main(String[] args) throws Exception {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        int n = Integer.parseInt(br.readLine().trim());
        System.out.println(countRoutes(n));
    }
}
```

## Code Deconstruction

### `solve` method

```java
private static BigInteger solve(int step, int lastJump, int n, BigInteger[][] memo)
```

This method stores one memoized recursion state.

- `step` is the current staircase position.
- `lastJump` is the previous jump size.
- `n` is the target step.
- `memo` stores already computed answers.

### Exact target base case

```java
if (step == n) {
    return BigInteger.ONE;
}
```

This counts one complete valid route.

### Crossed target base case

```java
if (step > n) {
    return BigInteger.ZERO;
}
```

This rejects routes that move beyond the target step.

### Memoization check

```java
if (memo[step][lastJump] != null) {
    return memo[step][lastJump];
}
```

If the same state appears again, the saved answer is reused.

### Trying all jump sizes

```java
for (int jump = 1; jump <= 3; jump++) {
```

This loop tries the three allowed jump sizes.

### Blocking repeated jump size

```java
if (jump == lastJump) {
    continue;
}
```

This is the main rule of the problem.
It prevents two consecutive jumps from having the same size.

### Recursive transition

```java

```

After choosing a jump, the volunteer moves forward.
The chosen jump becomes the new `lastJump`.

## Language-Specific Notes

- `BigInteger` values are immutable. Use `.add()` to add values.
- Do not use `+` to add `BigInteger` objects.
- `memo[step][lastJump] == null` means the state has not been computed.
- The memo table has `4` columns because `lastJump` can be `0`, `1`, `2`, or `3`.
- Recursion depth can reach `n`; this is acceptable for teaching memoization, but tabulation can avoid stack usage.

## Complexity Summary

- Time Complexity: `O(n)`
- Space Complexity: `O(n)`
- Auxiliary Space: `O(n)`

There are at most `4 * (n + 1)` states.
Each state tries exactly three jump choices.
