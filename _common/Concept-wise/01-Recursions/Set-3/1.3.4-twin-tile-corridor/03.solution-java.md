# Twin Tile Corridor — Java Solution

## Language

Java

## Code Concepts Used

| Java Built-in / Concept | Meaning / Role | Very Simple Definition | Why Used in This Program | How to Use It |
|---|---|---|---|---|
| `Scanner` | Reads input | A simple input reader | To read the corridor length `n` | `Scanner sc = new Scanner(System.in);` |
| Recursive method | Solves smaller remaining lengths | A method that calls itself | To try tile choices and solve the rest of the corridor | `solve(rem - 1, false)` |
| `boolean` | Stores true or false | A yes/no value | To remember whether the previous tile was length `2` | `boolean prevWasTwo` |

## Plain-English Implementation Walkthrough

1. We read `n` because the whole problem depends only on corridor length.

2. We start with `solve(n, false)`.  
   No tile has been placed yet, so the previous tile is not length `2`.

3. The recursive method needs `rem`.  
   This tells how much corridor is still uncovered.

4. The method also needs `prevWasTwo`.  
   This tells whether placing another length-`2` tile would break the adjacency rule.

5. If `rem == 0`, we return `1`.  
   This means the corridor has been covered exactly in one valid way.

6. We always try placing a length-`1` tile if `rem` is positive.  
   After this choice, the previous tile for the next call is not length `2`.

7. We try placing a length-`2` tile only when the previous tile was not length `2`.  
   This protects the no-adjacent-`2` rule.

8. We also require `rem >= 3` before placing a length-`2` tile.  
   If `rem == 2`, that tile would become the final tile, which is not allowed.

9. We add the results of valid choices.  
   The problem asks for the number of tilings, not the best tiling.

## Complete Solution Code

```java
import java.util.*;

public class Main {
    static long solve(int rem, boolean prevWasTwo) {
        if (rem == 0) {
            return 1;
        }

        long ways = 0;

        // Choice 1: place a tile of length 1.
        if (rem >= 1) {
            ways += solve(rem - 1, false);
        }

        // Choice 2: place a tile of length 2 if it does not break any rule.
        if (!prevWasTwo && rem >= 3) {
            ways += solve(rem - 2, true);
        }

        return ways;
    }

    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);

        int n = sc.nextInt();

        long answer = solve(n, false);
        System.out.println(answer);
    }
}
```

## Code Deconstruction

### `solve(rem, prevWasTwo)`

This method returns the number of valid ways to cover the remaining corridor length.

- `rem` tells how much length is still uncovered.
- `prevWasTwo` tells whether the last placed tile was length `2`.

### Base Case

```java
if (rem == 0) {
    return 1;
}
```

When no length remains, one valid tiling has been completed.

### Placing Length `1`

```java
ways += solve(rem - 1, false);
```

A length-`1` tile is always safe.  
After placing it, the previous tile is not length `2`.

### Placing Length `2`

```java
if (!prevWasTwo && rem >= 3) {
    ways += solve(rem - 2, true);
}
```

This condition enforces both important rules:

- `!prevWasTwo` prevents adjacent length-`2` tiles.
- `rem >= 3` prevents a length-`2` tile from becoming the final tile.

### Returning `ways`

```java
return ways;
```

`ways` contains the total count from all valid branches.

## Language-Specific Notes

- `long` is used because the answer can grow quickly.
- Java recursion uses the call stack, so very large `n` can cause stack issues.
- Pure recursion repeats many subproblems. This is intentional for Day 1.
- On Day 2, this exact state can be memoized as `dp[rem][prevWasTwo]`.

## Complexity Summary

- Time Complexity: `O(2^n)` for pure recursion.
- Space Complexity: `O(n)` due to recursion stack.
- Auxiliary Space: `O(n)` due to recursion stack.
