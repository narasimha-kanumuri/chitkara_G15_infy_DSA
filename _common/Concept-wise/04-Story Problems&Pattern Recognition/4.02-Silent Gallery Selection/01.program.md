## Title
Silent Gallery Selection

## Concepts Involved
Dynamic Programming, Take / Skip Pattern, Non-Adjacent Optimization

## Problem Statement
A curator is preparing a line of gallery rooms for a one-day exhibition.
Opening room `i` gives an artistic impact score `impact[i]`.

Two adjacent rooms cannot be opened together because the sound systems interfere.
Also, some rooms may have negative impact scores, meaning opening them hurts the overall exhibition quality.

Find the maximum total impact the curator can achieve.
The curator is allowed to open **no rooms at all** if that gives a better score.

## Input Format
- Integer `n` — number of rooms
- Array `impact[]` of length `n`

## Output Format
- Return the maximum total impact

## Constraints
- `1 <= n <= 2 * 10^5`
- `-10^9 <= impact[i] <= 10^9`

## Examples

### Example 1
Input:
`impact = [4, 2, 7, 3, 9]`

Output:
`20`

Explanation:
Open rooms with impact `4, 7, 9`.
These rooms are non-adjacent, so the selection is valid.
Total impact = `20`.

### Example 2
Input:
`impact = [5, -2, 1, 6]`

Output:
`11`

Explanation:
Open rooms with impact `5` and `6`.
Skipping the negative room is beneficial.

### Example 3
Input:
`impact = [-8, -3, -10]`

Output:
`0`

Explanation:
Opening no room is better than taking any negative total.

## Edge-Case Thinking Prompts
- What if all rooms have negative impact?
- What if there is only one room?
- What if the best answer comes from skipping several positive-looking rooms to take a better later combination?
- Why is choosing room `i` directly connected to the answer at `i - 2`?

## Thinking Like a Programmer
- At each room, should we open it or skip it?
- If we open the current room, which earlier room is the latest one we are still allowed to combine with?
- Can we define one DP meaning that always stores the best answer up to a position?
- How should we handle the possibility of choosing no room at all?
