## Title
Silent DP Builder

## Concepts Involved
Tabulation, State Compression, Counting DP

## Problem Statement
A monitoring system builds a silent binary signal of length `n`.

Each position in the signal may contain either `0` or `1`.
However, the signal must follow two rules:

1. It must not contain two consecutive `1`s.
2. Some positions are restricted. A restricted position cannot contain `1`.

Positions are numbered from `1` to `n`.
A signal is valid if every position is filled, no adjacent positions both contain `1`, and all restricted positions contain `0`.

Count the total number of valid signals of length `n`.
Print the exact count.

## Input Format
- First line: two integers `n` and `m`
  - `n` = length of the signal
  - `m` = number of restricted positions
- Second line: `m` space-separated integers representing restricted positions

If `m = 0`, the second line may be empty or absent.

## Output Format
- Print the total number of valid binary signals

## Constraints
- `1 <= n <= 5000`
- `0 <= m <= n`
- `1 <= restrictedPosition <= n`
- Restricted positions are distinct

## Examples
### Example 1
Input:
`3 0`

Output:
`5`

### Example 2
Input:
`4 1`
`2`

Output:
`6`

### Example 3
Input:
`5 2`
`2 4`

Output:
`8`

## Edge-Case Thinking Prompts
- What if there are no restricted positions?
- What if every position is restricted?
- What if position `1` is restricted?
- What if two unrestricted positions are adjacent?
- Why do we need to remember the previous bit?

## Thinking Like a Programmer
- What choices are available at every position?
- When is placing `1` not allowed?
- What does the DP state need to remember?
- How can we count strings ending with `0` separately from strings ending with `1`?
- Where should the final answer be collected?
