# Mirror Beacon

## Concepts Involved
- Base Case Design
- Return Meaning
- Choice Expansion
- Recursive State Definition
- Recursion Tree Analysis
- Constraint Propagation
- Counting / Value Recursion
- Optimization Awareness

## Problem Statement
A mirror-powered beacon changes its brightness every day.

The brightness of day `i` depends on brightness from earlier days:

- Day `0` brightness is `1`.
- Day `1` brightness is `2`.
- For every day `i >= 2`, first compute:

```text
brightness(i) = brightness(i - 1) + 2 * brightness(i - 2)
```

However, the mirror loses energy on every day whose index is a multiple of `4`.

So, if `i % 4 == 0`, subtract `1` from the computed brightness for that day.

Given `n`, return the final brightness value on day `n` using recursion.

## Input Format
- A single integer `n`.

## Output Format
- Print one integer: the brightness value on day `n`.

## Constraints
- `0 <= n <= 35`
- The answer fits in a 64-bit signed integer.

## Examples

### Example 1
**Input**
```text
0
```

**Output**
```text
1
```

**Explanation**
Day `0` is a base day, so its brightness is directly `1`.

### Example 2
**Input**
```text
4
```

**Output**
```text
15
```

**Explanation**
Brightness values are:

```text
day 0 = 1
day 1 = 2
day 2 = 2 + 2 * 1 = 4
day 3 = 4 + 2 * 2 = 8
day 4 = 8 + 2 * 4 = 16, then subtract 1 because 4 is a multiple of 4
```

So day `4` brightness is `15`.

### Example 3
**Input**
```text
6
```

**Output**
```text
61
```

**Explanation**
After day `4` becomes `15`, day `5` becomes `31`, and day `6` becomes `61`.

## Edge-Case Thinking Prompts
- What should the function return for day `0`?
- What should the function return for day `1`?
- When should the multiple-of-4 penalty be applied?
- Should the penalty be applied before or after combining previous results?
- Which recursive calls are repeated many times?

## Thinking Like a Programmer
Before writing code, define:

```text
State: current day number
Choice / expansion: solve day - 1 and day - 2
Transition: brightness(day - 1) + 2 * brightness(day - 2)
Base case: day 0 and day 1
Order: recursion shrinks day toward base cases
Answer: brightness(n)
```
