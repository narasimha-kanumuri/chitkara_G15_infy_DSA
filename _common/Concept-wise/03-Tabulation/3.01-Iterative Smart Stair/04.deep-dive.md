## deep-dive.md

### What Are We Really Solving?

We are not just climbing steps.

We are counting **valid sequences of moves** under a constraint:

- Moves allowed = `{1, 2}`

- Constraint = no two consecutive `2`-step moves

So the real problem is:

→ Count sequences of 1s and 2s whose total sum is `n`,

→ while **forbidding the pattern (2, 2)**.

This is a **state-dependent counting problem**, not a plain combinatorics problem.

---

### Brute Force Lens

Generate all sequences of 1 and 2 whose sum is `n`.

Example for `n = 4`:

- 1,1,1,1 ✅

- 1,1,2 ✅

- 1,2,1 ✅

- 2,1,1 ✅

- 2,2 ❌ (invalid)

Brute force:

- Generate all sequences (exponential)

- Filter invalid ones

Time complexity:

≈ `O(2^n)`

Why it fails:

- Too slow

- Does not reuse repeated sub-structures

---

### Key Observations

- The constraint depends on **what happened just before**.

- Whether we can place `2` depends on the **previous move**.

- So **step number alone is not enough state**.

We need:

→ `(current step, last move used)`

This is the key shift from simple DP → **state-dependent DP**.

---

### Mathematical Derivation

Let:

- `A[i]` = ways to reach step `i` ending with `1`

- `B[i]` = ways to reach step `i` ending with `2`

Now derive transitions.

#### Case 1: Ending with 1

If the last move is `1`:

- Previous step was `i - 1`

- We can come from:

- A[i-1] (ended with 1)

- B[i-1] (ended with 2)

So:

A[i] = A[i-1] + B[i-1]

```

#### Case 2: Ending with 2
If the last move is `2`:
- Previous step was `i - 2`
- But cannot come from B[i-2] (that would be 2,2)

So:
```

B[i] = A[i-2]

```

#### Final Answer
```

Answer = A[n] + B[n]

```

Meaning:
→ All valid sequences must end with either 1 or 2.

---

### Core Invariant
At every step `i`:

- `A[i]` stores **all valid sequences ending with 1**
- `B[i]` stores **all valid sequences ending with 2 and obeying the constraint**

Invariant guarantee:
→ No invalid `(2,2)` sequence is ever counted

Why?
- Because `B[i]` is built **only from `A[i-2]`**, never from `B[i-2]`

---

### Solution Construction
We build the solution bottom-up:

1. Initialize base cases:
   - `A[1] = 1` → only (1)
   - `B[2] = 1` → only (2)

2. Fill DP step by step

3. Maintain invariant automatically via transitions

4. Sum both states at the end

---

### Implementation Strategy
- Use two arrays or two variables:
  - `endWithOne`
  - `endWithTwo`

- Fill from `1 → n`

- Important details:
  - Index carefully (i-1, i-2)
  - Initialize base states correctly
  - Avoid invalid propagation

Optional optimization:
→ Use rolling variables instead of arrays (space O(1))

---

### English → Code Mapping

| English Idea | Code Representation |
|---|---|
| ways ending with 1 | `endWithOne[i]` |
| ways ending with 2 | `endWithTwo[i]` |
| total ways at i | `endWithOne[i] + endWithTwo[i]` |
| transition from previous step | `i - 1`, `i - 2` |
| forbid 2 → 2 | only use `endWithOne[i-2]` |

Core mapping:
```

endWithOne[i] = endWithOne[i-1] + endWithTwo[i-1] endWithTwo[i] = endWithOne[i-2]

```

---

### Correctness Reasoning

We prove correctness using the invariant.

#### Completeness
- Every valid sequence is counted.
- Because:
  - Any sequence must end in either 1 or 2.
  - Both cases are covered.

#### No Overcounting
- No sequence is counted twice:
  - Disjoint split based on last move.

#### Constraint Enforcement
- Invalid sequences `(2,2)` never enter DP:
  - Because `B[i]` never comes from `B[i-2]`

Hence:
→ Only valid sequences propagated
→ All valid sequences covered

---

### Deep Complexity Analysis

#### Time Complexity
- Each state computed once → `O(n)`

#### Space Complexity
- Array-based: `O(n)`
- Optimized: `O(1)` using two rolling pairs

#### Growth Behavior
The recurrence resembles Fibonacci-like growth but is slightly restricted.

Approx relation:
```

Total[i] ≈ Total[i-1] + Total[i-2] - (invalid sequences removed)

```

So:
→ Growth is exponential but slower than Fibonacci

---

### Variants and Extensions

1. **Forbid k consecutive 2-steps**
   - Add state counter: `(last move count)`

2. **Allow steps {1,2,3} with constraints**
   - Expand state dimension

3. **Cost minimization instead of counting**
   - Replace `+` with `min`

4. **Binary sequence variant**
   - Map:
     - 1-step → `0`
     - 2-step → `1`
   - Constraint becomes: no consecutive `1`

This becomes:
→ classic "binary strings without consecutive 1s"

---

### Common Mistakes

- Using single DP array → loses last-move information
- Writing:
```

dp[i] = dp[i-1] + dp[i-2]

```
→ This ignores the constraint

- Allowing:
```

B[i] = B[i-2]

```
→ Violates constraint

- Missing base case:
- `B[2] = 1`

- Forgetting final sum:
- Returning only `A[n]`
```
