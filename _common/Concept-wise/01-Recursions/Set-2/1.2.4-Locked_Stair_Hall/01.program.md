# Locked Stair Hall

## Concepts Involved
- Base Case Design
- Return Meaning
- Choice Expansion
- Recursive State Definition
- Recursion Tree Analysis
- Constraint Propagation
- Counting Recursion
- Optimization Awareness

## Problem Statement
A student must climb from step `0` to step `n` inside a locked stair hall.

From any step, the student may normally move either:
- `1` step forward, or
- `2` steps forward.

Some steps are locked and cannot be landed on.

There is one extra hall rule:
- If the student is currently standing on a prime-numbered step, the next move must be exactly `1` step.

Count the total number of valid ways to reach exactly step `n`.

A path is valid only if:
- it starts from step `0`,
- it ends exactly at step `n`,
- it never lands on a locked step, and
- it obeys the prime-step forced movement rule.

Step `0` is never considered prime.

## Input Format
- First line: two integers `n` and `m`
  - `n` is the target step.
  - `m` is the number of locked steps.
- Second line: `m` space-separated integers representing locked steps.
  - If `m = 0`, this line may be empty.

## Output Format
- Print one integer: the number of valid ways to reach step `n`.

## Constraints
- `0 <= n <= 30`
- `0 <= m <= n`
- Locked steps are between `1` and `n`.
- Step `0` is not locked.
- The answer fits in a 64-bit signed integer.

## Examples

### Example 1
**Input**
```text
4 0
```

**Output**
```text
3
```

**Explanation**
Valid paths are:

```text
0 -> 1 -> 2 -> 3 -> 4
0 -> 1 -> 3 -> 4
0 -> 2 -> 3 -> 4
```

Whenever the student reaches step `2` or step `3`, the next move must be exactly `1` because both are prime.

### Example 2
**Input**
```text
6 1
5
```

**Output**
```text
3
```

**Explanation**
Step `5` is locked, so no path may land on it.

Valid paths are:

```text
0 -> 1 -> 2 -> 3 -> 4 -> 6
0 -> 1 -> 3 -> 4 -> 6
0 -> 2 -> 3 -> 4 -> 6
```

Step `4` is not prime, so the student may jump from `4` to `6`.

### Example 3
**Input**
```text
0 0
```

**Output**
```text
1
```

**Explanation**
The student is already at the target. This empty path counts as one valid way.

## Edge-Case Thinking Prompts
- What should the answer be when `n = 0`?
- What if the target step itself is locked?
- What happens if a move goes beyond step `n`?
- Why does the next move depend on the current step?
- Why does step `0` not trigger the prime rule?

## Thinking Like a Programmer
Before writing code, define:

```text
State: current step
Choice: move 1 or 2 steps, but only 1 if current step is prime
Transition: go to current + move
Base case: current == n gives one valid path
Invalid cases: current > n or current step is locked
Answer: countWays(0)
```
