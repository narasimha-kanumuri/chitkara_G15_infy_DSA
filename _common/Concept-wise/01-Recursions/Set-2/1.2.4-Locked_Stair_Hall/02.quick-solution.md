# Locked Stair Hall — Quick Solution

## Idea (Simple)
We count all valid climbing paths using recursion.

At each step, the student chooses the next move.

Usually the choices are `1` and `2`, but if the current step is prime, only move `1` is allowed.

Locked steps stop that path immediately.

## Pattern Recognition Clues
- We are asked to count all valid ways.
- Every move creates a smaller remaining stair problem.
- A rule in the statement changes the allowed choices.
- Invalid states appear when we cross the target or land on a locked step.

## State Discovery
The first idea is `countWays(remaining)`.

But the prime rule is easier when we know the current step number.

The allowed next moves depend on where the student is standing now.

So the final state is:

- `current`: the step where the student is standing

No previous-move state is needed because the rule depends only on the current step.

## Pattern
Counting Recursion with Constraint Propagation

## Core Insight
- A locked step contributes `0` ways.
- Reaching exactly step `n` contributes `1` way.
- Prime-numbered current steps reduce the choice set to only one move.

## Invariant
For every call `countWays(current)`, the function returns the number of valid paths from `current` to `n`, assuming the path used to reach `current` has already obeyed all rules.

## Algorithm (with one-line why at important steps)
1. If `current > n`, return `0`.
   - Why: the student crossed the target.
2. If `current` is locked, return `0`.
   - Why: locked steps cannot be used.
3. If `current == n`, return `1`.
   - Why: one complete valid path has been formed.
4. If `current` is prime, only try move `1`.
   - Why: the hall forces a one-step move from prime steps.
5. Otherwise, try moves `1` and `2`.
   - Why: these are the normal allowed moves.
6. Add the recursive answers.
   - Why: each valid branch gives a different path.

## Mental Cheat Sheet
| Item | Meaning |
|---|---|
| Pattern | Counting recursion with constraints |
| State | `current` step |
| Normal choices | `+1`, `+2` |
| Prime-step choice | only `+1` |
| Invalid state | crossed target or locked step |
| Answer | `countWays(0)` |

## Complexity
- Time Complexity: `O(2^n)` in the worst case.
- Space Complexity: `O(n)` because recursion depth can reach `n`.
- Auxiliary Space: `O(n)` recursion stack plus locked-step storage.

## Tiny Dry Run
For `n = 4` and no locked steps:

- From `0`, try `1` and `2`.
- From `2`, only `+1` is allowed because `2` is prime.
- From `3`, only `+1` is allowed because `3` is prime.
- Valid paths are `0-1-2-3-4`, `0-1-3-4`, and `0-2-3-4`.

Answer = `3`.

## Notes / Pitfalls
- Do not treat step `0` as prime.
- Check locked steps before expanding choices.
- Return `1` when `current == n`, not `0`.
- Do not allow a `2-step` move from a prime-numbered current step.
