# Locked Steps Cache — Java Solution

## Language

Java

## Plain-English Implementation Walkthrough

1. We need exact counting, so we use `BigInteger`.
   A normal integer may overflow for large `n`.

2. We store locked steps in a `boolean[] locked` array.
   This makes each locked-step check constant time.

3. We precompute prime steps in a `boolean[] prime` array.
   This avoids checking primality repeatedly during recursion.

4. We create `memo[step]` to store the answer from each step.
   The state is only the current step.

5. The method `solve(step, n, locked, prime, memo)` returns the number of ways from `step` to `n`.
   This meaning stays the same for every recursive call.

6. If `step > n`, we return `BigInteger.ZERO`.
   A path that crosses the target is invalid.

7. If `locked[step]` is true, we return `BigInteger.ZERO`.
   The student cannot stand on a locked step.

8. If `step == n`, we return `BigInteger.ONE`.
   This means one complete valid route has reached the target.

9. If `memo[step]` is already filled, we return it.
   This avoids solving the same step again.

10. If `prime[step]` is true, we only call `solve(step + 1, ...)`.
    This applies the forced movement rule.

11. If the step is not prime, we add the results from `step + 1` and `step + 2`.
    These are the two normal movement choices.

12. We save the total in `memo[step]` and return it.
    Future calls can reuse the saved answer.

## Complete Solution Code

```java
import java.io.BufferedReader;
import java.io.InputStreamReader;
import java.math.BigInteger;
import java.util.StringTokenizer;

public class Main {

    private static class FastScanner {
        private final BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        private StringTokenizer st;

        String next() throws Exception {
            while (st == null || !st.hasMoreTokens()) {
                String line = br.readLine();
                if (line == null) {
                    return null;
                }
                st = new StringTokenizer(line);
            }
            return st.nextToken();
        }

        int nextInt() throws Exception {
            return Integer.parseInt(next());
        }
    }

    private static boolean[] buildPrimeTable(int n) {
        boolean[] prime = new boolean[n + 1];

        if (n >= 2) {
            for (int i = 2; i <= n; i++) {
                prime[i] = true;
            }

            for (int p = 2; p * p <= n; p++) {
                if (prime[p]) {
                    for (int multiple = p * p; multiple <= n; multiple += p) {
                        prime[multiple] = false;
                    }
                }
            }
        }

        return prime;
    }

    private static BigInteger solve(int step, int n, boolean[] locked,
                                    boolean[] prime, BigInteger[] memo) {
        if (step > n) {
            return BigInteger.ZERO;
        }

        if (locked[step]) {
            return BigInteger.ZERO;
        }

        if (step == n) {
            return BigInteger.ONE;
        }

        if (memo[step] != null) {
            return memo[step];
        }

        BigInteger total;

        if (prime[step]) {
            total = solve(step + 1, n, locked, prime, memo);
        } else {
            total = solve(step + 1, n, locked, prime, memo)
                    .add(solve(step + 2, n, locked, prime, memo));
        }

        memo[step] = total;
        return total;
    }

    private static BigInteger countWays(int n, boolean[] locked) {
        boolean[] prime = buildPrimeTable(n);
        BigInteger[] memo = new BigInteger[n + 1];
        return solve(0, n, locked, prime, memo);
    }

    public static void main(String[] args) throws Exception {
        FastScanner fs = new FastScanner();

        int n = fs.nextInt();
        int m = fs.nextInt();

        boolean[] locked = new boolean[n + 1];
        for (int i = 0; i < m; i++) {
            int step = fs.nextInt();
            locked[step] = true;
        }

        System.out.println(countWays(n, locked));
    }
}
```

## Code Deconstruction

### `FastScanner`

```java
private static class FastScanner
```

This helper reads input token by token.
It works even if the locked steps appear on the same line or a later line.

### Prime table creation

```java
private static boolean[] buildPrimeTable(int n)
```

This method precomputes which step numbers are prime.
Step `0` and step `1` remain non-prime.

### `solve` method

```java
private static BigInteger solve(int step, int n, boolean[] locked,
                                boolean[] prime, BigInteger[] memo)
```

This method represents one memoized state.
The only changing value is `step`.

### Invalid and target cases

```java
if (step > n) return BigInteger.ZERO;
if (locked[step]) return BigInteger.ZERO;
if (step == n) return BigInteger.ONE;
```

These conditions handle invalid paths, locked steps, and successful completion.
The locked check comes before the target check, so a locked final step gives zero ways.

### Prime-step rule

```java
if (prime[step]) {
    total = solve(step + 1, n, locked, prime, memo);
}
```

A prime-numbered current step allows only a `1`-step move.
So the `+2` transition is not included.

### Normal-step transition

```java
total = solve(step + 1, n, locked, prime, memo)
        .add(solve(step + 2, n, locked, prime, memo));
```

A non-prime step allows both normal moves.
The two counts are added because the task asks for total number of ways.

## Language-Specific Notes

- `BigInteger` is immutable. Use `.add()` instead of `+`.
- `memo[step] == null` means the state has not been computed yet.
- `boolean[]` gives fast checks for locked and prime steps.
- Step `1` is not prime by definition.
- The recursion depth can reach `n`; this is acceptable for memoization practice, but tabulation can avoid stack depth.

## Complexity Summary

- Time Complexity: `O(n log log n)` including prime precomputation
- Space Complexity: `O(n)`
- Auxiliary Space: `O(n)`

The DP part computes each step once.
The prime table is built using a sieve-style loop.
