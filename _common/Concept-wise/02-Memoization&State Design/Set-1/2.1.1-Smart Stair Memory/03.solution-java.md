## Plain-English Implementation Walkthrough

1. We need exact large counts, so we use `BigInteger` instead of `int` or `long`.

2. The previous jump only matters in one way: was it a `2`-step jump or not?
   So we compress it into `lastType`:
   - `0` means previous jump was not `2`
   - `1` means previous jump was `2`

3. We create `memo[n + 1][2]` because valid current steps are from `0` to `n`, and `lastType` has two possible values.

4. The method `solve(step, lastType, n, memo)` means:
   “How many valid ways can we finish from this step, knowing whether the previous jump was `2`?”

5. If `step == n`, we return `BigInteger.ONE`.
   This means the current sequence reached the target exactly.

6. If `step > n`, we return `BigInteger.ZERO`.
   This path crossed the target, so it cannot be counted.

7. If `memo[step][lastType]` already has a value, we return it.
   This avoids solving the same state again.

8. We always try a `1`-step jump.
   After a `1`-step jump, the new `lastType` becomes `0` because the previous jump is not `2`.

9. We try a `2`-step jump only when `lastType == 0`.
   This condition blocks two consecutive `2`-step jumps.

10. We add the valid choices, save the total in the memo table, and return it.
    This saved value can be reused by other recursive paths.

## Complete Solution Code

```java
import java.io.BufferedReader;
import java.io.InputStreamReader;
import java.math.BigInteger;

public class Main {

    private static BigInteger solve(int step, int lastType, int n, BigInteger[][] memo) {
        if (step == n) {
            return BigInteger.ONE;
        }

        if (step > n) {
            return BigInteger.ZERO;
        }

        if (memo[step][lastType] != null) {
            return memo[step][lastType];
        }

        BigInteger total = BigInteger.ZERO;

        // Take a 1-step jump. This is always allowed.
        total = total.add(solve(step + 1, 0, n, memo));

        // Take a 2-step jump only if the previous jump was not 2.
        if (lastType == 0) {
            total = total.add(solve(step + 2, 1, n, memo));
        }

        memo[step][lastType] = total;
        return total;
    }

    private static BigInteger countWays(int n) {
        BigInteger[][] memo = new BigInteger[n + 1][2];
        return solve(0, 0, n, memo);
    }

    public static void main(String[] args) throws Exception {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        int n = Integer.parseInt(br.readLine().trim());
        System.out.println(countWays(n));
    }
}
```

## Code Deconstruction

### `solve` method

```java
private static BigInteger solve(int step, int lastType, int n, BigInteger[][] memo)
```

This method represents one DP state.

- `step` tells the current position.
- `lastType` tells whether the previous jump was `2`.
- `n` is the target step.
- `memo` stores already computed answers.

### Exact target base case

```java
if (step == n) {
    return BigInteger.ONE;
}
```

This means one complete valid path has been formed.

### Crossed target base case

```java
if (step > n) {
    return BigInteger.ZERO;
}
```

This prevents paths that jump beyond `n` from being counted.

### Memo check

```java
if (memo[step][lastType] != null) {
    return memo[step][lastType];
}
```

This is the memoization step.
If the same state appears again, the program returns the saved answer.

### Taking a `1`-step jump

```java
total = total.add(solve(step + 1, 0, n, memo));
```

A `1`-step jump is always allowed.
The new `lastType` becomes `0` because the last jump is not `2`.

### Taking a `2`-step jump

```java
if (lastType == 0) {
    total = total.add(solve(step + 2, 1, n, memo));
}
```

This is the key rule.
A `2`-step jump is allowed only when the previous jump was not `2`.

### Saving the result

```java
memo[step][lastType] = total;
return total;
```

The computed answer is saved before returning.
This makes future calls faster.

## Language-Specific Notes

- `BigInteger` objects are immutable. Use `.add()` to create a new sum.
- Do not use `+` to add `BigInteger` values.
- `memo[step][lastType] == null` means the state has not been computed yet.
- The recursion depth can reach `n`. For very large `n`, an iterative version avoids stack depth risk, but this problem is designed to teach memoization.
- `lastType` is used instead of storing the actual previous jump value. This keeps the memo table small.

## Complexity Summary

- Time Complexity: `O(n)`
- Space Complexity: `O(n)`
- Auxiliary Space: `O(n)`

There are at most `2 * (n + 1)` memo states.
Each state does constant work, except for `BigInteger` addition cost, which depends on the number of digits in the answer.
