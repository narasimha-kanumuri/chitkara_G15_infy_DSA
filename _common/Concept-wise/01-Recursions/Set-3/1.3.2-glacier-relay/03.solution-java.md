## Language

Java

## Plain-English Implementation Walkthrough

1. We store platform data from index `1` to index `n`.  
   This keeps array indexes close to platform numbers.

2. We start from platform `1`.  
   The climber is already standing there, so no landing cost is added for platform `1`.

3. The recursive method needs `current` because the next jump depends on the current platform.

4. The method needs `lastJumpWasTwo` because a `2`-jump may be blocked by the previous jump.

5. If `current == n`, we return `0`.  
   The climber has reached the final platform.

6. We try jump length `1` and jump length `2`.  
   These are the only allowed jump sizes.

7. If jump length is `2` and `lastJumpWasTwo` is true, we skip that choice.  
   This enforces the consecutive jump rule.

8. If the next platform goes beyond `n`, we skip it.  
   The climber must land on an existing platform.

9. We compute jump cost using height difference.  
   If the landing platform is blue, we add the cold penalty.

10. We recursively solve from the landing platform.  
    The new value of `lastJumpWasTwo` depends on the jump just taken.

11. We return the minimum valid total because the problem asks for minimum energy.

## Complete Solution Code

```java
import java.util.*;

public class Main {
    static final int INF = 1_000_000_000;

    static int solve(int current, boolean lastJumpWasTwo, int n, int[] height, int[] blue, int coldPenalty) {
        if (current == n) {
            return 0;
        }

        int answer = INF;

        for (int jump = 1; jump <= 2; jump++) {
            if (jump == 2 && lastJumpWasTwo) {
                continue;
            }

            int next = current + jump;

            if (next > n) {
                continue;
            }

            int landingCost = Math.abs(height[next] - height[current]);

            if (blue[next] == 1) {
                landingCost += coldPenalty;
            }

            int remainingCost = solve(next, jump == 2, n, height, blue, coldPenalty);
            int totalCost = landingCost + remainingCost;

            answer = Math.min(answer, totalCost);
        }

        return answer;
    }

    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);

        int n = sc.nextInt();
        int coldPenalty = sc.nextInt();

        int[] height = new int[n + 1];
        int[] blue = new int[n + 1];

        for (int i = 1; i <= n; i++) {
            height[i] = sc.nextInt();
        }

        for (int i = 1; i <= n; i++) {
            blue[i] = sc.nextInt();
        }

        int result = solve(1, false, n, height, blue, coldPenalty);
        System.out.println(result);
    }
}
```

## Code Deconstruction

### `solve(current, lastJumpWasTwo, n, height, blue, coldPenalty)`

This method returns the minimum energy needed from the current platform to the final platform.

- `current` tells where the climber is standing now.
- `lastJumpWasTwo` tells whether a `2`-jump is currently blocked.
- `height` is used to calculate jump energy.
- `blue` is used to check cold penalty.

### Base Case

```java
if (current == n) {
    return 0;
}
```

When the climber reaches the final platform, there is no more movement cost.

### Jump Choices

```java
for (int jump = 1; jump <= 2; jump++)
```

The loop tries both possible jump lengths.

### Consecutive 2-Jump Rule

```java
if (jump == 2 && lastJumpWasTwo) {
    continue;
}
```

This prevents two `2`-jumps from happening back to back.

### Landing Cost

```java
int landingCost = Math.abs(height[next] - height[current]);
```

This gives the normal energy cost for moving between two platforms.

### Blue Platform Cost

```java
if (blue[next] == 1) {
    landingCost += coldPenalty;
}
```

Cold cost is added only when the next platform is blue and the climber lands on it.

### Recursive Continuation

```java
int remainingCost = solve(next, jump == 2, n, height, blue, coldPenalty);
```

The recursive call moves the climber to the next platform.  
It also updates whether the latest jump was length `2`.

## Language-Specific Notes

- Java array index `0` is unused here so platform numbers can start from `1`.
- `boolean` makes the previous-jump state easy to read.
- Pure recursion repeats subproblems. This is intentional for Day 1.
- For larger input, this same state can be memoized as `dp[current][lastJumpWasTwo]` on Day 2.

## Complexity Summary

- Time Complexity: `O(2^n)` for pure recursion.
- Space Complexity: `O(n)` due to recursion stack.
- Auxiliary Space: `O(n)` due to recursion stack.
