## Title
Glacier Iterative Traverse

## Concepts Involved
Tabulation, Constraint-aware Transitions, Optimization DP

## Problem Statement
A climber is crossing a line of icy platforms.

There are `n` platforms numbered from `1` to `n`.
Each platform `i` has a height `height[i]`.
Some platforms are marked as blue platforms.

The climber starts on platform `1` and wants to reach platform `n`.
From any platform, the climber may jump:
- `1` platform forward, or
- `2` platforms forward.

The energy cost of a jump from platform `i` to platform `j` is:

`absolute difference between height[i] and height[j]`

If the climber lands on a blue platform, an extra cold penalty `c` is added for that landing.
The starting platform does not add cold penalty because the climber begins there.

There is one more rule:
- The climber cannot make two consecutive `2`-platform jumps.

Find the minimum total energy required to reach platform `n`.

## Input Format
- First line: integer `n`
- Second line: `n` space-separated integers `height[1], height[2], ..., height[n]`
- Third line: `n` space-separated integers `blue[1], blue[2], ..., blue[n]`
  - `blue[i] = 1` means platform `i` is blue
  - `blue[i] = 0` means platform `i` is normal
- Fourth line: integer `c`

## Output Format
- Print the minimum total energy required to reach platform `n`

## Constraints
- `1 <= n <= 200000`
- `0 <= height[i] <= 1000000000`
- `blue[i]` is either `0` or `1`
- `0 <= c <= 1000000000`

## Examples
### Example 1
Input:
`5`
`10 30 40 20 50`
`0 0 1 0 0`
`5`

Output:
`45`

### Example 2
Input:
`4`
`5 100 5 100`
`0 0 0 0`
`50`

Output:
`95`

### Example 3
Input:
`5`
`1 100 2 100 3`
`0 1 0 1 0`
`10`

Output:
`206`

## Edge-Case Thinking Prompts
- What if there is only one platform?
- Should the starting platform add cold penalty?
- What if the best-looking path uses two consecutive `2`-jumps?
- What if all platforms are blue?
- Why is platform index alone not enough as a DP state?

## Thinking Like a Programmer
- What cost is paid when we land on a platform?
- Which previous jump affects the next valid move?
- How can we store whether the previous jump was `2`?
- What should the base case at platform `1` mean?
- Where is the final minimum answer stored?
