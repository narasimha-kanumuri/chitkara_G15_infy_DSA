# Glacier Relay

## Concepts Involved

- Base Case Design
- Return Meaning
- Choice Expansion
- Recursive State Definition
- Recursion Tree Analysis
- Constraint Propagation
- Counting vs Optimization Recursion
- Optimization Recursion

## Problem Statement

A climber is crossing `n` icy platforms arranged in a straight line.  
The climber starts on platform `1` and must reach platform `n`.

Each platform has a height.  
From the current platform, the climber may jump either `1` platform forward or `2` platforms forward.

The energy cost of a jump from platform `i` to platform `j` is:

```text
absolute difference between height[i] and height[j]
```

Some platforms are blue.  
Landing on a blue platform adds an extra cold cost `coldPenalty`.

There is one more restriction:

```text
The climber cannot take two 2-platform jumps consecutively.
```

Return the minimum total energy required to reach platform `n`.

The starting platform does not add cold cost because the climber is already standing there.

## Input Format

- The first line contains two integers `n` and `coldPenalty`.
- The second line contains `n` integers: `height[1], height[2], ..., height[n]`.
- The third line contains `n` integers: `blue[1], blue[2], ..., blue[n]`.
  - `blue[i] = 1` means platform `i` is blue.
  - `blue[i] = 0` means platform `i` is not blue.

## Output Format

Print one integer: the minimum total energy needed to reach platform `n`.

## Constraints

- `1 <= n <= 30`
- `1 <= height[i] <= 10^5`
- `0 <= coldPenalty <= 10^5`
- `blue[i]` is either `0` or `1`
- The intended solution for this file is pure recursion, so the input size is kept small for teaching.

## Examples

### Example 1

**Input**

```text
5 4
10 14 9 20 18
0 1 0 1 0
```

**Output**

```text
18
```

**Explanation**

One minimum-energy route is:

```text
1 -> 3 -> 4 -> 5
```

Energy:

- `1 -> 3`: `|10 - 9| = 1`
- `3 -> 4`: `|9 - 20| + 4 = 15` because platform `4` is blue
- `4 -> 5`: `|20 - 18| = 2`

Total energy = `1 + 15 + 2 = 18`.

The route `1 -> 3 -> 5` is not allowed because it uses two consecutive 2-platform jumps.

### Example 2

**Input**

```text
4 5
5 11 12 20
0 1 1 0
```

**Output**

```text
20
```

**Explanation**

Two valid low-cost routes are:

```text
1 -> 2 -> 4
1 -> 3 -> 4
```

Both have total energy `20` after adding cold cost on the blue platform that is landed on.

### Example 3

**Input**

```text
1 7
15
1
```

**Output**

```text
0
```

**Explanation**

The climber is already on the final platform.  
The starting platform does not add cold cost.

## Edge-Case Thinking Prompts

- What should the function return when the current platform is already `n`?
- What should happen if a jump goes beyond `n`?
- Why does the recursive function need to remember whether the previous jump was length `2`?
- Should blue cost be added for the starting platform?
- Is this a counting recursion problem or a minimum-cost recursion problem?

## Thinking Like a Programmer

Before writing code, define:

```text
State:
Choice:
Transition:
Base case:
Order:
Answer:
```

For this problem:

```text
State: current platform and whether the previous jump was length 2
Choice: jump 1 platform or jump 2 platforms
Transition: jump cost + blue landing cost + recursive answer from next platform
Base case: when current platform is n, no more energy is needed
Order: recursively explore all valid forward paths
Answer: minimum energy returned by solve(1, false)
```
