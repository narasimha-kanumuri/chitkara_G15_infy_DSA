## Plain-English Implementation Walkthrough

1. We need to know which rungs are broken, so we create a `boolean[] broken` array.
   Related code: `boolean[] broken = new boolean[n + 1];`

2. We read every broken rung and mark it as `true`.
   This lets us check a broken rung in constant time.
   Related code: `broken[rung] = true;`

3. We start before the first rung at position `0`.
   No previous jump exists yet, so `lastWasTwo` is `false`.
   Related code: `solve(0, false, n, broken)`

4. If `pos == n`, the path has reached the top exactly.
   We return `1` because this is one valid sequence.
   Related code: `if (pos == n) return 1;`

5. If `pos > n`, the path crossed the top and is invalid.
   We return `0` because this path must not be counted.
   Related code: `if (pos > n) return 0;`

6. If the current position is a broken rung, this path is invalid.
   We do not continue from a broken rung.
   Related code: `if (pos > 0 && broken[pos]) return 0;`

7. We always try a `1-rung` jump.
   After a `1-rung` jump, the previous jump is not `2`.
   Related code: `ways += solve(pos + 1, false, n, broken);`

8. We try a `2-rung` jump only if the previous jump was not `2`.
   This enforces the consecutive `2-rung` rule.
   Related code: `if (!lastWasTwo) ...`

9. We return the total number of valid ways from both choices.
   The sum is correct because each choice starts a different path.
   Related code: `return ways;`

## Complete Solution Code

```java
import java.util.Scanner;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);

        int n = sc.nextInt();
        int m = sc.nextInt();

        boolean[] broken = new boolean[n + 1];
        for (int i = 0; i < m; i++) {
            int rung = sc.nextInt();
            broken[rung] = true;
        }

        long answer = solve(0, false, n, broken);
        System.out.println(answer);

        sc.close();
    }

    private static long solve(int pos, boolean lastWasTwo, int n, boolean[] broken) {
        if (pos == n) {
            return 1;
        }

        if (pos > n) {
            return 0;
        }

        if (pos > 0 && broken[pos]) {
            return 0;
        }

        long ways = 0;

        ways += solve(pos + 1, false, n, broken);

        if (!lastWasTwo) {
            ways += solve(pos + 2, true, n, broken);
        }

        return ways;
    }
}
```

## Code Deconstruction

- `main` reads the tower size and the broken rung list.
- `broken[rung] = true` marks a rung that cannot be used.
- `solve(0, false, n, broken)` starts the recursive search.
- `pos == n` is the successful base case.
- `pos > n` is the overshoot base case.
- `pos > 0 && broken[pos]` rejects a path that lands on a broken rung.
- `solve(pos + 1, false, n, broken)` represents taking a `1-rung` jump.
- `solve(pos + 2, true, n, broken)` represents taking a `2-rung` jump.
- `!lastWasTwo` prevents two consecutive `2-rung` jumps.

## Language-Specific Notes

- `long` is safer than `int` for counting paths.
- This solution intentionally uses plain recursion because Day 1 focuses on recursion understanding.
- For larger `n`, the same state can repeat many times. That will be improved using memoization in the next day.
- `Scanner` is simple and readable for teaching. For very large input, `BufferedReader` is faster.

## Complexity Summary

- Time Complexity: `O(2^n)` in the worst case without memoization
- Space Complexity: `O(n)` because recursion depth can go up to `n`
- Auxiliary Space: `O(n)` for recursion stack and the broken-rung array
