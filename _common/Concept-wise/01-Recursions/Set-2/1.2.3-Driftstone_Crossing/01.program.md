# Driftstone Crossing

## Concepts Involved
- Base Case Design
- Return Meaning
- Choice Expansion
- Recursive State Definition
- Recursion Tree Analysis
- Constraint Propagation
- Route Comparison Recursion
- Optimization Recursion

## Problem Statement
A robot must cross a line of driftstones numbered from `0` to `n - 1`.

Each stone has a height. The robot starts on stone `0` and must reach stone `n - 1`.

From any stone, the robot may jump:
- `1` stone forward,
- `2` stones forward, or
- `3` stones forward exactly once in the entire journey.

Energy rules:
- A `1-step` or `2-step` jump costs `abs(height[next] - height[current])`.
- A `3-step` jump costs `2 * abs(height[next] - height[current])`.
- If the robot takes two `2-step` jumps consecutively, it pays an extra fatigue penalty `p` on the second `2-step` jump.

Return the minimum energy required to reach the last stone.

## Input Format
- First line: integer `n`
- Second line: `n` space-separated integers representing stone heights
- Third line: integer `p`, the fatigue penalty

## Output Format
- Print one integer: the minimum energy required to reach the last stone.

## Constraints
- `1 <= n <= 20`
- `0 <= height[i] <= 10^5`
- `0 <= p <= 10^5`
- The answer fits in a 64-bit signed integer.

## Examples

### Example 1
**Input**
```text
5
10 30 20 40 25
7
```

**Output**
```text
22
```

**Explanation**
One best route is:

```text
stone 0 -> stone 2 -> stone 4
```

Energy:

```text
abs(20 - 10) + abs(25 - 20) + fatigue penalty = 10 + 5 + 7 = 22
```

The fatigue penalty is paid because the route uses two consecutive `2-step` jumps.

### Example 2
**Input**
```text
6
5 100 6 100 7 8
20
```

**Output**
```text
5
```

**Explanation**
One best route is:

```text
stone 0 -> stone 2 -> stone 5
```

Energy:

```text
abs(6 - 5) + 2 * abs(8 - 6) = 1 + 4 = 5
```

The jump `2 -> 5` is the one-time `3-step` jump, so its cost is doubled.

### Example 3
**Input**
```text
1
42
10
```

**Output**
```text
0
```

**Explanation**
The robot is already on the last stone.

## Edge-Case Thinking Prompts
- What should the answer be when there is only one stone?
- What happens if a jump goes beyond the last stone?
- Why do we need to remember the previous jump?
- Why do we need to remember whether the `3-step` jump was already used?
- When exactly should the fatigue penalty be added?

## Thinking Like a Programmer
Before writing code, define:

```text
State: current index, previous jump size, whether 3-step jump is already used
Choice: try jump 1, jump 2, and maybe jump 3
Transition: jump to the next index and add jump energy
Base case: current index reaches last stone
Invalid case: jump crosses beyond the last stone
Answer: minimum energy from state (0, 0, false)
```
