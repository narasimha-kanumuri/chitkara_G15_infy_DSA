# Driftstone Memory Path — Quick Solution

## Idea (Simple)

We need the minimum energy to reach the last stone.
The robot can jump `1` or `2` stones normally.
It can also use one special `3-step` jump once.
A second consecutive `2-step` jump gets fatigue penalty `p`.

So we use memoized recursion with three state parts: current index, whether the special jump was used, and whether the previous jump was `2`.

## Pattern Recognition Clues

- The task asks for minimum total energy.
- From each stone, similar jump choices repeat.
- Future choices depend on past decisions.
- Repeated path evaluations suggest memoization.

## State Discovery

The first simple state is `solve(i)`.

But `solve(i)` is not enough.
It does not know whether the special `3-step` jump is still available.
It also does not know whether the previous jump was a `2-step` jump.

Both facts affect future cost.

So we use:

```text
solve(i, usedThree, prevTwo)
```

Meaning:

```text
minimum energy from stone i to the last stone,
when usedThree tells whether the special jump is used,
and prevTwo tells whether the previous jump was 2-step
```

## Pattern

Top-Down DP with Multi-constraint State

## Core Insight

- A `1-step` jump is always allowed.
- A `2-step` jump may need fatigue penalty if `prevTwo = 1`.
- A `3-step` jump is allowed only if `usedThree = 0`.
- Cache every `(i, usedThree, prevTwo)` result.

## Invariant

For every state `solve(i, usedThree, prevTwo)`:

- The robot is currently on stone `i`.
- `usedThree` correctly records whether the special jump has already been used.
- `prevTwo` correctly records whether the jump used to arrive here was length `2`.
- The function returns the minimum remaining energy from this state.

## Algorithm (with one-line why at important steps)

1. Start with `solve(0, 0, 0)`.
   - Why: the robot starts at stone `0`, no special jump is used, and no previous jump exists.
2. If `i == n - 1`, return `0`.
   - Why: no more energy is needed after reaching the last stone.
3. If the state is cached, return it.
   - Why: the same state may be reached by different paths.
4. Try a `1-step` jump if possible.
   - Why: it is a normal move and resets `prevTwo` to `0`.
5. Try a `2-step` jump if possible.
   - Why: it is a normal move, but add penalty if `prevTwo = 1`.
6. Try a `3-step` jump only if the special jump is unused.
   - Why: the special move can be used at most once.
7. Store and return the minimum value.
   - Why: the problem asks for minimum energy.

## Mental Cheat Sheet

| Item | Meaning |
|---|---|
| Pattern | Memoization with multi-constraint state |
| State | `(i, usedThree, prevTwo)` |
| Normal choices | jump `1` or jump `2` |
| Special choice | jump `3` once at double height-difference cost |
| Fatigue rule | add `p` when a `2-step` follows another `2-step` |
| Answer | `solve(0, 0, 0)` |

## Complexity

- Time Complexity: `O(n)`
- Space Complexity: `O(n)`
- Auxiliary Space: `O(n)` for memo table and recursion stack

There are at most `n * 2 * 2` states.
Each state checks at most three jumps.

## Tiny Dry Run

For:

```text
height = [10, 30, 20, 40, 15], p = 10
```

One good route is:

```text
0 -> 2 -> 4
```

Cost:

```text
0 -> 2 = 10
2 -> 4 = 5 + 10 penalty
Total = 25
```

## Notes / Pitfalls

- Do not cache only by index `i`.
- The special jump usage must be part of the state.
- The previous `2-step` status must be part of the state.
- Add penalty only when the current jump is `2` and previous jump was also `2`.
- Use `long` for energy values.
