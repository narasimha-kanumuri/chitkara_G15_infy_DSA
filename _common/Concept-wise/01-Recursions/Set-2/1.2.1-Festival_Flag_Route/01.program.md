# Festival Flag Route

## Concepts Involved
- Base Case Design
- Return Meaning
- Choice Expansion
- Recursive State Definition
- Recursion Tree Analysis
- Constraint Propagation
- Counting Recursion
- Optimization Awareness

## Problem Statement
A festival volunteer must climb from step `0` to step `n` to place flags on the final platform.

From any current step, the volunteer may jump `1`, `2`, or `3` steps. However, the same jump size cannot be used in two consecutive moves.

Count the total number of valid routes that reach exactly step `n`.

A route is valid only if:
- the sum of all jumps is exactly `n`, and
- no two neighboring jumps are equal.

## Input Format
- A single integer `n`.

## Output Format
- Print one integer: the number of valid routes to reach step `n`.

## Constraints
- `0 <= n <= 25`
- The answer fits in a 64-bit signed integer.

## Examples

### Example 1
**Input**
```text
3
```

**Output**
```text
3
```

**Explanation**
Valid routes are:
- `1 + 2`
- `2 + 1`
- `3`

The route `1 + 1 + 1` is not valid because jump `1` is repeated consecutively.

### Example 2
**Input**
```text
4
```

**Output**
```text
3
```

**Explanation**
Valid routes are:
- `1 + 2 + 1`
- `1 + 3`
- `3 + 1`

Routes like `2 + 2` are not valid because the same jump size appears consecutively.

### Example 3
**Input**
```text
0
```

**Output**
```text
1
```

**Explanation**
There is one empty route: already being at the target.

## Edge-Case Thinking Prompts
- What should the answer be when `n = 0`?
- What happens if a jump goes beyond `n`?
- Why do we need to remember the previous jump?
- Can the first jump be any of `1`, `2`, or `3`?

## Thinking Like a Programmer
Before writing code, define:

```text
State: remaining steps and previous jump
Choice: try jump 1, 2, or 3
Transition: reduce remaining steps by chosen jump
Base case: remaining == 0 gives one valid route
Invalid case: remaining < 0 gives zero routes
Answer: countWays(n, 0)
```
