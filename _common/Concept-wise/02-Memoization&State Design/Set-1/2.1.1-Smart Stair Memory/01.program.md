# Smart Stair Memory

## Concepts Involved

Memoization, Recursion to DP, State Design, Counting DP, Last Move Constraint

## Problem Statement

A climber wants to reach exactly step `n`.

From any step, the climber may jump either `1` step or `2` steps.

However, two `2`-step jumps are not allowed to happen consecutively.

A plain recursive solution will solve the same subproblems many times.
Your task is to count the number of valid ways to reach step `n` using memoization.

A valid way is defined by the sequence of jumps used.
Two ways are different if at least one jump length differs.

## Input Format

- A single integer `n`

## Output Format

- Print the total number of valid ways to reach exactly step `n`.

## Constraints

- `1 <= n <= 2000`
- The answer may be very large.
- Use exact large number arithmetic.

## Examples

### Example 1

Input:

```text
1
```

Output:

```text
1
```

Explanation:

Only one sequence is possible:

```text
1
```

### Example 2

Input:

```text
2
```

Output:

```text
2
```

Explanation:

The valid sequences are:

```text
1 + 1
2
```

### Example 3

Input:

```text
4
```

Output:

```text
4
```

Explanation:

The valid sequences are:

```text
1 + 1 + 1 + 1
1 + 1 + 2
1 + 2 + 1
2 + 1 + 1
```

The sequence `2 + 2` is invalid because it has two consecutive `2`-step jumps.

## Edge-Case Thinking Prompts

- What should the answer be when `n = 1`?
- Why is `2` a valid single jump for `n = 2`?
- Why is `2 + 2` invalid for `n = 4`?
- Is the current step alone enough to decide future moves?
- What extra information is needed to know whether a `2`-step jump is allowed now?

## Thinking Like a Programmer

Before writing code, answer these questions:

```text
State:
Choice:
Transition:
Base case:
Cache key:
Answer:
```

The important idea is that the same step can be reached in different ways.
For example, reaching step `3` after a `1`-step jump is different from reaching step `3` after a `2`-step jump.
That difference matters because it decides whether the next `2`-step jump is allowed.

## Expected Answers

- **State**: The current step number and the type of the last jump taken (1-step or 2-step).
- **Choice**: The decision of whether to take a 1-step or 2-step jump at each position.
- **Transition**: The effect of that choice on the state of the system, specifically how it affects the next possible moves.
- **Base case**: Reaching the exact step `n`.
- **Cache key**: The current step and the type of the last jump (1-step or 2-step).
- **Answer**: The total number of valid ways to reach step `n`.
