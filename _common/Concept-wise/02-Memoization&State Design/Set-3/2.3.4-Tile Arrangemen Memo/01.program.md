# Tile Arrangement Memo

## Concepts Involved

Memoization, State Design, Combinatorial DP, Last Tile Constraint, Recursion to DP

## Problem Statement

A corridor of length `n` must be fully covered using tiles.

There are two tile sizes available:

- a small tile of length `1`
- a long tile of length `2`

The corridor must be covered exactly from left to right.
No gap and no overlap are allowed.

The tiling must satisfy two rules:

1. Two long tiles of length `2` cannot be placed next to each other.
2. The final tile placed at the end of the corridor must be a small tile of length `1`.

A plain recursive solution repeatedly solves the same remaining-length states.
Use memoization to avoid recomputation.

Return the total number of valid tilings of the corridor.

A tiling is defined by the sequence of tile lengths used.
Two tilings are different if at least one tile length differs.

## Input Format

- A single integer `n`

## Output Format

- Print the total number of valid tilings of a corridor of length `n`.

## Constraints

- `1 <= n <= 2000`
- The answer may be very large.
- Use exact large number arithmetic.

## Examples

### Example 1

Input:

```text
1
```

Output:

```text
1
```

Explanation:

Only one tiling is possible:

```text
1
```

The final tile is length `1`, so it is valid.

### Example 2

Input:

```text
2
```

Output:

```text
1
```

Explanation:

The valid tiling is:

```text
1 + 1
```

The tiling `2` is not valid because the final tile is length `2`.

### Example 3

Input:

```text
4
```

Output:

```text
3
```

Explanation:

The valid tilings are:

```text
1 + 1 + 1 + 1
2 + 1 + 1
1 + 2 + 1
```

The tiling `1 + 1 + 2` is not valid because it ends with a length-`2` tile.
The tiling `2 + 2` is not valid because two long tiles are adjacent and it also ends with length `2`.

## Edge-Case Thinking Prompts

- What should the answer be when `n = 1`?
- Why is the single tile `2` invalid for `n = 2`?
- Why is `2 + 2` invalid for `n = 4`?
- Is remaining length alone enough to know whether a length-`2` tile can be placed next?
- What must be remembered to prevent adjacent long tiles?

The important idea is that the same remaining length can behave differently.
If the previous tile was length `2`, then the next tile cannot also be length `2`.
So the memoized state must remember the remaining length and whether the previous tile was long.
