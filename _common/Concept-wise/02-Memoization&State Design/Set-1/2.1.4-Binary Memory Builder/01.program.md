# Binary Memory Builder

## Concepts Involved

Memoization, State Design, Counting DP, Binary Sequence DP, Recursion to DP

## Problem Statement

A system builds a binary sequence of length `n` one position at a time.

Each position may contain either `0` or `1`.

However, the sequence must satisfy two rules:

1. It must not contain two consecutive `1`s.
2. It must end with `0`.

A direct recursive solution may repeatedly solve the same smaller sequence-building states.
Use memoization to avoid recomputation.

Return the total number of valid binary sequences of length `n`.

## Input Format

- A single integer `n`

## Output Format

- Print the total number of valid binary sequences of length `n` that satisfy the rules.

## Constraints

- `1 <= n <= 2000`
- The answer may be very large.
- Use exact large number arithmetic.

## Examples

### Example 1

Input:

```text
1
```

Output:

```text
1
```

Explanation:

The only valid sequence is:

```text
0
```

The sequence must end with `0`, so `1` is not valid.

### Example 2

Input:

```text
2
```

Output:

```text
2
```

Explanation:

The valid sequences are:

```text
00
10
```

### Example 3

Input:

```text
4
```

Output:

```text
5
```

Explanation:

The valid sequences are:

```text
0000
0010
0100
1000
1010
```

Sequences such as `0110` are invalid because they contain consecutive `1`s.
Sequences ending in `1` are also invalid.

## Edge-Case Thinking Prompts

- What should happen when `n = 1`?
- Why is a sequence ending in `1` not counted?
- Why does the previous bit affect the next choice?
- Is the current position alone enough to decide all future choices?
- What repeated subproblems appear in the recursion tree?
