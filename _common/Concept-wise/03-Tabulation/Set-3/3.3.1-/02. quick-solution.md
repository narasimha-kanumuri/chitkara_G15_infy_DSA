## Idea (Simple)
We need the minimum toll to cross the staircase.
The agent can move `1` or `2` steps.
One real landed step may become free using a coupon.
So we track the minimum cost with the coupon unused and with the coupon already used.

## Pattern Recognition Clues
- We are minimizing total cost.
- Each position depends on earlier positions.
- A one-time coupon changes future choices.
- This suggests DP with a coupon-used state.

## State Discovery
The first idea is `dp[i]`: minimum toll to reach position `i`.
But `dp[i]` is not enough.
It forgets whether the coupon is still available.
That matters because future landed steps may become free.
So we split the state:
- `withoutCoupon[i]`: minimum cost to reach `i` with coupon unused
- `withCoupon[i]`: minimum cost to reach `i` after coupon has been used

## Pattern
Bottom-Up Min-Cost DP with One-Time Coupon State

## Core Insight
- To keep the coupon unused, we must pay the current landed toll.
- To enter the used-coupon state, either the coupon was used earlier, or we use it now.
- The final position has cost `0`, and the coupon cannot be used there.

## Invariant
For every position `i`:
- `withoutCoupon[i]` is the minimum toll to reach `i` with the coupon still unused.
- `withCoupon[i]` is the minimum toll to reach `i` after the coupon has been used.
Both values are best results after considering all valid `1`-step and `2`-step moves.

## Algorithm (with one-line why at important steps)
1. Add a virtual finish position `n + 1` with cost `0`.
   - Why: crossing the staircase becomes a normal destination.
2. Set `withoutCoupon[0] = 0` and `withCoupon[0] = INF`.
   - Why: we start on the ground with zero toll and coupon unused.
3. For each position, take the best previous value from `i - 1` and `i - 2`.
   - Why: only `1`-step and `2`-step moves are allowed.
4. Update `withoutCoupon[i]` by paying `cost[i]`.
   - Why: the coupon remains unused only if we do not use it here.
5. Update `withCoupon[i]` by comparing using the coupon earlier and using it now.
   - Why: these are the two ways to be in the used-coupon state.
6. Do not use the coupon at `n + 1`.
   - Why: the final position is not a real toll step.
7. Return the smaller of both states at `n + 1`.
   - Why: the coupon is optional, so either ending state may be optimal.

## Mental Cheat Sheet
| Item | Meaning |
|---|---|
| Pattern | Min-cost DP with one-time coupon |
| Simple failed state | `dp[i]` forgets coupon availability |
| State | position + coupon status |
| Real step | coupon may cancel toll |
| Virtual finish | cost `0`, coupon cannot be used |
| Answer | `min(withoutCoupon[n + 1], withCoupon[n + 1])` |

## Complexity
- Time Complexity: `O(n)`
- Space Complexity: `O(n)`
- Auxiliary Space: `O(n)`

## Tiny Dry Run
For `cost = [4, 1, 6, 3]`:
- The finish is position `5` with cost `0`.
- One cheap route lands on step `2`, then step `4`, then finish.
- Pay `1` at step `2` and use coupon on step `4`.
- Minimum toll = `1`.

## Notes / Pitfalls
- Do not use one DP array; coupon status matters.
- Do not use the coupon on the virtual finish.
- The coupon is optional, so compare both final states.
- Use `long` because total toll can be large.
