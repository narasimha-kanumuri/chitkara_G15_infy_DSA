## Plain-English Implementation Walkthrough
1. We need exact counting, and the count can become large.  
   So the code uses `BigInteger` instead of `int` or `long`.

2. We must remember the last jump used to reach each step.  
   So the code creates three arrays: `endWithOne`, `endWithTwo`, and `endWithThree`.

3. Java object arrays start with `null`.  
   So the code fills every cell with `BigInteger.ZERO` before using `.add(...)`.

4. A route can start with a direct jump from the ground.  
   So step `1` gets one route ending with `1`, step `2` gets one route ending with `2`, and step `3` gets one route ending with `3`.

5. To end at step `i` with jump `1`, the previous step is `i - 1`.  
   The previous route must end with `2` or `3`, not `1`.

6. To end at step `i` with jump `2`, the previous step is `i - 2`.  
   The previous route must end with `1` or `3`, not `2`.

7. To end at step `i` with jump `3`, the previous step is `i - 3`.  
   The previous route must end with `1` or `2`, not `3`.

8. Some previous indices may not exist for small `i`.  
   So the code checks `i >= 2` and `i >= 3` before using those transitions.

9. At the end, the final route may end with any jump size.  
   So the method returns the sum of the three states at step `n`.

## Complete Solution Code
```java
import java.io.BufferedReader;
import java.io.InputStreamReader;
import java.math.BigInteger;

public class Main {
    private static BigInteger countRoutes(int n) {
        BigInteger[] endWithOne = new BigInteger[n + 1];
        BigInteger[] endWithTwo = new BigInteger[n + 1];
        BigInteger[] endWithThree = new BigInteger[n + 1];

        for (int i = 0; i <= n; i++) {
            endWithOne[i] = BigInteger.ZERO;
            endWithTwo[i] = BigInteger.ZERO;
            endWithThree[i] = BigInteger.ZERO;
        }

        endWithOne[1] = BigInteger.ONE;

        if (n >= 2) {
            endWithTwo[2] = BigInteger.ONE;
        }

        if (n >= 3) {
            endWithThree[3] = BigInteger.ONE;
        }

        for (int step = 2; step <= n; step++) {
            endWithOne[step] = endWithTwo[step - 1].add(endWithThree[step - 1]);

            if (step >= 2) {
                endWithTwo[step] = endWithOne[step - 2].add(endWithThree[step - 2]);
            }

            if (step >= 3) {
                endWithThree[step] = endWithOne[step - 3].add(endWithTwo[step - 3]);
            }
        }

        return endWithOne[n].add(endWithTwo[n]).add(endWithThree[n]);
    }

    public static void main(String[] args) throws Exception {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        int n = Integer.parseInt(br.readLine().trim());
        System.out.println(countRoutes(n));
    }
}
```

## Code Deconstruction
- `countRoutes(int n)`
  - Computes the exact number of valid routes to step `n`.
- `endWithOne[step]`
  - Stores routes that reach `step` using jump `1` as the last jump.
- `endWithTwo[step]`
  - Stores routes that reach `step` using jump `2` as the last jump.
- `endWithThree[step]`
  - Stores routes that reach `step` using jump `3` as the last jump.
- Base cases
  - `endWithOne[1] = 1` represents route `[1]`.
  - `endWithTwo[2] = 1` represents route `[2]`.
  - `endWithThree[3] = 1` represents route `[3]`.
- Transition for jump `1`
  - Comes from step `step - 1` ending with jump `2` or `3`.
- Transition for jump `2`
  - Comes from step `step - 2` ending with jump `1` or `3`.
- Transition for jump `3`
  - Comes from step `step - 3` ending with jump `1` or `2`.
- Final return
  - Adds all three ending states because the last valid jump may be `1`, `2`, or `3`.

## Language-Specific Notes
- Use `BigInteger` because exact route counts may exceed `long`.
- Use `.add(...)` for `BigInteger`; the `+` operator is not used for this type.
- Initialize every `BigInteger[]` cell before using it.
- The DP arrays are 1-indexed by step number for readability.
- The solution is iterative and avoids recursion stack issues.
