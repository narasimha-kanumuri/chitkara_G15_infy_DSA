# Beacon Cache Flow — Quick Solution

## Idea (Simple)

The beacon value for a day depends on the previous two days.
This creates many repeated recursive calls.
So we compute each day's brightness once and store it in a memo table.

If the day index is a multiple of `4`, we subtract `1` after adding the previous two values.

## Pattern Recognition Clues

- The value of day `i` depends on smaller days.
- Direct recursion repeats the same day computations.
- The correction rule depends only on the current day index.
- This suggests top-down DP with memoization.

## State Discovery

The first simple state is `brightness(day)`.

This state is enough.
The value for a given day never changes based on how we reached it.

No extra state like previous choice or used coupon is needed.

So the final state is:

```text
solve(day)
```

Meaning:

```text
brightness value on that day
```

## Pattern

Top-Down Memoization on Linear Recurrence

## Core Insight

- Base days are known directly.
- Every later day depends on the previous two days.
- The periodic correction is applied after computing the recurrence value.
- Store each day result to avoid repeated recursion.

## Invariant

For every computed `solve(day)`:

- It stores the correct brightness for that day.
- The multiple-of-4 correction has already been applied if needed.
- The value is stored in `memo[day]` before returning.

## Algorithm (with one-line why at important steps)

1. If `day == 0`, return `1`.
   - Why: day `0` is a base value.
2. If `day == 1`, return `2`.
   - Why: day `1` is also a base value.
3. If `memo[day]` already exists, return it.
   - Why: this avoids recomputation.
4. Compute `solve(day - 1) + solve(day - 2)`.
   - Why: this is the recurrence rule.
5. If `day % 4 == 0`, subtract `1`.
   - Why: this is the periodic correction rule.
6. Store and return the value.
   - Why: future calls can reuse it.

## Mental Cheat Sheet

| Item | Meaning |
|---|---|
| Pattern | Memoized recurrence |
| State | `day` |
| Transition | `solve(day - 1) + solve(day - 2)` |
| Correction | subtract `1` if `day` is multiple of `4` |
| Base cases | day `0 = 1`, day `1 = 2` |
| Answer | `solve(n)` |

## Complexity

- Time Complexity: `O(n)`
- Space Complexity: `O(n)`
- Auxiliary Space: `O(n)` for memo table and recursion stack

Each day from `0` to `n` is computed once.

## Tiny Dry Run

For `n = 4`:

```text
day 0 = 1
day 1 = 2
day 2 = 1 + 2 = 3
day 3 = 2 + 3 = 5
day 4 = 3 + 5 - 1 = 7
```

Answer = `7`.

## Notes / Pitfalls

- Do not subtract `1` before adding the previous two values.
- Apply the correction only for multiples of `4`.
- Do not forget base cases for day `0` and day `1`.
- Use `BigInteger` because values grow quickly.
- A single `day` state is enough for this problem.
