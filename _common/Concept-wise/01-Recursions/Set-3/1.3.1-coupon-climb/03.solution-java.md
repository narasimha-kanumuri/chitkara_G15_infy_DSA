# Coupon Climb — Java Solution

## Language

Java

## Plain-English Implementation Walkthrough

1. We store the costs from index `1` to index `n`.  
   This makes step numbers match array indexes.

2. We start recursion from position `0`.  
   This means the agent is before the first step.

3. The method needs `position` because the next jumps depend on where the agent is now.

4. The method needs `couponUsed` because the cost choices depend on whether the coupon is still available.

5. If `position == n`, we return `0`.  
   The agent has reached the target, so no more cost is needed.

6. If `position > n`, we return `INF`.  
   This path is invalid because the agent crossed the target without landing exactly on it.

7. For each jump length `1` and `2`, we compute the next landing step.

8. If the next step is valid, we first try paying the step cost.

9. If the coupon is still unused, we also try using it on the next step.

10. We return the minimum value found because the problem asks for minimum total cost.

## Complete Solution Code

```java
import java.util.*;

public class Main {
    static final int INF = 1_000_000_000;

    static int solve(int position, int couponUsed, int n, int[] cost) {
        if (position == n) {
            return 0;
        }

        if (position > n) {
            return INF;
        }

        int answer = INF;

        for (int jump = 1; jump <= 2; jump++) {
            int next = position + jump;

            if (next > n) {
                continue;
            }

            int payCost = cost[next] + solve(next, couponUsed, n, cost);
            answer = Math.min(answer, payCost);

            if (couponUsed == 0) {
                int couponCost = solve(next, 1, n, cost);
                answer = Math.min(answer, couponCost);
            }
        }

        return answer;
    }

    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);

        int n = sc.nextInt();
        int[] cost = new int[n + 1];

        for (int i = 1; i <= n; i++) {
            cost[i] = sc.nextInt();
        }

        int result = solve(0, 0, n, cost);
        System.out.println(result);
    }
}
```

## Code Deconstruction

### `INF`

`INF` is used for impossible paths.  
If a recursive path jumps beyond the last step, it should never be chosen as the minimum.

### `solve(position, couponUsed, n, cost)`

This method returns the minimum extra cost from the current position to step `n`.

- `position` tells where the agent currently stands.
- `couponUsed` tells whether the coupon is already used.
- `n` is the target step.
- `cost` stores the landing costs.

### Base Cases

```java
if (position == n) return 0;
if (position > n) return INF;
```

The first case means success.  
The second case means invalid path.

### Jump Loop

```java
for (int jump = 1; jump <= 2; jump++)
```

This checks both allowed choices from the current position.

### Paying the Cost

```java
int payCost = cost[next] + solve(next, couponUsed, n, cost);
```

The agent lands on `next`, pays that step cost, and continues.

### Using the Coupon

```java
if (couponUsed == 0) {
    int couponCost = solve(next, 1, n, cost);
}
```

This is only allowed if the coupon has not been used before.  
After this choice, `couponUsed` becomes `1`.

## Language-Specific Notes

- Java arrays are `0`-indexed, but this solution uses steps from `1` to `n` for easier reading.
- Pure recursion may repeat many calls. This is intentional for Day 1 recursion teaching.
- For larger `n`, convert this to memoization using `dp[position][couponUsed]`.
- The value of `INF` is chosen large enough so invalid paths are never selected.

## Complexity Summary

- Time Complexity: `O(4^n)` for pure recursion because each landing decision may branch into jump and coupon choices.
- Space Complexity: `O(n)` due to recursion stack.
- Auxiliary Space: `O(n)` due to recursion stack.
