# Flag Route Optimizer — Quick Solution

## Idea (Simple)

We need to count all valid routes to reach step `n`.
At each move, we may jump `1`, `2`, or `3` steps.
But the same jump size cannot be used two times in a row.

So we use memoized recursion and remember the previous jump size.

## Pattern Recognition Clues

- We are asked to count all valid routes.
- Each position has repeated choices: jump `1`, `2`, or `3`.
- Future choices depend on the previous jump size.
- Repeated states suggest recursion with memoization.

## State Discovery

The first simple state is `ways(step)`.

But `ways(step)` is not enough.
It tells where we are, but not what jump was used just before this step.

That missing information matters because the next jump cannot equal the previous jump.

So we use:

```text
solve(step, lastJump)
```

Meaning:

```text
number of valid routes from current step to n,
when the previous jump size was lastJump
```

## Pattern

Top-Down DP with Last Choice State

## Core Insight

- Try jump sizes `1`, `2`, and `3`.
- Skip a jump if it is equal to `lastJump`.
- Cache each `(step, lastJump)` result.

## Invariant

For every state `solve(step, lastJump)`:

- `step` is the current position.
- `lastJump` is the jump used to reach this position.
- No route counted by this state has two equal consecutive jumps.

## Algorithm (with one-line why at important steps)

1. Start with `solve(0, 0)`.
   - Why: no jump has been used before the start.
2. If `step == n`, return `1`.
   - Why: one valid route reached the target exactly.
3. If `step > n`, return `0`.
   - Why: crossing the target is not valid.
4. If the state is cached, return the saved value.
   - Why: repeated states should be solved once.
5. For each jump in `{1, 2, 3}`, try it only if `jump != lastJump`.
   - Why: this enforces the no-consecutive-same-jump rule.
6. Add valid recursive answers, cache the total, and return it.
   - Why: the task asks for total count.

## Mental Cheat Sheet

| Item | Meaning |
|---|---|
| Pattern | Memoized recursion with last choice state |
| State | `(step, lastJump)` |
| Choices | jump `1`, `2`, or `3` |
| Restriction | current jump must not equal `lastJump` |
| Base case | `step == n` gives one route |
| Answer | `solve(0, 0)` |

## Complexity

- Time Complexity: `O(n)`
- Space Complexity: `O(n)`
- Auxiliary Space: `O(n)` for memo table and recursion stack

There are at most `4 * (n + 1)` states because `lastJump` can be `0`, `1`, `2`, or `3`.

## Tiny Dry Run

For `n = 3`:

```text
solve(0, 0)
├── jump 1 -> solve(1, 1)
│   └── jump 2 -> solve(3, 2) = 1
├── jump 2 -> solve(2, 2)
│   └── jump 1 -> solve(3, 1) = 1
└── jump 3 -> solve(3, 3) = 1
```

Total valid routes = `3`.

## Notes / Pitfalls

- Do not cache only by `step`.
- The previous jump must be part of the cache key.
- Do not allow `jump == lastJump`.
- Return `0` when `step > n`.
- Use `BigInteger` because the count can become large.
