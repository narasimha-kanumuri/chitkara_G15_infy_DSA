# Driftstone Memory Path

## Concepts Involved

Memoization, State Design, Multi-constraint DP, Optimization DP, Recursion to DP

## Problem Statement

A robot has to cross a row of `n` driftstones.

The height of stone `i` is given as `height[i]`.
The robot starts on stone `0` and wants to reach stone `n - 1`.

From stone `i`, the robot may normally jump to:

- stone `i + 1`
- stone `i + 2`

The energy cost of a normal jump from stone `i` to stone `j` is:

```text
abs(height[i] - height[j])
```

There are two extra rules:

1. The robot may use one special `3-step` jump at most once in the entire journey.
   The cost of this special jump is:

```text
2 * abs(height[i] - height[i + 3])
```

2. If the robot takes two `2-step` jumps consecutively, an extra fatigue penalty `p` is added to the second consecutive `2-step` jump.

Many recursive paths evaluate the same stone again with the same remaining special-jump status and previous-jump status.
Use memoization to avoid recomputation.

Return the minimum total energy required to reach the last stone.

## Input Format

- The first line contains two integers `n` and `p`.
- The second line contains `n` integers, where `height[i]` is the height of stone `i`.

## Output Format

- Print the minimum total energy required to reach stone `n - 1`.

## Constraints

- `1 <= n <= 2000`
- `0 <= p <= 10^9`
- `0 <= height[i] <= 10^9`
- The answer fits in a signed 64-bit integer.

## Examples

### Example 1

Input:

```text
5 10
10 30 20 40 15
```

Output:

```text
25
```

Explanation:

One optimal route is:

```text
0 -> 2 -> 4
```

The first jump is a `2-step` jump:

```text
abs(10 - 20) = 10
```

The second jump is also a `2-step` jump:

```text
abs(20 - 15) + penalty = 5 + 10 = 15
```

Total energy:

```text
10 + 15 = 25
```

### Example 2

Input:

```text
4 5
10 100 100 10
```

Output:

```text
0
```

Explanation:

The robot can use the special `3-step` jump directly:

```text
0 -> 3
```

The special jump cost is:

```text
2 * abs(10 - 10) = 0
```

### Example 3

Input:

```text
1 100
50
```

Output:

```text
0
```

Explanation:

The robot is already on the final stone.
No jump is needed.

- What should the answer be when `n = 1`?
- Why is the current index alone not enough as a memoization state?
- What information is needed to know whether a `3-step` jump is still allowed?
- What information is needed to know whether a `2-step` jump receives fatigue penalty?
- Why should a jump beyond the final stone not be considered?

The important idea is that the same stone can be reached under different conditions.
If the special jump is already used, the future choices change.
If the previous jump was a `2-step` jump, the next `2-step` jump becomes more expensive.
So both pieces of information must be part of the memoized state.
