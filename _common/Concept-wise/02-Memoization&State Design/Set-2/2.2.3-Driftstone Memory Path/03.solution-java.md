## Plain-English Implementation Walkthrough

1. We store heights in a `long[]` array.
   This avoids overflow when height values are large.

2. We create a recursive method named `solve`.
   It returns the minimum energy needed from the current stone.

3. The parameter `i` stores the current stone index.
   Future jump choices depend on this index.

4. The parameter `usedThree` stores whether the special `3-step` jump was already used.
   This is needed because that jump is allowed only once.

5. The parameter `prevTwo` stores whether the previous jump was a `2-step` jump.
   This is needed because a consecutive `2-step` jump gets penalty.

6. If `i == height.length - 1`, we return `0`.
   The robot has reached the final stone, so no more energy is needed.

7. We use `seen[i][usedThree][prevTwo]` before reading the memo value.
   This is safer because `0` can be a real minimum answer.

8. We first set `best` to a very large value.
   Then every valid jump can reduce it.

9. For a `1-step` jump, we add the normal height difference.
   The next state's `prevTwo` becomes `0`.

10. For a `2-step` jump, we add the normal height difference.
    If `prevTwo == 1`, we also add the fatigue penalty.

11. For a `3-step` jump, we allow it only when `usedThree == 0`.
    Its cost is double the height difference, and the next state's `usedThree` becomes `1`.

12. We save the best answer in the memo table and return it.
    This makes repeated states fast.

## Complete Solution Code

```java
import java.io.BufferedReader;
import java.io.InputStreamReader;
import java.util.StringTokenizer;

public class Main {

    private static final long INF = Long.MAX_VALUE / 4;

    private static long solve(int i, int usedThree, int prevTwo, long[] height, long penalty,
                              long[][][] memo, boolean[][][] seen) {
        if (i == height.length - 1) {
            return 0L;
        }

        if (seen[i][usedThree][prevTwo]) {
            return memo[i][usedThree][prevTwo];
        }

        long best = INF;
        int n = height.length;

        if (i + 1 < n) {
            long cost = Math.abs(height[i] - height[i + 1]);
            long total = cost + solve(i + 1, usedThree, 0, height, penalty, memo, seen);
            best = Math.min(best, total);
        }

        if (i + 2 < n) {
            long cost = Math.abs(height[i] - height[i + 2]);
            if (prevTwo == 1) {
                cost += penalty;
            }
            long total = cost + solve(i + 2, usedThree, 1, height, penalty, memo, seen);
            best = Math.min(best, total);
        }

        if (usedThree == 0 && i + 3 < n) {
            long cost = 2L * Math.abs(height[i] - height[i + 3]);
            long total = cost + solve(i + 3, 1, 0, height, penalty, memo, seen);
            best = Math.min(best, total);
        }

        seen[i][usedThree][prevTwo] = true;
        memo[i][usedThree][prevTwo] = best;
        return best;
    }

    private static long minimumEnergy(long[] height, long penalty) {
        int n = height.length;
        long[][][] memo = new long[n][2][2];
        boolean[][][] seen = new boolean[n][2][2];
        return solve(0, 0, 0, height, penalty, memo, seen);
    }

    public static void main(String[] args) throws Exception {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));

        StringTokenizer first = new StringTokenizer(br.readLine());
        int n = Integer.parseInt(first.nextToken());
        long penalty = Long.parseLong(first.nextToken());

        long[] height = new long[n];
        StringTokenizer second = new StringTokenizer(br.readLine());
        for (int i = 0; i < n; i++) {
            height[i] = Long.parseLong(second.nextToken());
        }

        System.out.println(minimumEnergy(height, penalty));
    }
}
```

## Code Deconstruction

### `INF`

```java
private static final long INF = Long.MAX_VALUE / 4;
```

This value acts like a very large answer.
It keeps minimum comparison safe and reduces overflow risk during addition.

### `solve` method

```java
private static long solve(int i, int usedThree, int prevTwo, long[] height, long penalty,
                          long[][][] memo, boolean[][][] seen)
```

This method represents one DP state.

- `i` is the current stone.
- `usedThree` tells whether the special jump is already used.
- `prevTwo` tells whether the previous jump was length `2`.
- `memo` stores answers.
- `seen` tells whether a state has already been computed.

### Base case

```java
if (i == height.length - 1) {
    return 0L;
}
```

The robot reached the last stone.
There is no remaining cost.

### One-step jump

```java
long total = cost + solve(i + 1, usedThree, 0, height, penalty, memo, seen);
```

A `1-step` jump does not create a consecutive `2-step` condition.
So the next state's `prevTwo` is `0`.

### Two-step jump

```java
if (prevTwo == 1) {
    cost += penalty;
}
```

This adds fatigue only when the current jump is `2` and the previous jump was also `2`.

### Special three-step jump

```java
if (usedThree == 0 && i + 3 < n) {
```

This block runs only if the special jump is still unused and the target stone exists.
After using it, the next state stores `usedThree = 1`.

### Saving result

```java
seen[i][usedThree][prevTwo] = true;
memo[i][usedThree][prevTwo] = best;
return best;
```

This stores the best answer for the state before returning.

## Language-Specific Notes

- Use `long` because energy totals can be large.
- Use a `seen` array because `0` can be a valid minimum value.
- `Math.abs(height[i] - height[j])` works here because `height` values are stored as `long`.
- The 3D array size is only `n * 2 * 2`, so it is still linear in `n`.
- Recursion depth can reach `n`; this is acceptable for practicing memoization, but tabulation can avoid stack depth.

## Complexity Summary

- Time Complexity: `O(n)`
- Space Complexity: `O(n)`
- Auxiliary Space: `O(n)`

There are at most `4n` states.
Each state tries at most three transitions.
