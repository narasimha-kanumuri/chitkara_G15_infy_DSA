## Idea (Simple)
We need the minimum energy to reach the last platform.
A jump costs height difference, and landing on a blue platform adds cold penalty.
The extra rule is that two `2`-platform jumps cannot happen consecutively.
So we must remember whether the previous jump was a `2`-jump.

## Pattern Recognition Clues
- We are minimizing total energy.
- Each platform can be reached from earlier platforms.
- Future movement depends on the previous jump length.
- This suggests bottom-up DP with a last-jump state.

## State Discovery
The first idea is `dp[i]`: minimum energy to reach platform `i`.
But `dp[i]` is not enough.
It forgets whether the last jump was `2`.
That information is needed because another `2`-jump may be forbidden.
So we split the state:
- `lastNotTwo[i]`: minimum energy to reach `i` when the last jump is not `2`
- `lastTwo[i]`: minimum energy to reach `i` when the last jump is `2`

## Pattern
Bottom-Up Min-Cost DP with Last Jump State

## Core Insight
- A `1`-platform jump can follow any previous jump.
- A `2`-platform jump can only follow a state where the last jump was not `2`.
- Landing cost must include cold penalty if the destination platform is blue.

## Invariant
For every platform `i`:
- `lastNotTwo[i]` stores the best cost to reach `i` with the latest jump not equal to `2`.
- `lastTwo[i]` stores the best cost to reach `i` with the latest jump equal to `2`.
Both states contain only valid paths that respect the no-consecutive-`2` rule.

## Algorithm (with one-line why at important steps)
1. Create two DP arrays: `lastNotTwo` and `lastTwo`.
   - Why: the next valid jump depends on whether the previous jump was `2`.
2. Set `lastNotTwo[1] = 0`.
   - Why: the climber starts at platform `1` with no previous `2`-jump.
3. For each platform `i`, compute a `1`-jump from `i - 1`.
   - Why: a `1`-jump can follow both states.
4. Add height difference and blue landing penalty for platform `i`.
   - Why: cost is paid when the climber lands on the destination platform.
5. For each platform `i >= 3`, compute a `2`-jump from `i - 2` only from `lastNotTwo[i - 2]`.
   - Why: this prevents two consecutive `2`-jumps.
6. Return `min(lastNotTwo[n], lastTwo[n])`.
   - Why: the final jump may be either `1` or `2`.

## Mental Cheat Sheet
| Item | Meaning |
|---|---|
| Pattern | Min-cost DP with last-jump state |
| Simple failed state | `dp[i]` forgets whether last jump was `2` |
| `lastNotTwo[i]` | best cost reaching `i` without ending by `2` |
| `lastTwo[i]` | best cost reaching `i` ending by `2` |
| Landing penalty | add `c` if destination is blue |
| Answer | `min(lastNotTwo[n], lastTwo[n])` |

## Complexity
- Time Complexity: `O(n)`
- Space Complexity: `O(n)`
- Auxiliary Space: `O(n)`

## Tiny Dry Run
For `height = [10, 30, 40, 20, 50]`, `blue = [0, 0, 1, 0, 0]`, `c = 5`:
- A route `1 -> 2 -> 3 -> 5` costs `20 + 15 + 10 = 45`.
- Platform `3` adds cold penalty `5`.
- The route `1 -> 3 -> 5` is not allowed because it uses two consecutive `2`-jumps.
- Answer = `45`.

## Notes / Pitfalls
- Do not add cold penalty for the starting platform.
- Do not allow `lastTwo[i]` to come from `lastTwo[i - 2]`.
- A `1`-jump resets the consecutive long-jump risk.
- Use `long` for large height differences and penalties.
