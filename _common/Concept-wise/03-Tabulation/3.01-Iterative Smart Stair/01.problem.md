# 3 — Tabulation (Bottom-Up DP)

## problem.md

### Title

Iterative Smart Stair

### Concepts Involved

Tabulation, State Tracking, Counting DP

### Problem Statement

A climber wants to reach step `n`.

On each move, the climber may go up either `1` step or `2` steps.

However, two `2`-step moves are not allowed to happen consecutively.

Find the total number of valid ways to reach exactly step `n`.

A valid way is defined only by the sequence of jumps.

Two ways are different if at least one jump length differs at some position.

### Input Format

- A single integer `n`

### Output Format

- Print the total number of valid ways to reach exactly step `n`

### Constraints

- `1 <= n <= 2000`

### Examples

#### Example 1

Input:

`3`

Output:

`3`

#### Example 2

Input:

`4`

Output:

`4`

#### Example 3

Input:

`5`

Output:

`6`

### Edge-Case Thinking Prompts

- What happens when `n = 1`?

- What changes when `n = 2`?

- Why is the sequence `2, 2` invalid?

- How should the first `2`-step move be treated?

### Thinking Like a Programmer

- What small piece of information must be remembered at every step?

- Is it enough to know only the current step number?

- Can you count ways separately based on the last move used?

- How can smaller answers help build bigger answers?
