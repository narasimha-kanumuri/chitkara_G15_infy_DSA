## Plain-English Implementation Walkthrough

1. We store checkpoint tolls in a `long[] cost` array.
   This keeps the input values easy to access by index.

2. We use `long` because each toll can be large.
   The total cost may not fit in `int`.

3. We create a recursive method `solve(i, used, cost, memo, seen)`.
   It returns the minimum toll needed from checkpoint `i` onward.

4. The parameter `i` stores the current checkpoint.
   This is the main position in the journey.

5. The parameter `used` stores whether the coupon has already been used.
   This is needed because the coupon can be used only once.

6. If `i >= cost.length`, we return `0`.
   The agent has moved beyond the final checkpoint.

7. We check `seen[i][used]` before using `memo[i][used]`.
   This is needed because `0` can be a valid answer.

8. For the pay option, we add `cost[i]`.
   Then we choose the cheaper future path from `i + 1` or `i + 2`.

9. For the coupon option, we use it only when `used == 0`.
   The current checkpoint contributes `0`, and the next state gets `used = 1`.

10. We compare the pay option and coupon option.
    The smaller one is the best answer for this state.

11. We save the answer in `memo[i][used]`.
    This avoids recomputation when the same state appears again.

12. The final call is `solve(0, 0, cost, memo, seen)`.
    It starts at checkpoint `0` with the coupon unused.

## Complete Solution Code

```java
import java.io.BufferedReader;
import java.io.InputStreamReader;
import java.util.StringTokenizer;

public class Main {

    private static long solve(int i, int used, long[] cost, long[][] memo, boolean[][] seen) {
        if (i >= cost.length) {
            return 0L;
        }

        if (seen[i][used]) {
            return memo[i][used];
        }

        long nextAfterPay = Math.min(
            solve(i + 1, used, cost, memo, seen),
            solve(i + 2, used, cost, memo, seen)
        );
        long best = cost[i] + nextAfterPay;

        if (used == 0) {
            long nextAfterCoupon = Math.min(
                solve(i + 1, 1, cost, memo, seen),
                solve(i + 2, 1, cost, memo, seen)
            );
            best = Math.min(best, nextAfterCoupon);
        }

        seen[i][used] = true;
        memo[i][used] = best;
        return best;
    }

    private static long minimumToll(long[] cost) {
        int n = cost.length;
        long[][] memo = new long[n][2];
        boolean[][] seen = new boolean[n][2];
        return solve(0, 0, cost, memo, seen);
    }

    public static void main(String[] args) throws Exception {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));

        int n = Integer.parseInt(br.readLine().trim());
        long[] cost = new long[n];

        StringTokenizer st = new StringTokenizer(br.readLine());
        for (int i = 0; i < n; i++) {
            cost[i] = Long.parseLong(st.nextToken());
        }

        System.out.println(minimumToll(cost));
    }
}
```

## Code Deconstruction

### `solve` method

```java
private static long solve(int i, int used, long[] cost, long[][] memo, boolean[][] seen)
```

This method represents one memoized DP state.

- `i` is the current checkpoint.
- `used` tells whether the coupon has already been used.
- `cost` stores all checkpoint tolls.
- `memo` stores computed minimum values.
- `seen` marks states that have already been solved.

### Journey complete case

```java
if (i >= cost.length) {
    return 0L;
}
```

When the agent moves beyond the last checkpoint, no more toll is paid.

### Memoization check

```java
if (seen[i][used]) {
    return memo[i][used];
}
```

This avoids solving the same `(i, used)` state again.

### Paying current toll

```java
long best = cost[i] + nextAfterPay;
```

This option pays the current checkpoint toll.
Then it takes the cheaper of the two next possible positions.

### Using the coupon

```java
if (used == 0) {
    ...
    best = Math.min(best, nextAfterCoupon);
}
```

This block runs only if the coupon is still unused.
The current checkpoint becomes free, and future states receive `used = 1`.

### Saving and returning

```java
seen[i][used] = true;
memo[i][used] = best;
return best;
```

The computed answer is saved before it is returned.

## Language-Specific Notes

- Use `long` because toll totals can be large.
- Use `seen` because `0` can be a valid minimum answer.
- `StringTokenizer` is used to read the toll array cleanly.
- The recursion depth can reach `n`; this is acceptable for memoization practice.
- The coupon state is represented as `0` or `1` so it can index the memo table directly.

## Complexity Summary

- Time Complexity: `O(n)`
- Space Complexity: `O(n)`
- Auxiliary Space: `O(n)`

There are `2 * n` states.
Each state checks a constant number of transitions.
