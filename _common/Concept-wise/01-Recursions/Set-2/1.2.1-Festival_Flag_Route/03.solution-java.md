# Festival Flag Route — Java Solution

## Language
Java

## Plain-English Implementation Walkthrough
1. We read `n` because the target step is the only input.
   Related code: `int n = sc.nextInt();`

2. We call `countWays(n, 0)` because all `n` steps are still remaining.
   The previous jump is `0` because no jump has happened yet.

3. Inside the recursive method, if `remaining == 0`, we return `1`.
   This means the current route reached the target exactly.

4. If `remaining < 0`, we return `0`.
   This means the route jumped beyond the target.

5. We try jumps `1`, `2`, and `3` one by one.
   These are the only jump sizes allowed in the problem.

6. If the chosen jump equals `lastJump`, we skip it.
   This keeps the rule that the same jump cannot appear consecutively.

7. For every valid jump, we call the same method with fewer remaining steps.
   We also pass the current jump as the new previous jump.

8. We add all valid branch answers into `total`.
   We return `total` because it stores all valid routes from this state.

## Complete Solution Code
```java
import java.util.Scanner;

public class Main {
    static long countWays(int remaining, int lastJump) {
        if (remaining == 0) {
            return 1;
        }

        if (remaining < 0) {
            return 0;
        }

        long total = 0;

        for (int jump = 1; jump <= 3; jump++) {
            if (jump == lastJump) {
                continue;
            }

            total += countWays(remaining - jump, jump);
        }

        return total;
    }

    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        int n = sc.nextInt();
        System.out.println(countWays(n, 0));
        sc.close();
    }
}
```

## Code Deconstruction
- `countWays(int remaining, int lastJump)` is the recursive state.
  - `remaining` tells how many more steps are needed.
  - `lastJump` tells what jump was used just before this state.

- `remaining == 0` is the successful base case.
  - One valid complete route has been found.

- `remaining < 0` is the invalid base case.
  - The route crossed the target.

- The `for` loop represents choice expansion.
  - It tries all possible jumps: `1`, `2`, and `3`.

- `if (jump == lastJump)` propagates the constraint.
  - It blocks repeated consecutive jump sizes.

- `total += countWays(remaining - jump, jump)` counts valid routes after taking the current jump.
  - The new previous jump becomes the current jump.

## Test Cases

## Language-Specific Notes
- `long` is safer than `int` for counting routes.
- This is a pure recursive solution, so it may repeat work.
- The constraint limit is kept small because Day 1 focuses on recursion, not memoization.
- A later memoized version can store answers for `(remaining, lastJump)`.

## Complexity Summary
- Time Complexity: `O(3^n)` worst case.
- Space Complexity: `O(n)` recursion stack.
- Auxiliary Space: `O(n)` recursion stack.
