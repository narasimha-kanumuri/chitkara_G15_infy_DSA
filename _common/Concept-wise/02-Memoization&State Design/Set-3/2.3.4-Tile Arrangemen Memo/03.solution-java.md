## Plain-English Implementation Walkthrough

1. We use `BigInteger` because the number of valid tilings can become very large.

2. We create a method `solve(remaining, prevTwo, memo)`.
   It returns the number of valid ways to cover the remaining corridor length.

3. The parameter `remaining` tells how much corridor is still uncovered.
   It decreases when we place a tile.

4. The parameter `prevTwo` tells whether the previous tile was length `2`.
   This is needed because another length-`2` tile may be blocked.

5. If `remaining == 0`, the corridor is fully covered.
   We return `1` only when `prevTwo == 0` because the final tile must be length `1`.

6. If `remaining < 0`, we return `0`.
   This means the chosen tile crossed the corridor length.

7. We check `memo[remaining][prevTwo]` before solving the state.
   If it is already stored, we return it directly.

8. We always try placing a length-`1` tile.
   After placing it, the next state has `prevTwo = 0`.

9. We try placing a length-`2` tile only when `prevTwo == 0`.
   This prevents two length-`2` tiles from being placed next to each other.

10. We add the counts from valid choices.
    This works because the problem asks for the total number of valid tilings.

11. We save the total in `memo[remaining][prevTwo]`.
    This makes repeated states fast.

12. The final answer is `solve(n, 0, memo)`.
    At the start, no previous tile exists, so it is safe to treat `prevTwo` as `0`.

## Complete Solution Code

```java
import java.io.BufferedReader;
import java.io.InputStreamReader;
import java.math.BigInteger;

public class Main {

    private static BigInteger solve(int remaining, int prevTwo, BigInteger[][] memo) {
        if (remaining == 0) {
            return prevTwo == 0 ? BigInteger.ONE : BigInteger.ZERO;
        }

        if (remaining < 0) {
            return BigInteger.ZERO;
        }

        if (memo[remaining][prevTwo] != null) {
            return memo[remaining][prevTwo];
        }

        BigInteger total = BigInteger.ZERO;

        // Place a tile of length 1. It is always allowed.
        total = total.add(solve(remaining - 1, 0, memo));

        // Place a tile of length 2 only if the previous tile was not length 2.
        if (prevTwo == 0) {
            total = total.add(solve(remaining - 2, 1, memo));
        }

        memo[remaining][prevTwo] = total;
        return total;
    }

    private static BigInteger countTilings(int n) {
        BigInteger[][] memo = new BigInteger[n + 1][2];
        return solve(n, 0, memo);
    }

    public static void main(String[] args) throws Exception {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        int n = Integer.parseInt(br.readLine().trim());
        System.out.println(countTilings(n));
    }
}
```

## Code Deconstruction

### `solve` method

```java
private static BigInteger solve(int remaining, int prevTwo, BigInteger[][] memo)
```

This method represents one memoized state.

- `remaining` is the corridor length still uncovered.
- `prevTwo` tells whether the previous tile was length `2`.
- `memo` stores already computed answers.

### Fully covered case

```java
if (remaining == 0) {
    return prevTwo == 0 ? BigInteger.ONE : BigInteger.ZERO;
}
```

This line enforces the final-tile rule.
If the last tile was length `1`, the tiling is counted.
If the last tile was length `2`, the tiling is rejected.

### Crossed length case

```java
if (remaining < 0) {
    return BigInteger.ZERO;
}
```

This rejects choices that cover more than the corridor length.

### Memoization check

```java
if (memo[remaining][prevTwo] != null) {
    return memo[remaining][prevTwo];
}
```

This avoids solving the same state again.

### Placing tile length `1`

```java
total = total.add(solve(remaining - 1, 0, memo));
```

A length-`1` tile is always allowed.
It also makes the latest tile not length `2`.

### Placing tile length `2`

```java
if (prevTwo == 0) {
    total = total.add(solve(remaining - 2, 1, memo));
}
```

A length-`2` tile is allowed only when the previous tile was not length `2`.
After placing it, the next state has `prevTwo = 1`.

### Saving result

```java
memo[remaining][prevTwo] = total;
return total;
```

The computed answer is saved before returning.

## Language-Specific Notes

- `BigInteger` is immutable. Use `.add()` to combine counts.
- Do not use `+` for `BigInteger` addition.
- `memo[remaining][prevTwo] == null` means the state has not been solved yet.
- `prevTwo` is stored as `0` or `1` so it can index the memo table directly.
- The recursion depth can reach `n`; this is acceptable for memoization practice, but tabulation can avoid stack depth.

## Complexity Summary

- Time Complexity: `O(n)`
- Space Complexity: `O(n)`
- Auxiliary Space: `O(n)`

There are at most `2 * (n + 1)` states.
Each state performs constant work, except for `BigInteger` digit cost.
