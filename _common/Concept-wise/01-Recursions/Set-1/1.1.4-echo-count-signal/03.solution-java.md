## Plain-English Implementation Walkthrough

1. We read the target level `n`.
   The program needs only this one input value.
   Related code: `int n = sc.nextInt();`

2. We start from level `0`.
   No move has been taken, so the path has zero trailing `+1` moves.
   Related code: `solve(0, 0, n)`

3. The recursive method receives the current level and the ending information.
   We need ending information because the final rule checks the last two moves.
   Related code: `solve(int level, int endingOnes, int n)`

4. If `level > n`, the path has crossed the target.
   This path cannot become valid, so we return `0`.
   Related code: `if (level > n) return 0;`

5. If `level == n`, the path has reached the target exactly.
   Now we check whether the ending is allowed.
   Related code: `if (level == n) { ... }`

6. If `endingOnes >= 2`, the sequence ends with two `+1` moves.
   This violates the rule, so we return `0`.
   Related code: `return endingOnes >= 2 ? 0 : 1;`

7. For the `+1` choice, we move to `level + 1`.
   The number of trailing `+1` moves increases by one.
   Related code: `Math.min(2, endingOnes + 1)`

8. We cap the trailing `+1` count at `2`.
   We only need to know whether it has reached two or not.
   Related code: `Math.min(2, endingOnes + 1)`

9. For the `+2` choice, we move to `level + 2`.
   A `+2` move breaks the ending chain of `+1` moves.
   Related code: `solve(level + 2, 0, n)`

10. We add the results of both choices.
    This is correct because each choice creates different construction sequences.
    Related code: `return takeOne + takeTwo;`

## Complete Solution Code

```java
import java.util.Scanner;

public class Main {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);

        int n = sc.nextInt();

        long answer = solve(0, 0, n);
        System.out.println(answer);

        sc.close();
    }

    private static long solve(int level, int endingOnes, int n) {
        if (level > n) {
            return 0;
        }

        if (level == n) {
            return endingOnes >= 2 ? 0 : 1;
        }

        long takeOne = solve(level + 1, Math.min(2, endingOnes + 1), n);
        long takeTwo = solve(level + 2, 0, n);

        return takeOne + takeTwo;
    }
}
```

## Code Deconstruction

- `main` reads the target level and prints the answer.
- `solve(0, 0, n)` starts the recursive search from level `0`.
- `level` tells where the signal currently is.
- `endingOnes` tells how many consecutive `+1` moves are at the end of the current sequence.
- `level > n` rejects paths that overshoot the target.
- `level == n` is the exact target base case.
- `endingOnes >= 2 ? 0 : 1` applies the final stability rule.
- `takeOne` represents choosing a `+1` increase.
- `takeTwo` represents choosing a `+2` increase.
- `takeOne + takeTwo` counts all valid constructions from both recursive branches.

## Language-Specific Notes

- `long` is used because counts can grow as `n` increases.
- `Math.min(2, endingOnes + 1)` keeps the state small and readable.
- This solution intentionally uses plain recursion because Day 1 focuses on recursion thinking.
- The same states can repeat many times. Day 2 can optimize this using memoization with state `(level, endingOnes)`.
- The maximum recursion depth is at most `n`, because the smallest move is `+1`.

## Complexity Summary

- Time Complexity: `O(2^n)` in the worst case without memoization
- Space Complexity: `O(n)` because the deepest recursion path can use `n` calls
- Auxiliary Space: `O(n)` for the recursion call stack
