## Plain-English Implementation Walkthrough
1. We store the modulo in `MOD` because the number of schedules can become very large.

2. We read `n` and `m` first.
   `n` gives the schedule length.
   `m` tells how many maintenance positions follow.

3. We create a `boolean[] blocked` array of size `n + 1`.
   We use `n + 1` because positions are 1-indexed.

4. For each maintenance position, we set `blocked[position] = true`.
   This lets the DP check blocked positions in constant time.

5. We create two DP arrays: `endWithZero` and `endWithOne`.
   We need two arrays because the next decision depends on the last bit.

6. For position `1`, `endWithZero[1] = 1`.
   The one-bit schedule `0` is always valid.

7. For position `1`, `endWithOne[1]` is `1` only if position `1` is not blocked.
   If it is blocked, placing `1` there is not allowed.

8. For each later position, we compute schedules ending with `0` from both previous states.
   This is valid because `0` can follow either `0` or `1`.

9. For schedules ending with `1`, we first check whether the current position is blocked.
   If it is blocked, the count must be `0`.
   Otherwise, `1` can come only after a previous `0`.

10. We return `endWithZero[n]` because the final schedule must end with `0`.

## Complete Solution Code
```java
import java.io.BufferedInputStream;
import java.io.IOException;

public class Main {
    private static final long MOD = 1_000_000_007L;

    private static long countSchedules(int n, boolean[] blocked) {
        long[] endWithZero = new long[n + 1];
        long[] endWithOne = new long[n + 1];

        endWithZero[1] = 1L;
        endWithOne[1] = blocked[1] ? 0L : 1L;

        for (int i = 2; i <= n; i++) {
            endWithZero[i] = (endWithZero[i - 1] + endWithOne[i - 1]) % MOD;

            if (blocked[i]) {
                endWithOne[i] = 0L;
            } else {
                endWithOne[i] = endWithZero[i - 1];
            }
        }

        return endWithZero[n];
    }

    public static void main(String[] args) throws Exception {
        FastScanner fs = new FastScanner();

        int n = fs.nextInt();
        int m = fs.nextInt();

        boolean[] blocked = new boolean[n + 1];
        for (int i = 0; i < m; i++) {
            int position = fs.nextInt();
            blocked[position] = true;
        }

        System.out.println(countSchedules(n, blocked));
    }

    private static class FastScanner {
        private final BufferedInputStream in = new BufferedInputStream(System.in);
        private final byte[] buffer = new byte[1 << 16];
        private int ptr = 0;
        private int len = 0;

        private int read() throws IOException {
            if (ptr >= len) {
                len = in.read(buffer);
                ptr = 0;
                if (len <= 0) return -1;
            }
            return buffer[ptr++];
        }

        int nextInt() throws IOException {
            int c;
            do {
                c = read();
            } while (c <= ' ' && c != -1);

            int sign = 1;
            if (c == '-') {
                sign = -1;
                c = read();
            }

            int value = 0;
            while (c > ' ') {
                value = value * 10 + (c - '0');
                c = read();
            }

            return value * sign;
        }
    }
}
```

## Code Deconstruction
- `MOD`
  - Stores `1_000_000_007` as required by the problem.
  - It is used after addition to keep the answer within range.

- `countSchedules(int n, boolean[] blocked)`
  - Contains the main DP logic.
  - It receives the schedule length and blocked-position information.

- `endWithZero[i]`
  - Counts valid schedules of length `i` ending with `0`.

- `endWithOne[i]`
  - Counts valid schedules of length `i` ending with `1`.

- `endWithZero[i] = endWithZero[i - 1] + endWithOne[i - 1]`
  - A `0` can follow any valid previous ending.

- `endWithOne[i] = endWithZero[i - 1]`
  - A `1` can follow only a previous `0`.

- `if (blocked[i]) endWithOne[i] = 0`
  - A blocked position cannot contain `1`.

- `return endWithZero[n]`
  - The schedule must end with `0`, so only this state is the answer.

## Language-Specific Notes
- `long` is used because counts may grow before modulo is applied.
- Arrays are 1-indexed to match input positions directly.
- `FastScanner` is useful for large input.
- A blocked final position is still allowed to end with `0`; it only blocks `1`.

## Complexity Summary
- Time Complexity: `O(n + m)`
- Space Complexity: `O(n)`
- Auxiliary Space: `O(n)`
