# Festival Flag Route — Quick Solution

## Idea (Simple)
We count routes recursively.

At every step, we may choose jump `1`, `2`, or `3`.

But the current jump must not be equal to the previous jump.

So the recursion must remember two things: how many steps are still left and what jump was used last.

## Pattern Recognition Clues
- We are asked to count all valid routes.
- Each route is built by making one jump and then solving a smaller remaining problem.
- The next choice depends on the previous jump.
- This suggests counting recursion with an extra previous-choice state.

## State Discovery
The first idea is `countWays(remaining)`.

But this is not enough.

It tells how many steps are left, but it forgets the previous jump.

The rule says the same jump size cannot be used consecutively.

So the next valid choices depend on the last jump.

Final state:
- `remaining`: how many steps are still needed
- `lastJump`: the jump size used in the previous move

We start with `lastJump = 0` because no jump has been taken yet.

## Pattern
Counting Recursion with Previous Choice State

## Core Insight
- A route becomes invalid only when the current jump equals the previous jump.
- If a jump is valid, the remaining problem is the same type of problem with fewer steps.
- Each valid recursive branch contributes to the total count.

## Invariant
For every recursive call `countWays(remaining, lastJump)`, the path used to reach this call has no two equal consecutive jumps.

The function returns the number of valid ways to finish from this state.

## Algorithm (with one-line why at important steps)
1. If `remaining == 0`, return `1`.
   - Why: one complete valid route has been formed.
2. If `remaining < 0`, return `0`.
   - Why: the route crossed the target and cannot be used.
3. Try each jump from `1` to `3`.
   - Why: these are the only allowed move sizes.
4. Skip the jump if it equals `lastJump`.
   - Why: consecutive equal jumps are forbidden.
5. Recursively count ways after taking the jump.
   - Why: the remaining path is a smaller version of the same problem.
6. Add all returned counts.
   - Why: each branch gives a different valid route.

## Mental Cheat Sheet
| Item | Meaning |
|---|---|
| Pattern | Counting recursion with previous choice |
| State | `(remaining, lastJump)` |
| Choice | Try jump `1`, `2`, or `3` |
| Invalid choice | `jump == lastJump` |
| Complete route | `remaining == 0` |
| Answer | `countWays(n, 0)` |

## Complexity
- Time Complexity: `O(3^n)` in the worst case.
- Space Complexity: `O(n)` because the recursion stack can go as deep as `n`.
- Auxiliary Space: `O(n)` recursion stack.

## Tiny Dry Run
For `n = 3`:

- Start: `remaining = 3`, `lastJump = 0`
- Try `1`, then remaining becomes `2`, last jump becomes `1`
- From there, jump `1` is blocked, but jump `2` works: route `1 + 2`
- Try `2`, then jump `1` works: route `2 + 1`
- Try `3`: route `3`

Answer = `3`.

## Notes / Pitfalls
- Do not use only `remaining`; it forgets the previous jump.
- Do not allow two equal jump sizes next to each other.
- Return `1` for `remaining == 0`, not `0`.
- Use `long` in Java because counts can grow.
