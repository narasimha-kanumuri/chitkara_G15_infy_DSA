## Idea (Simple)
We need the minimum energy to reach the last stone.
The robot can jump `1` or `2` stones normally.
It can also use one special `3`-stone jump once.
The cost of a `2`-stone jump depends on whether the previous jump was also `2`, so we must remember extra state.

## Pattern Recognition Clues
- We are minimizing total cost.
- Each stone can be reached from earlier stones.
- Future cost depends on the last jump length.
- One special jump can be used only once, so a used/not-used state is needed.

## State Discovery
The first idea is `dp[i]`: minimum energy to reach stone `i`.
But `dp[i]` is not enough.
It forgets whether the previous jump was `2`, which affects fatigue.
It also forgets whether the special `3`-stone jump has already been used.
So we use:
- stone index
- whether the last jump was `2`
- whether the special `3`-stone jump was used

## Pattern
Bottom-Up DP with Multi-constraint State

## Core Insight
- A `1`-stone jump resets the last-jump-is-2 state.
- A `2`-stone jump sets the last-jump-is-2 state and may add penalty.
- A `3`-stone jump is allowed only if the special jump is unused.

## Invariant
For every stone `i`, last-jump flag, and special-jump flag, the DP value stores the minimum energy to reach exactly that state after considering all valid earlier jumps.

## Algorithm (with one-line why at important steps)
1. Create `dp[stone][lastTwo][usedThree]` and fill it with `INF`.
   - Why: each constraint affects future transition cost.
2. Set `dp[1][0][0] = 0`.
   - Why: the robot starts at stone `1` with no previous long jump and special jump unused.
3. From every reachable state, try a `1`-stone jump.
   - Why: it is always allowed if the target stone exists.
4. Try a `2`-stone jump and add penalty only when `lastTwo = 1`.
   - Why: fatigue happens only for consecutive `2`-stone jumps.
5. Try a `3`-stone jump only when `usedThree = 0`.
   - Why: the special jump can be used at most once.
6. Take the minimum among all states at stone `n`.
   - Why: the journey may end with any last-jump and special-jump status.

## Mental Cheat Sheet
| Item | Meaning |
|---|---|
| Pattern | Min-cost DP with extra state |
| Simple failed state | `dp[i]` forgets fatigue and special jump usage |
| State | `(stone, last jump was 2?, special 3-jump used?)` |
| `1` jump | normal cost, lastTwo becomes `0` |
| `2` jump | normal cost, add penalty if previous was `2` |
| `3` jump | double cost, allowed once |
| Answer | minimum over all states at stone `n` |

## Complexity
- Time Complexity: `O(n)`
- Space Complexity: `O(n)`
- Auxiliary Space: `O(n)`

## Tiny Dry Run
For `height = [1, 100, 2, 100, 3, 4]`, `p = 10`:
- Jump `1 -> 3`: cost `1`, last jump becomes `2`.
- Jump `3 -> 6`: use special `3`-stone jump, cost `2 * |2 - 4| = 4`.
- Total = `5`.
- Another valid route may be close, so DP checks all states and returns the minimum.

## Notes / Pitfalls
- Do not use only one DP value per stone.
- A `3`-stone jump is not a `2`-stone jump for fatigue.
- Add fatigue penalty only to the second consecutive `2`-stone jump.
- Use `long` because energy values can become large.
