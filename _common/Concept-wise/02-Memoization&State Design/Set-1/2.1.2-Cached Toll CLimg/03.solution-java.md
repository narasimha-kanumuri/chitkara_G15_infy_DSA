## Plain-English Implementation Walkthrough

1. We store all toll values in the `cost` array.
   The index of this array is the step number.

2. We use `solve(i, used, cost, memo, seen)` to represent one state.
   This state means: “minimum cost to finish from step `i`.”

3. The variable `used` tells whether the coupon has already been used.
   `0` means unused.
   `1` means already used.

4. If `i >= cost.length`, the courier has crossed the staircase.
   So the function returns `0` because no more toll is paid.

5. We use `seen[i][used]` before reading `memo[i][used]`.
   This is safer because a valid minimum cost can also be `0`.

6. The first option is to pay the current step cost.
   After paying, the courier may jump to `i + 1` or `i + 2`.

7. If the coupon is unused, the second option is to use it on the current step.
   Then the current step contributes `0`, and `used` becomes `1`.

8. We keep the smaller value between paying and using the coupon.
   This matches the goal of minimizing total toll.

9. We save the result in `memo[i][used]`.
   If the same state appears again, the answer is returned directly.

10. The final answer is `solve(0, 0, cost, memo, seen)`.
    This starts at step `0` with the coupon unused.

## Complete Solution Code

```java
import java.io.BufferedReader;
import java.io.InputStreamReader;
import java.util.StringTokenizer;

public class Main {

    private static long solve(int i, int used, long[] cost, long[][] memo, boolean[][] seen) {
        if (i >= cost.length) {
            return 0L;
        }

        if (seen[i][used]) {
            return memo[i][used];
        }

        long nextAfterPay = Math.min(
            solve(i + 1, used, cost, memo, seen),
            solve(i + 2, used, cost, memo, seen)
        );
        long best = cost[i] + nextAfterPay;

        if (used == 0) {
            long nextAfterCoupon = Math.min(
                solve(i + 1, 1, cost, memo, seen),
                solve(i + 2, 1, cost, memo, seen)
            );
            best = Math.min(best, nextAfterCoupon);
        }

        seen[i][used] = true;
        memo[i][used] = best;
        return best;
    }

    private static long minimumToll(long[] cost) {
        int n = cost.length;
        long[][] memo = new long[n][2];
        boolean[][] seen = new boolean[n][2];
        return solve(0, 0, cost, memo, seen);
    }

    public static void main(String[] args) throws Exception {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));

        int n = Integer.parseInt(br.readLine().trim());
        long[] cost = new long[n];

        StringTokenizer st = new StringTokenizer(br.readLine());
        for (int i = 0; i < n; i++) {
            cost[i] = Long.parseLong(st.nextToken());
        }

        System.out.println(minimumToll(cost));
    }
}
```

## Code Deconstruction

### `solve` method

```java
private static long solve(int i, int used, long[] cost, long[][] memo, boolean[][] seen)
```

This method solves one memoized DP state.

- `i` is the current step.
- `used` tells whether the coupon has been used.
- `cost` stores all step tolls.
- `memo` stores computed answers.
- `seen` marks states that are already computed.

### Base case

```java
if (i >= cost.length) {
    return 0L;
}
```

If the courier moves beyond the last step, the climb is complete.
No extra toll is added.

### Memo check

```java
if (seen[i][used]) {
    return memo[i][used];
}
```

This avoids repeated calculation.
The same `(i, used)` state may be reached by different routes.

### Paying the current toll

```java
long best = cost[i] + nextAfterPay;
```

This option pays the current step toll.
Then it chooses the cheaper future route from `i + 1` or `i + 2`.

### Using the coupon

```java
if (used == 0) {
    ...
    best = Math.min(best, nextAfterCoupon);
}
```

This block runs only when the coupon is still available.
The current step cost becomes `0`, and future states receive `used = 1`.

### Saving and returning

```java
seen[i][used] = true;
memo[i][used] = best;
return best;
```

The result is stored so future calls can reuse it.

## Language-Specific Notes

- Use `long` because total cost can exceed the range of `int`.
- Use a `seen` array because `0` can be a real answer after using the coupon.
- `StringTokenizer` assumes all costs are available in the second input line.
- The recursion depth can be `O(n)`. This is acceptable for the given teaching constraint, but tabulation can avoid recursion stack usage.
- The coupon state is represented with `0` and `1` to make array indexing simple.

## Complexity Summary

- Time Complexity: `O(n)`
- Space Complexity: `O(n)`
- Auxiliary Space: `O(n)`

There are `2 * n` states.
Each state computes a constant number of transitions.
