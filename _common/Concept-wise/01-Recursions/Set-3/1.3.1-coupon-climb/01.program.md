# Coupon Climb

## Concepts Involved

- Base Case Design
- Return Meaning
- Choice Expansion
- Recursive State Definition
- Recursion Tree Analysis
- Constraint Propagation
- Counting vs Optimization Recursion
- Optimization Recursion

## Problem Statement

A delivery agent has to climb a staircase with `n` steps.  
Each step has a positive cost. When the agent lands on a step, that step's cost is added to the total cost.

The agent may move either `1` step or `2` steps at a time.  
The agent also has exactly one coupon. The coupon may be used at most once to cancel the cost of one landed step.

The agent starts before step `1`.  
The agent must reach exactly step `n`.

Return the minimum total cost needed to reach step `n`.

If the coupon is not useful, the agent may finish without using it.

## Input Format

- The first line contains an integer `n`.
- The second line contains `n` integers: `cost[1], cost[2], ..., cost[n]`.

## Output Format

Print one integer: the minimum total cost to reach step `n`.

## Constraints

- `1 <= n <= 30`
- `1 <= cost[i] <= 10^5`
- The intended solution for this file is recursive, so the constraint is kept small enough for recursion-based teaching.

## Examples

### Example 1

**Input**

```text
5
8 3 6 2 10
```

**Output**

```text
5
```

**Explanation**

One good path is:

- Land on step `2`, pay `3`.
- Land on step `4`, pay `2`.
- Land on step `5`, use the coupon, pay `0`.

Total cost = `3 + 2 + 0 = 5`.

### Example 2

**Input**

```text
3
5 100 6
```

**Output**

```text
5
```

**Explanation**

The best path is:

- Land on step `1`, pay `5`.
- Jump to step `3`, use the coupon, pay `0`.

Total cost = `5`.

This is a trap case. The answer is not `0` because the agent must still pay for every landed step except the one where the coupon is used.

### Example 3

**Input**

```text
1
9
```

**Output**

```text
0
```

**Explanation**

The agent lands on the only step and uses the coupon there.

## Edge-Case Thinking Prompts

- What should the function return if the current step goes beyond `n`?
- What should happen when the agent lands exactly on step `n`?
- Does the recursive call need to know whether the coupon was already used?
- Can using the coupon early be worse than saving it for a later expensive step?
- Why is this a minimum-cost problem, not a counting problem?

## Thinking Like a Programmer

Before writing code, define:

```text
State:
Choice:
Transition:
Base case:
Order:
Answer:
```

For this problem:

```text
State: current step index and whether coupon is already used
Choice: from current position, jump 1 or 2 steps
Transition: pay the landing cost, or use coupon if still available
Base case: reached step n, or moved beyond n
Order: recursive exploration from current position toward n
Answer: minimum cost among all valid paths
```
