# Lantern Ladder

## Concepts Involved

- Base Case Design
- Return Meaning
- Choice Expansion
- Recursive State Definition
- Recursion Tree Analysis
- Constraint Propagation
- Counting Recursion
- Optimization Recursion

## Problem Statement

A technician climbs a lantern tower with `n` rungs, numbered from `1` to `n`.
The technician starts before the first rung, at position `0`, and wants to land exactly on rung `n`.

From any position, the technician may climb either `1` rung or `2` rungs at a time.
Some rungs are broken and cannot be landed on.
Also, the technician is not allowed to make two `2-rung` jumps continuously.

Your task is to count how many valid climbing sequences can reach rung `n`.

A sequence is valid only if:

- Every move is either `1` or `2` rungs.
- The final position is exactly `n`.
- No landed rung is broken.
- No two consecutive moves are both `2`.

## Input Format

```text
n m
b1 b2 b3 ... bm
```

Where:

- `n` is the number of rungs.
- `m` is the number of broken rungs.
- `b1 b2 ... bm` are the broken rung numbers.

If `m = 0`, the second line may be empty or absent.

## Output Format

Print one integer: the number of valid climbing sequences.

## Constraints

```text
1 <= n <= 35
0 <= m <= n
1 <= bi <= n
All broken rung numbers are distinct
```

## Examples

### Example 1

```text
4 1
2
```

```text
1
```

### Example 2

```text
5 0
```

```text
4
```

### Example 3

```text
3 1
3
```

```text
0
```

## Edge-Case Thinking Prompts

- What should happen if the final rung is broken?
- What should happen if a jump crosses a broken rung but does not land on it?
- What should happen when the current position becomes greater than `n`?
- Why is it not enough to know only the current rung?
