# Twin Tile Corridor — Quick Solution

## Idea (Simple)

We cover the corridor from left to right.  
A length-`1` tile is always safe if space remains.  
A length-`2` tile is allowed only if the previous tile was not length `2`, and it must not become the final tile.  
So recursion must remember the remaining length and whether the previous tile was length `2`.

## Pattern Recognition Clues

- The problem asks for the number of valid arrangements, so this is counting recursion.
- Each partially covered corridor gives repeated choices: place tile `1` or tile `2`.
- Future choices depend on the previous tile because two length-`2` tiles cannot touch.
- The final-tile rule changes whether a length-`2` tile is allowed near the end.

## State Discovery

The first simple state is:

```text
solve(rem)
```

This means the number of ways to cover `rem` remaining length.

But `solve(rem)` is not enough.  
At the same remaining length, a length-`2` tile may be allowed or blocked depending on the previous tile.

The missing information is:

```text
prevWasTwo
```

So the final state is:

```text
solve(rem, prevWasTwo)
```

## Pattern

Counting Recursion with Previous Choice State

## Core Insight

- Placing a length-`1` tile is always valid when `rem >= 1`.
- Placing a length-`2` tile is blocked if the previous tile was also length `2`.
- Placing a length-`2` tile is also blocked when `rem == 2`, because then the final tile would be length `2`.

## Invariant

For every call `solve(rem, prevWasTwo)`, the already covered left part is valid, and `prevWasTwo` correctly tells whether the tile immediately before the remaining part was length `2`.

## Algorithm (with one-line why at important steps)

1. Start with `solve(n, false)`.  
   - Why: no tile has been placed before the corridor starts.
2. If `rem == 0`, return `1`.  
   - Why: one full valid tiling has been completed.
3. Always try placing a length-`1` tile when `rem >= 1`.  
   - Why: it never violates the final-tile or adjacency rule.
4. Try placing a length-`2` tile only if `prevWasTwo` is false.  
   - Why: adjacent length-`2` tiles are forbidden.
5. Also require `rem >= 3` before placing length `2`.  
   - Why: if `rem == 2`, the length-`2` tile would be the final tile.
6. Add the counts from all valid choices.  
   - Why: counting recursion combines branches using addition.

## Mental Cheat Sheet

| Item | Meaning |
|---|---|
| Pattern | Counting recursion with previous choice state |
| State | `(remainingLength, prevWasTwo)` |
| Choice | place tile `1` or valid tile `2` |
| Constraint | no adjacent `2` tiles, final tile must be `1` |
| Answer | `solve(n, false)` |

## Complexity

- Time Complexity: `O(2^n)` in pure recursion.
- Space Complexity: `O(n)` due to recursion stack.
- Auxiliary Space: `O(n)` due to recursion stack.

## Tiny Dry Run

For `n = 3`:

```text
1 + 1 + 1
2 + 1
```

The tiling `1 + 2` is invalid because the final tile is length `2`.

So the answer is `2`.

## Notes / Pitfalls

- Do not allow a length-`2` tile when `rem == 2`.
- Do not allow two length-`2` tiles back to back.
- Do not use only `solve(rem)`. The previous tile affects the next valid choices.
- This is counting recursion, so valid branch answers must be added.
