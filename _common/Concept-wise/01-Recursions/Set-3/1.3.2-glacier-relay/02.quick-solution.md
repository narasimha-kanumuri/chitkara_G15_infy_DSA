# Glacier Relay — Quick Solution

## Idea (Simple)

From each platform, try a `1`-jump and a `2`-jump.  
Each jump has height-difference cost, and blue platforms add extra cold cost when landed on.  
But a `2`-jump is not allowed if the previous jump was also `2`.  
So recursion must remember the current platform and the previous jump type.

## Pattern Recognition Clues

- The problem asks for minimum energy, so this is optimization recursion.
- Each platform creates repeated choices: jump `1` or jump `2`.
- A future move depends on the previous move because two `2`-jumps cannot be consecutive.
- Blue landing cost is a constraint that must be propagated into each transition.

## State Discovery

The first simple state is:

```text
solve(i)
```

This means minimum energy needed from platform `i` to platform `n`.

But `solve(i)` is not enough.  
At the same platform `i`, the next choices are different depending on whether the previous jump was `2`.

The missing information is:

```text
lastJumpWasTwo
```

So the final state is:

```text
solve(i, lastJumpWasTwo)
```

## Pattern

Recursive Optimization with Last Choice State

## Core Insight

- A `1`-jump is always allowed if it stays inside the platform range.
- A `2`-jump is allowed only when the previous jump was not `2`.
- The best answer is the smaller total cost among all valid choices.

## Invariant

For every call `solve(i, lastJumpWasTwo)`, the function returns the minimum extra energy needed to reach platform `n` from platform `i`, while correctly respecting whether the next `2`-jump is allowed.

## Algorithm (with one-line why at important steps)

1. Start from platform `1` with `lastJumpWasTwo = false`.  
   - Why: no jump has been taken yet.
2. If the current platform is `n`, return `0`.  
   - Why: the destination is already reached.
3. Try jump length `1`.  
   - Why: a `1`-jump is always allowed if it does not cross `n`.
4. Try jump length `2` only if `lastJumpWasTwo` is false.  
   - Why: two consecutive `2`-jumps are forbidden.
5. For every allowed landing platform, add height-difference cost.  
   - Why: every jump consumes energy.
6. Add cold penalty if the landing platform is blue.  
   - Why: blue platforms add extra cost only when landed on.
7. Return the minimum total among valid options.  
   - Why: the task asks for minimum energy.

## Mental Cheat Sheet

| Item | Meaning |
|---|---|
| Pattern | Recursion with last choice state |
| State | `(platform, lastJumpWasTwo)` |
| Choice | jump `1` or jump `2` |
| Constraint | no two consecutive `2`-jumps |
| Answer | `solve(1, false)` |

## Complexity

- Time Complexity: `O(2^n)` in pure recursion.
- Space Complexity: `O(n)` due to recursion stack.
- Auxiliary Space: `O(n)` due to recursion stack.

## Tiny Dry Run

For:

```text
n = 5, coldPenalty = 4
height = [10, 14, 9, 20, 18]
blue = [0, 1, 0, 1, 0]
```

One valid route is:

```text
1 -> 3 -> 4 -> 5
```

Cost:

```text
1 + 15 + 2 = 18
```

The route `1 -> 3 -> 5` is invalid because both jumps are length `2`.

## Notes / Pitfalls

- Do not allow a `2`-jump after another `2`-jump.
- Do not add blue cost for the starting platform.
- Do not forget to add blue cost for the final platform if the final platform is reached by a jump.
- This is minimum-cost recursion, not counting recursion.
