## Plain-English Implementation Walkthrough

1. We read the number of stones and the fatigue penalty first.
   The penalty is needed only for consecutive `2-step` jumps.
   Related code: `int n = sc.nextInt(); int penalty = sc.nextInt();`

2. We store all stone heights in an array.
   This lets the code find the height of any stone using its index.
   Related code: `int[] height = new int[n];`

3. The frog starts at stone `0`.
   No previous jump exists at the start, so `lastWasTwo` is `false`.
   Related code: `solve(0, false, height, penalty)`

4. If the current index is the last stone, no more energy is needed.
   We return `0` because the destination is already reached.
   Related code: `if (i == height.length - 1) return 0;`

5. We keep a large initial value for `best`.
   This helps us safely take the minimum among valid jumps.
   Related code: `long best = INF;`

6. We try the `1-step` jump only if `i + 1` exists.
   Its cost is only the height difference.
   Related code: `if (i + 1 < height.length) ...`

7. After a `1-step` jump, the next call gets `lastWasTwo = false`.
   This is because the latest jump was not a `2-step` jump.
   Related code: `solve(i + 1, false, height, penalty)`

8. We try the `2-step` jump only if `i + 2` exists.
   Its base cost is the height difference between stone `i` and stone `i + 2`.
   Related code: `if (i + 2 < height.length) ...`

9. If the previous jump was also `2-step`, we add the fatigue penalty.
   This is the main twist in the problem.
   Related code: `if (lastWasTwo) twoStepCost += penalty;`

10. After a `2-step` jump, the next call gets `lastWasTwo = true`.
    This lets the future call know whether another `2-step` jump should be penalized.
    Related code: `solve(i + 2, true, height, penalty)`

11. We return the smallest valid total cost.
    That value is the minimum energy from the current stone to the end.
    Related code: `return best;`

## Complete Solution Code

```java
import java.util.Scanner;

public class Main {
    private static final long INF = Long.MAX_VALUE / 4;

    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);

        int n = sc.nextInt();
        int penalty = sc.nextInt();

        int[] height = new int[n];
        for (int i = 0; i < n; i++) {
            height[i] = sc.nextInt();
        }

        long answer = solve(0, false, height, penalty);
        System.out.println(answer);

        sc.close();
    }

    private static long solve(int i, boolean lastWasTwo, int[] height, int penalty) {
        if (i == height.length - 1) {
            return 0;
        }

        long best = INF;

        if (i + 1 < height.length) {
            long oneStepCost = Math.abs(height[i] - height[i + 1]);
            long total = oneStepCost + solve(i + 1, false, height, penalty);
            best = Math.min(best, total);
        }

        if (i + 2 < height.length) {
            long twoStepCost = Math.abs(height[i] - height[i + 2]);
            if (lastWasTwo) {
                twoStepCost += penalty;
            }

            long total = twoStepCost + solve(i + 2, true, height, penalty);
            best = Math.min(best, total);
        }

        return best;
    }
}
```

## Code Deconstruction

- `height` stores the height of each stone.
- `solve(i, lastWasTwo, height, penalty)` returns the minimum energy needed from stone `i` to the last stone.
- `i == height.length - 1` is the success base case.
- `best` stores the cheapest valid jump result from the current stone.
- The `1-step` block checks whether stone `i + 1` exists before jumping.
- `oneStepCost` uses `Math.abs` because height difference cannot be negative energy.
- The `2-step` block checks whether stone `i + 2` exists before jumping.
- `if (lastWasTwo)` adds the penalty only for consecutive `2-step` jumps.
- The recursive call after a `2-step` jump passes `true` for `lastWasTwo`.
- The method returns `best`, which is the minimum among all valid choices.

## Language-Specific Notes

- `long` is used for total energy to avoid overflow in larger variations.
- `Math.abs(int)` returns an `int`, but it is safely stored in a `long` variable.
- This Day 1 solution intentionally uses plain recursion to show the decision tree.
- The same state can repeat many times. Day 2 can optimize this using memoization with state `(i, lastWasTwo)`.
- The recursion depth is at most `n`, so it is safe for the given constraints.

## Complexity Summary

- Time Complexity: `O(2^n)` in the worst case without memoization
- Space Complexity: `O(n)` because the deepest recursion path can have `n` calls
- Auxiliary Space: `O(n)` for the recursion call stack
