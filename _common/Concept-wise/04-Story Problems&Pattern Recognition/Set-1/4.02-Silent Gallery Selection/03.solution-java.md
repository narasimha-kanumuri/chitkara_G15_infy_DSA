## Plain-English Implementation Walkthrough
1. We read `n` first because it tells us how many exhibit values follow.

2. We store exhibit values in a `long[]`.
   We use `long` because the maximum total can be larger than `int`.

3. If there is only one exhibit, we return its value.
   The circular conflict does not matter here.

4. If there are two exhibits, we return the larger value.
   Both exhibits are adjacent, so we cannot select both.

5. For larger input, we cannot solve the whole array directly.
   The first and last exhibits are adjacent.

6. We solve one case from index `0` to `n - 2`.
   This keeps the first exhibit available but excludes the last exhibit.

7. We solve another case from index `1` to `n - 1`.
   This excludes the first exhibit and keeps the last exhibit available.

8. The helper method uses two rolling values.
   `prevOne` means the best value up to the previous position.
   `prevTwo` means the best value before that.

9. For each exhibit inside the helper, we compute two options.
   Taking the current exhibit uses `prevTwo + values[i]`.
   Skipping it keeps `prevOne`.

10. We return the better of the two circular cases.
    This covers every valid solution without allowing first and last together.

## Complete Solution Code
```java
import java.io.BufferedReader;
import java.io.InputStreamReader;
import java.util.StringTokenizer;

public class Main {
    private static long bestLinear(long[] values, int start, int end) {
        long prevTwo = 0L;
        long prevOne = 0L;

        for (int i = start; i <= end; i++) {
            long take = prevTwo + values[i];
            long skip = prevOne;
            long current = Math.max(take, skip);

            prevTwo = prevOne;
            prevOne = current;
        }

        return prevOne;
    }

    private static long bestGallerySelection(long[] values) {
        int n = values.length;

        if (n == 1) {
            return values[0];
        }

        if (n == 2) {
            return Math.max(values[0], values[1]);
        }

        long excludeLast = bestLinear(values, 0, n - 2);
        long excludeFirst = bestLinear(values, 1, n - 1);

        return Math.max(excludeLast, excludeFirst);
    }

    public static void main(String[] args) throws Exception {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));

        int n = Integer.parseInt(br.readLine().trim());
        long[] values = new long[n];

        StringTokenizer st = new StringTokenizer(br.readLine());
        for (int i = 0; i < n; i++) {
            values[i] = Long.parseLong(st.nextToken());
        }

        System.out.println(bestGallerySelection(values));
    }
}
```

## Code Deconstruction
- `bestLinear(values, start, end)`
  - Solves the non-adjacent selection problem on a straight range.
  - It does not know or care about circular adjacency.

- `prevTwo`
  - Stores the best value before the previous exhibit.
  - It is used when taking the current exhibit.

- `prevOne`
  - Stores the best value up to the previous exhibit.
  - It is used when skipping the current exhibit.

- `take = prevTwo + values[i]`
  - If we take the current exhibit, we must use the best answer before the previous exhibit.

- `skip = prevOne`
  - If we skip the current exhibit, the best answer remains the previous best.

- `excludeLast`
  - Solves the case where the last exhibit is not selected.

- `excludeFirst`
  - Solves the case where the first exhibit is not selected.

- Final `Math.max`
  - Chooses the better circular-safe selection.

## Language-Specific Notes
- `long` is safer than `int` for large totals.
- The helper uses rolling variables instead of an array, so memory stays constant.
- The input code assumes all `n` values are on one line.
- Index ranges are inclusive in the helper method.

## Complexity Summary
- Time Complexity: `O(n)`
- Space Complexity: `O(1)`
- Auxiliary Space: `O(1)`
