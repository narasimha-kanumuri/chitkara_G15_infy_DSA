# Coupon Climb — Quick Solution

## Idea (Simple)

At every step, the agent can jump to the next step or skip one step.  
When the agent lands on a step, that step has a cost.  
The coupon adds one extra decision: pay the cost or cancel it once.  
So recursion must remember both the current position and whether the coupon is already used.

## Pattern Recognition Clues

- The problem asks for a minimum total cost, so this is optimization recursion.
- From each position, there are repeated choices: jump `1` or jump `2`.
- A future decision depends on whether the coupon has already been used.
- This means the state must carry coupon status, not only the step number.

## State Discovery

The first simple state is:

```text
solve(i)
```

This means: minimum cost to reach step `n` from position `i`.

But `solve(i)` is not enough.  
At the same step `i`, the answer may be different depending on whether the coupon is still available.

The missing information is:

```text
couponUsed
```

So the final recursive state is:

```text
solve(i, couponUsed)
```

Where:

- `i` is the current position.
- `couponUsed = 0` means the coupon is still available.
- `couponUsed = 1` means the coupon is already used.

## Pattern

Recursive Optimization with Extra Decision State

## Core Insight

- Every landing step creates a payment decision.
- If the coupon is unused, the agent has two cost choices: pay or use coupon.
- The answer is the minimum among all valid jump and coupon choices.

## Invariant

For every recursive call `solve(i, couponUsed)`, the function returns the minimum extra cost needed to reach exactly step `n` from current position `i`, respecting the coupon status.

## Algorithm (with one-line why at important steps)

1. Start from position `0`, before the first step.  
   - Why: the agent has not landed on any step yet.
2. Try jumping `1` step and `2` steps.  
   - Why: these are the only allowed moves.
3. Ignore a jump if it goes beyond `n`.  
   - Why: the agent must reach exactly step `n`.
4. For every valid landing step, first try paying its cost.  
   - Why: paying is always allowed.
5. If the coupon is unused, also try using the coupon on that landing step.  
   - Why: this may reduce the total cost.
6. Take the minimum over all choices.  
   - Why: the task asks for minimum total cost.
7. If the current position is exactly `n`, return `0`.  
   - Why: no more cost is needed after reaching the top.

## Mental Cheat Sheet

| Item | Meaning |
|---|---|
| Pattern | Recursive optimization with state |
| State | `(position, couponUsed)` |
| Choice | jump `1` or `2`; pay or use coupon |
| Invalid case | position goes beyond `n` |
| Answer | `solve(0, 0)` |

## Complexity

- Time Complexity: `O(4^n)` in the pure recursive version.
- Space Complexity: `O(n)` recursion stack depth.
- Auxiliary Space: `O(n)` for recursion stack.

## Tiny Dry Run

For:

```text
n = 3
cost = [5, 100, 6]
```

Start at position `0`.

Possible useful path:

```text
0 -> 1 -> 3
```

- Pay step `1`: cost `5`.
- Use coupon on step `3`: cost `0`.
- Total = `5`.

The recursive function compares this with all other valid paths and returns `5`.

## Notes / Pitfalls

- Do not return `0` just because the coupon exists. Only one landed step can be free.
- Do not forget the exact landing rule. Jumping beyond `n` is invalid.
- Do not use only `solve(i)`. Coupon status changes the future answer.
