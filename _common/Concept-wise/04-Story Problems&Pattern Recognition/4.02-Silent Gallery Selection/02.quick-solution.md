## Idea (Simple)
At every room, we have two choices:
- skip this room
- open this room

If we open room `i`, we cannot open room `i - 1`.
So the best total at `i` comes from either:
- the best answer till `i - 1`
- or the best answer till `i - 2` plus `impact[i]`

Because negative values are allowed, opening no room must also remain a valid possibility.

## Pattern
1D Dynamic Programming — Take / Skip

## Core Insight
- `dp[i]` should mean: maximum total impact using rooms from index `0` to `i`.
- If we skip room `i`, answer stays `dp[i - 1]`.
- If we open room `i`, answer becomes `dp[i - 2] + impact[i]`.
- Since we may choose no room, the running answer should never go below `0`.

## Invariant
After processing index `i`, `dp[i]` stores the maximum valid impact obtainable from the prefix `0...i` under the non-adjacent rule, while still allowing the option of choosing no room.

## Algorithm (with one-line why at important steps)
1. If the array is empty, return `0`.
   - Why: no rooms means no impact.
2. Set `dp[0] = max(0, impact[0])`.
   - Why: we may either open the first room or skip it.
3. Set `dp[1] = max(dp[0], impact[1], 0)` when `n >= 2`.
   - Why: we cannot take both adjacent rooms.
4. For each index `i` from `2` to `n - 1`:
   - `skip = dp[i - 1]`
   - `take = dp[i - 2] + impact[i]`
   - `dp[i] = max(skip, take, 0)`
   - Why: every optimal answer must either take or skip the current room.
5. Return `dp[n - 1]`.
   - Why: it stores the best valid answer for the full array.

## Code Cheat Sheet
| Code / Built-in | Meaning / Role |
|---|---|
| `dp[i]` | best valid impact till index `i` |
| `skip` | answer if current room is ignored |
| `take` | answer if current room is opened |
| `Math.max(...)` | selects the better choice |
| `long` | prevents overflow for large totals |

## Complexity
- Time: `O(n)`
- Space: `O(n)`
- Auxiliary Space: `O(n)`

## Tiny Dry Run
For `impact = [4, 2, 7, 3, 9]`:
- `dp[0] = 4`
- `dp[1] = max(4, 2, 0) = 4`
- `dp[2] = max(4, 4 + 7, 0) = 11`
- `dp[3] = max(11, 4 + 3, 0) = 11`
- `dp[4] = max(11, 11 + 9, 0) = 20`
Answer = `20`

## Notes / Pitfalls
- Do not greedily pick every positive room; adjacency can block a better future total.
- For negative arrays, the answer can be `0`.
- Base cases for `n = 1` and `n = 2` must be handled carefully.
- Use `long` in Java because sums can exceed `int` range.
