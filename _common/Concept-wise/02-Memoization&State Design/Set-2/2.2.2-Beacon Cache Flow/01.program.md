# Beacon Cache Flow

## Concepts Involved

Memoization, Recurrence Optimization, State Design, Top-Down DP, Big Integer Arithmetic

## Problem Statement

A lighthouse beacon produces a brightness value every day.

The brightness follows these rules:

- On day `0`, brightness is `1`.
- On day `1`, brightness is `2`.
- For every day `i >= 2`, brightness is first computed as:

```text
brightness(i) = brightness(i - 1) + brightness(i - 2)
```

There is one periodic correction rule:

- If `i` is a multiple of `4`, subtract `1` from that day's computed brightness.

A direct recursive solution will recompute the same day values many times.
Use memoization to compute the brightness of day `n` efficiently.

Return the brightness value on day `n`.

## Input Format

- A single integer `n`

## Output Format

- Print the brightness value on day `n`.

## Constraints

- `0 <= n <= 2000`
- The answer may be very large.
- Use exact large number arithmetic.

## Examples

### Example 1

Input:

```text
0
```

Output:

```text
1
```

Explanation:

Day `0` is a base day, so brightness is `1`.

### Example 2

Input:

```text
1
```

Output:

```text
2
```

Explanation:

Day `1` is a base day, so brightness is `2`.

### Example 3

Input:

```text
4
```

Output:

```text
7
```

Explanation:

Brightness values are:

```text
brightness(0) = 1
brightness(1) = 2
brightness(2) = 3
brightness(3) = 5
brightness(4) = brightness(3) + brightness(2) - 1
              = 5 + 3 - 1
              = 7
```

### Example 4

Input:

```text
5
```

Output:

```text
12
```

Explanation:

```text
brightness(5) = brightness(4) + brightness(3)
              = 7 + 5
              = 12
```

Day `5` is not a multiple of `4`, so no correction is applied.

## Edge-Case Thinking Prompts

- What should be returned for `n = 0`?
- What should be returned for `n = 1`?
- When exactly should the `-1` correction be applied?
- Why does plain recursion recompute the same day values?
- Is the day index alone enough as a memoization state?
